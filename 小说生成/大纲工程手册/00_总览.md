# 大纲工程手册 v1.2

> 整理自 15 个实战验证skill，覆盖网文大纲从零创建到质量优化的全流程。  
> 适用规模：万行级 / 500章+ / 多卷（5卷以上）网文项目。

| 场景 | 入口工作流 | 核心任务 |

## 通用工具链

| 工具 | 用途 |
|------|------|
| `execute_code` + Python | 大文件解析、批量正则、统计验证 |
| `patch` | 精准替换（需唯一old_string） |
| `write_file` | 文件整体写入（大于12KB用execute_code代写） |
| `read_file` | 分段读取大文件（offset/limit） |
| `search_files` | 全项目关键词搜索 |
| `terminal` | 批量shell命令（wc/grep/find） |

---

## 铁律（所有场景通用）

1. **先读后动** — 不读完大纲不动手
2. **patch必须验证** — patch后立即 `wc -l` 确认行数变化
3. **章节号必须为正整数** — 禁止小数章节号（如Ch305.5）
4. **联动修改** — 改大纲必须同步更新伏笔线、爽点表、时间线
5. **备份优先** — 大改之前先 `cp` 备份

---

## 坑点速查

| 坑 | 现象 | 解法 |
|----|------|------|
| write_file 0字节 | 大文件直接写失败 | 用execute_code的open()写入 |
| patch非唯一 | old_string在文件出现多次 | 加前后文（3行以上）使其唯一 |
| 子目录路径 | 绝对路径引用错误 | 使用前验证路径是否存在 |
| 章节重复副本 | 多次追加导致同一章节号出现多个 | 用去重流程清理 |
| 格式不统一 | 各卷字段名不一致 | 先扫描实际标签分布，再针对性修改 |

# 大纲工程手册 v2.0

> 从"一次生成"到"逐层收敛"的7阶段渐进式大纲生成流水线。
> 核心理念：每一层只做一件事，每一层人确认后再往下走。
> 适用规模：万行级 / 500章+ / 多卷（5卷以上）网文项目。
> 最后更新：2026-04-30

---

## 一、v2.0 主流程速查

```
Phase 1: 素材征集    →  Phase 2: 内核定型    →  Phase 3: 分卷骨架
（人机交互收集）         （AI生成+人拍板）         （AI生成+人审）
      ↓门禁1                  ↓门禁2                  ↓门禁3

Phase 4: 剧情块落地   →  Phase 5: 章节细纲     →  Phase 6: 大纲全维审核
（AI填充+人审细节）       （AI逐章展开+人审）        （五视角深度审查）
      ↓门禁4                  ↓门禁5                  ↓门禁6

Phase 7: 审核修复闭环
（修复→复审→归档）
      ↓门禁7
  大纲定稿，移交正文写作
```

| Phase | 核心问题 | 人审要点 | 主文档 |
|-------|---------|---------|--------|
| **P1 素材征集** | 作家到底想写什么？ | 核心理念清不清晰？ | [WF-06_P1](WF-06_P1_素材征集.md) |
| **P2 内核定型** | 这本书的创作纲领是什么？ | 题材定位+世界观+禁区分明？ | [WF-06_P2](WF-06_P2_内核定型.md) |
| **P3 分卷骨架** | 全书怎么分卷？卷间逻辑？ | 节奏合理？卷间钩子有吸引力？ | [WF-06_P3](WF-06_P3_分卷骨架.md) |
| **P4 剧情块落地** | 每卷怎么分成幕/块？ | 每个剧情块能在脑中"看到"画面吗？ | [WF-06_P4](WF-06_P4_剧情块落地.md) |
| **P5 章节细纲** | 每一章怎么写？ | 这章给写手能直接写吗？ | [WF-06_P5](WF-06_P5_章节细纲.md) |
| **P6 全维审核** | 大纲有什么隐藏问题？ | 事实/爽感/市场/伏笔都过关？ | [WF-06_P6](WF-06_P6_大纲全维审核.md) |
| **P7 修复闭环** | 审核发现的问题怎么改？ | 修复后复审通过了吗？ | [WF-06_P7](WF-06_P7_审核修复闭环.md) |

## 四、通用工具链

| 工具 | 用途 |
|------|------|
| `execute_code` + Python | 大文件解析、批量正则、统计验证 |
| `patch` | 精准替换（需唯一old_string） |
| `write_file` | 文件整体写入（大于12KB用execute_code代写） |
| `read_file` | 分段读取大文件（offset/limit） |
| `search_files` | 全项目关键词搜索 |
| `terminal` | 批量shell命令（wc/grep/find） |

## 五、铁律（所有场景通用）

1. **先读后动** — 不读完上一Phase产出不动手
2. **逐层收敛** — 每一层只做一件事，确认后解锁下一层
3. **门禁先行** — 每道门禁不通过，不得进入下一Phase
4. **patch必须验证** — patch后立即 `wc -l` 确认行数变化
5. **章节号必须为正整数** — 禁止小数章节号（如Ch305.5）
6. **回退范围最小化** — 门禁失败只回退到对应Phase，不波及已完成部分
7. **备份优先** — 大改之前先 `cp` 备份

## 六、v2.0 新概念速查

| 概念 | 说明 | 来源 |
|------|------|------|
| 倒果为因 | 先定落点（读者看完后的画面），再反推前置铺垫 | wiki/concepts/倒果为因写作法.md |
| 九线写法 | 主角线/势力线/感情线等9条线同时推进，每层检查是否掉线 | wiki/concepts/网文九线写法.md |
| 剧情块 | 介于"幕"和"章"之间的粒度——一个完整的事件团（2-5章） | v2.0新增 |
| 状态追踪 | 每卷维护主角/配角/世界的当前状态快照，AI生成时作为前置约束 | wiki/concepts/AI大纲一致性管理.md |
| 爽感类型 | 公平需求/优越需求/掌控需求/归属需求 四种心理满足类型 | wiki/concepts/爽感四大基石.md |
| 门禁 | 每个Phase结束后的强制审核关口，不通过不能进入下一步 | v2.0新增 |
| 五视角审核 | 监督者/挑剔读者/编辑/伏笔审计/终审，5个视角独立审查 | wiki/concepts/四层审核体系P0-P3.md |

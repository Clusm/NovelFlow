# WF-06-P4：剧情块落地

> v2.0 大纲生成流水线 · Phase 4
> 上一Phase：[P3 分卷骨架](WF-06_P3_分卷骨架.md) | 下一Phase：[P5 章节细纲](WF-06_P5_章节细纲.md)
> 版本：v2.0 | 最后更新：2026-04-30

---

**目标**：把每卷从"骨架"展开为"幕级 + 剧情块"的可执行蓝图。

**新概念——剧情块**：介于"幕"和"章"之间的粒度。一个剧情块是一个完整的事件团（通常2-5章），有自己的"小落点"。剧情块是 Phase 5 展开细纲的直接输入。

**耗时**：每卷AI生成约3分钟 + 人审每卷约5分钟

---

## Step 4.1：AI为每卷生成「剧情块」

**AI使用的wiki知识**：
- `网文架构师` — 三幕式/五幕式/起承转合的节拍点（激励事件/中点转折/高潮/结局）
- `倒果为因` — 幕级落点反推 + 递进铺垫设计
- `爽感四大基石` — 每块爽感类型标注 + 写法要点
- `叙述技巧体系` — 幕间衔接
- `读者情绪地图` — 每块的情绪目标
- `节奏模组` — 单章节奏公式（压抑5%/蓄势20%/转机10%/爆发15%/余韵50%）应用于剧情块的块内情绪比例设计；悬念接力法指导块间钩子类型选择

**产出物**：`04_第N卷_剧情块.md`

**每卷按幕展开，每幕包含**：

| 字段 | 说明 |
|------|------|
| 幕目标 | 这一群章节要完成什么 |
| 幕级落点 | 读者看完这一幕最想记住的画面（倒果为因） |
| 幕的情绪曲线 | 起始情绪→中段变化→转折情绪→幕末情绪+钩子 |
| 剧情块列表 | 每个块包含以下子字段 |

**每个剧情块包含的子字段**：

| 子字段 | 说明 |
|--------|------|
| 块名 + 章节范围 | 如"穿越反杀（Ch1-Ch5）" |
| 核心事件 | 一句话描述 |
| 爽感类型+写法要点 | 类型（如"优越需求·信息差碾压"）+ 具体写法令 |
| 倒推铺垫 | 这一块的落点需要前面哪几章铺垫什么 |
| 块内情绪曲线 | 起→承→转→合（每个节点的情绪描述）。起=建立场景+抛出事件、承=矛盾积累+压力上升、转=爆发点+爽感释放、合=收束+钩子。此为起承转合在剧情块级的应用 |
| 章末钩子类型 | 反转式/选择式/悬念式/情感式 |
| 参与角色+动机 | 本块出场的所有角色及其动机 |
| 伏笔操作 | 埋设：[Fxxx+内容]、回收：[Fxxx+回收方式] |
| 九线推进 | 本块推进了哪几条线 |

**幕间衔接**：第一幕结束→第二幕开始的钩子类型+过渡方式（直接切入/时间跳跃/视角切换/信息揭示）

## Step 4.2：AI为每卷初始化「状态追踪表」

**AI使用的wiki知识**：
- `AI大纲一致性管理_工程控制论视角` — 状态变量五大类别 + 数字孪生

**产出物**：`05_第N卷_状态追踪.md`

**必须包含的字段**：

| 表 | 字段 |
|----|------|
| 主角状态快照 | 实力等级/身体状态（含伤势限制）/心理状态/当前位置/金手指解锁项/可用资源/关键人际关系 |
| 女主状态快照 | 实力等级/当前位置/与主角关系阶段 |
| 反派/配角状态 | 角色名/当前状态/本卷动作 |
| 本卷约束清单 | ⚠️铁律 + ⚠️主角身体限制 + ⚠️金手指限制 |
| 需回收的旧伏笔 | ID/来源/内容/承诺回收章/剩余章数/状态（⏳/⚠️即将到期） |

## 🚪 门禁4：剧情块评审

| # | 检查项 | 不通过的表现 |
|---|--------|------------|
| 1 | 幕的章节占比符合结构模型？ | 第一幕占80%的章 |
| 2 | 每个剧情块能在脑中"看到"画面？ | 模糊，看不清 |
| 3 | 每块核心事件够具体？ | "主角调查事件" = 太模糊 |
| 4 | 爽点标注明确，写法要点可执行？ | 只说"这里要爽"，没说怎么爽 |
| 5 | 幕间衔接自然？ | 断裂/跳跃 |
| 6 | 一事不烦二主？ | 新角色/设定无后续作用 |
| 7 | 状态追踪约束够硬？ | 有主角受伤但某块安排了全力战斗 |

**通过标准**：全部 ✅ → 进入 [Phase 5](WF-06_P5_章节细纲.md)。

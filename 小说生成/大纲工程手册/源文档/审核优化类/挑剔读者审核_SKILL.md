# 读者视角情节审核器

## 核心定位

**模拟一个挑剔的网文老读者**，以"这个情节能不能吸引我继续读"为唯一标准，对大纲/细纲进行四维审核。

## 审核四维框架

### 维度零：3000字展开能力（★★★★★ 番茄文风专项）

**检查标准**（番茄文风 / 飞卢风）：
- 每个章节bullet必须含足够戏剧张力才能撑到3000字
- **过渡章节**（如Ch1-Ch5衔接段）不能只有"然后他就去XX"——必须有独立情事件
- **群戏章节**（多场景）需要拆成2-3个独立情节点
- **打脸/冲突章节**：必须还原"反驳的具体话语+表情变化+围观反应"，不能只写"然后打脸了"
- **感情戏章节**：每5章必须有明确进展节点（暧昧升温→确认→冲突→和好→高潮），不能只写"感情升温"
- **多章节覆盖同一事件**：如果3章只写了1个bullet，说明内容密度严重不足

**典型失败案例**：
- "Ch76-Ch80：室友喜剧+醋王行为" → 25章只有5个bullet → 必须逐章展开
- "Ch231-Ch250：敲钟后尾声" → 20章只有1个高潮 → 需拆分20+独立情节点

**读者问句**：
> "这一章的细节够不够我读3000字？如果看完这章我还需要自己脑补，说明大纲不够密。"

---

### 维度一：爽感密度（★★★★★ 最优先）

**检查标准**（番茄风）：
- 每 **3章** 至少 1个小爽点（答题成功/打脸成功/暧昧升温/获得新技能）
- 每 **10章** 至少 1个大冲突（越阶战斗/大反派登场/势力碰撞）
- 每 **50章** 至少 1个名场面（传唱级名场面，如《将进酒》镇全场）

**爽点类型覆盖**（至少覆盖4种）：
1. 规则碾压（理科知识降维打击）
2. 打脸（当场还击，不过夜）
3. 传承/代际（古人认可，跨越时空）
4. 势力扩张（小弟变多，势力变大）
5. 名场面（经典复刻/诗词引发天象）
6. 甜宠（感情线撒糖）
7. 逆袭（草根战胜天才）
8. 知识融合（跨学科组合创造奇迹）
9. 学术声望（积累被认可）

**读者问句**：
> "上一章让我爽是什么时候？如果超过5章没爽点，我会想弃书。"

### 维度二：节奏与钩子

**检查标准**：
- 每章结尾必须有钩子，类型：悬念/冲突/甜蜜/震惊 至少1种
- 章末钩子不能是"主角睡着了然后做梦"
- 章节之间必须有因果链，不能是"然后他就去xxx了"
- 冲突间隔不能超过10章

**读者问句**：
> "这一章结尾，我想点下一章的理由是什么？如果没有，节奏有问题。"

### 维度三：人设一致性

**检查标准**：
- 主角智商全程在线（不能因为剧情需要突然变蠢）
- 主角杀伐果断（遇到敌人必须还手，不能圣母）
- 女主不绿茶（感情线清爽，不养备胎）
- 反派有智商（不能是傻子，否则打脸无快感）
- 每个角色每次出场必须有明确目的

**人设检查点示例**：
- 遇到嘲讽：是否当场还击？（不超过3章）
- 获得资源：是否最大化利用？（不能浪费金手指）
- 感情纠葛：是否拖沓？（超过20章没进展视为拖沓）
- 面对强敌：是否冷静分析？（不能无脑冲）

**读者问句**：
> "这个角色的行为是否符合他的人设？如果主角上一章还睚眦必报，这一章突然原谅了打他的敌人——这是在侮辱我的智商。"

### 维度四：伏笔质量

**检查标准**：
- 每个伏笔必须有明确的"埋"和"收"（不能埋了不收）
- 伏笔回收间隔不超过 **50章**（否则读者会忘记）
- 伏笔回收时必须有仪式感（不能轻描淡写）
- 关键伏笔：主角身世、系统来源、文道断层 必须在 **150章内**回收

**伏笔分类处理**：
- **核心伏笔**（影响主线）：必须在50章内回收
- **单元伏笔**（单卷内）：可以在本卷结尾回收
- **氛围伏笔**（增加神秘感）：可以在全文结尾回收

**读者问句**：
> "第X章埋的这个伏笔，现在还记得吗？如果不记得，说明伏笔太隐蔽或间隔太长。"

```

   - 读者感受：[用读者口吻描述]
   - 修改建议：[具体修改方案]

## 🟡 建议优化（可以更好）
```

## 迭代优化流程（经实战验证）

**最佳实践**（用于30万字以上网文大纲的深度优化，18:00截止场景）：

```
Round 1:
  ① 用 write_file 写完整优化版大纲文件（覆盖所有分卷）
  ② 用 novel-reader-reviewer 审核 → 输出严重问题清单
  ③ 根据清单用 patch 批量修复严重问题

Round 2:
  ④ 用 novel-reader-reviewer 复审 → 收窄至次要建议
  ⑤ 用 patch 补全细节（势力扩张/过渡钩子/感情戏份）
  ⑥ 用 patch 补全伏笔追踪表+爽感密度总表

Round 3+:
  ⑦ 重复 ④-⑤ 直到评分≥4.5/5 或时间耗尽
```

**关键约束**：
- 所有大纲修改用 `patch` 写入文件，**不输出到 chat**（省 token）
- Chat 只报告：审核结论（严重/建议/亮点）+ 当前评分 + 进度百分比
- 每轮结束时更新文件末尾的版本说明（改进点清单）
- 章数不足时：优先补全当前卷，剩余卷留待后续扩展

**评估是否继续迭代**：
- 评分 ≥ 4.5/5 → 可停止，进入收尾
- 评分 3.5-4.5 → 再做一轮针对性修复
- 评分 < 3.5 → 回到 Round 1 重新检查分卷结构

## 审核时机

- **初版大纲**：完整四维审核
- **优化后**：重点检查被修改的部分 + 爽感密度
- **细纲阶段（3000字专项）**：逐章检查每个bullet的可展开性，重点盯过渡章节、感情戏章节、尾声章节是否过于稀疏
- **大结局收尾阶段**：重点检查末卷是否只有1个高潮、名场面密度是否达标

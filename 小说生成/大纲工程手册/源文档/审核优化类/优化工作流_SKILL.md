# 网文大纲系统性优化流程（16步标准版）

适用于：《文道至尊》《数学太凶猛》等万行级多卷项目

---

## 核心原则

1. **先审计后动手** — 步骤1-7建立全局视图，步骤8才出优化方案
2. **自驱动执行** — 用户授权后，每步完成立即发通知，不等待确认
3. **补丁优先于重写** — 用patch精准修改，不大面积重写
4. **每步交付物明确** — 步骤完成后发送"✅ 步骤X 完成"通知
5. **P0问题立即修** — 高优先级问题在发现步骤立即修复，不等最后

---

## 16步标准流程

### 步骤1-7：初始审计
- 扫描项目文件结构（`wc -l`、`du -sh`）
- 识别分卷结构、7要素覆盖、伏笔链、爽点分布
- 产出：综合审核报告（含P0/P1/P2问题清单）

### 步骤8：创建5卷优化设计文档
- 每卷一个优化文档，列出该卷的问题和修复方案
- 格式：问题ID + 描述 + 修复方式 + 优先级

### 步骤9：执行P0/P1/P2补丁
- 按优先级批量patch修复（高优先级先修）
- 验证：`wc -l`确认行数变化
- 产出：综合审核报告更新（标记已修复项）

### 步骤10：7要素补全
- 扫描全卷细纲，识别7要素缺口（情节主线/金手指/冲突/爽点/情绪/伏笔/钩子）
- 优先补全高价值章节（名场面所在章节）
- 操作：读取原文件 → patch替换简表格式章节为完整7要素
- 已知坑：patch的old_string必须唯一，必要时加前后文

### 步骤11：名场面标注补充
- 扫描全卷，识别有"名场面"描述但缺少【名场面】标注符的章节
- 批量patch添加【名场面】标注符
- 重点：S级历史时刻章节

### 步骤12：挑剔读者视角四维审核
- 爽感：爽点密度够不够？打脸够不够狠？
- 节奏：章节详略是否合理？高能情节是否扎堆？
- 人设：主角是否有缺陷？配角是否脸谱化？
- 伏笔：伏笔回收是否及时？是否有断裂？
- 产出：四维评分 + 问题清单 + 高优先级修复

### 步骤13：编辑视角市场适应性审核
- 对标目标平台（番茄/起点/飞卢）
- 评估：开篇节奏、爽点密度、感情线、市场差异化
- 产出：市场适应性报告 + 修复建议

### 步骤14：伏笔链审计
- 逐条核对F001-F0XX伏笔表，检查：埋设章节→回收章节→状态
- 识别断裂伏笔（埋设后未回收）和缺失伏笔
- 产出：伏笔链完整性评分 + F0XX问题修复

### 步骤15：生成最终综合审核报告
- 汇总前14步所有发现和修复
- 输出：问题修复清单（高/中/低优先级）
- 输出：各维度质量评分

### 步骤16：最终复审评分交付
- 最终评分卡（各维度满分5.0）
- 交付清单确认（文件数、行数）
- 声明第一轮优化完成 + 第二轮优化方向建议

## 常见高优先级修复（执行优先级）

| 问题类型 | 典型表现 | 修复方式 |
|---------|---------|---------|
| S级过密 | 某10章区间有5+个S级 | 插入ChXX.5温情缓冲章节 |
| 逝世前铺垫不足 | 卷末角色逝世缺乏情感铺垫 | 增加4-8个温情缓冲章节 |
| 失败场景缺失 | 主角只有困难→解决，无真正失败 | 增加1个ChXX.5失败场景 |
| 配角背景不完整 | 配角有故事但未揭示 | 在相关情节章节补充背景 |
| 伏笔断裂 | 埋设后未回收 | 在相关章节补充揭示 |
| 开篇爽点不够 | 前50章爽点稀疏 | 增加Ch3.5等小爽场景 |
| 时间线不完整 | 年份/时间跳跃 | 补充F_时间线.md |

## 已知坑点

### patch失败常见原因
1. **old_string不唯一** — 同一文本在文件出现多次
   - 解决：加前后文（至少3行上下文）使其唯一
   - 验证：先用`search_files`定位确认唯一性

2. **章节号重复** — patch后出现两个相同章节号
   - 解决：patch后立即`read_file`确认当前状态

3. **old_string含特殊字符** — 括号、星号等
   - 解决：用`read_file`读取原文获取精确字符

### 7要素补全识别方法
- 简表格式（章节/事件/爽点/情绪）= 缺少完整7要素
- 完整格式（含【金手指应用】【冲突对手】等标题）= 7要素齐全

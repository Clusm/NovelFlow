# 大纲大幅度修改技能

## 适用场景
对万行级小说大纲进行大幅度改造（涉及多章节重写、伏笔线调整、人物弧光翻转）。当用户说"让剧情更有新意"、"剧情审校"、"优化建议"时使用。

## 核心原则
1. **先通读再动** — 完整读所有细纲和相关知识库文件，建立全局视图
2. **识别最大问题** — 找出"套路感最强"的地方，优先改那里
3. **修改即验证** — 每改完一个文件，用`grep`或`wc -l`确认改动生效
4. **伏笔线同步更新** — 结构性的变化（如女主变主动）必须同步更新B_伏笔线.md
5. **先建立三表再动手** — 读完细纲后先用三表检测法识别所有P0/P1问题，列出修改清单再执行
6. **扩充前先修复矛盾** — 先消除261章中的矛盾重复，再规划扩充到500章

## 工作流

### 第一步：通读所有细纲
```bash
wc -l 项目/细纲/*.md  # 快速了解各卷体量
# 并行读取，用offset/limit分段读大文件
```

### 第二步：识别问题，列出改造方案
格式：
| 改造点 | 涉及卷 | 核心变化 | 工作量 |
|--------|--------|---------|--------|
| 某个新意点 | ChXX-ChYY | 从X变成Y | 大/中/小 |

### 第三步：执行修改（优先级顺序）
1. 最小改动验证路径（如先加一个伏笔点）
2. 中等锦上添花（如新增名场面）
3. 大幅度重写（如17章重写女主设局）

### 第四步：伏笔线同步
改完结构后，必须更新B_伏笔线.md：
- 新增已回收伏笔 → 第一部分
- 新增已种伏笔 → 第二部分
- 修正章节号 → 同步更新

### 第五步：验收
```bash
# 关键段落抽检
grep -n "关键词" 项目/细纲/某卷细纲.md | head -10
# 文件行数确认
wc -l 项目/细纲/*.md
```

## 伏笔线维护规范
每次大纲修改后，检查并更新B_伏笔线.md：
- 已回收的伏笔：保留在"一"部分，加✅
- 新种伏笔：加到"二"部分，标🔨
- 章节号变化：同步更新所有相关章节号
- 语法：标准Markdown表格，单竖线分隔，|和内容之间有空格

## 关键经验：细纲已有内容时只改配套文件

**发现问题**：当细纲中某个新意点（如②③④⑤）已经写到位了，大纲配套文件（D_分卷大纲/伏笔线/人物/爽点节奏表）却还是旧套路。

**处理策略**：
- **细纲层面**：如果细纲中已有该内容但写法老套，**不一定要改细纲**——先确认用户是否要求细纲级重写
- **配套文件层面**：必须同步更新四类文件，确保全局一致：
  1. `D_分卷大纲.md` — 分卷事件描述对齐新意
  2. `B_伏笔线.md` — 伏笔回收状态更新（已种→已收✅ / 新增伏笔🔨）
  3. `A_人物.md` — 新角色卡 / 角色弧光变化
  4. `C_爽点节奏表.md` — 爽点序号重新编排（含去除重复序号）

**判断是否需要改细纲**：用户说"新意改造"但细纲里已有类似情节 → 先只改配套文件，向用户确认是否还需要细纲层面深化。

## 子智能体执行陷阱（2026-04-18 新增）

### 陷阱一：子智能体 ~400 秒超时中断

**现象**：`delegate_task` 运行时长约 400 秒后被中断，但文件可能已写入（子智能体内部先 write_file 再 report），需要验证。

**解决**：
- 单次 rewrite 任务 ≤30 章（而非 50 章），避免超时
- `delegate_task` 返回后，立即用 `terminal` + `wc -l` 验证文件行数是否变化
- 写入行数符合预期（如 50 章展开版应在 500-1500 行）才算成功

**验证命令**：
```bash
wc -l ~/小说流/她还在读大三_万亿帝国/细纲/第一卷细纲.md
# 如果行数从 ~200 变成 ~1500，说明写入成功
```

### 陷阱二：子智能体生成内容违反卷间约束

**现象**：子智能体将第四卷的"闪拍线"情节提前写入了第三卷，导致两卷内容重叠、时间线混乱。

**原因**：子智能体在生成第三卷 Ch136-Ch150 时，参考了第四卷的闪拍情节，但用户明确要求闪拍是第四卷的内容。

**解决**：
- 在提交子智能体任务前，先手工明确每卷的核心主线和边界（哪些情节必须不在某卷）
- 批量修复时用 `patch` 替换整块内容（而非逐章 patch）
- 用 `execute_code` + `re.findall` 做卷间交叉验证：
  - 第三卷不应出现"闪拍立项/闪拍上线"
  - 第二卷不应出现"闪拍"
  - 求婚时间在各卷保持一致

**验证脚本**：
```python
import re
for vol, path in [("第三卷", "第三卷路径"), ("第二卷", "第二卷路径")]:
    with open(path) as f:
        content = f.read()
    # 第三卷可以有闪拍回忆，不能有立项；闪拍主线在第四卷
    flash_立 = "闪拍立项" in content or "闪拍上线" in content
    print(f"{vol}闪拍立项/上线: {'❌ 有' if flash_立 else '✅ 无'}")
```

## QA 验证脚本模板

每次大纲修改后，用 `execute_code` 执行以下检查：

```python

files = {
    "第一卷": "~/小说流/她还在读大三_万亿帝国/细纲/第一卷细纲.md",
    "第二卷": "~/小说流/她还在读大三_万亿帝国/细纲/第二卷细纲.md",

    "第五卷": "~/小说流/她还在读大三_万亿帝国/细纲/第五卷细纲.md",
}

for vol, path in files.items():
    with open(path) as f:
        content = f.read()
    sz = os.path.getsize(path)/1024
    chapters = sorted([int(m.group(1)) for m in re.finditer(r'### Ch(\d+)', content)])
    missing = sorted(set(range(chapters[0], chapters[-1]+1)) - set(chapters))
    dup = [c for c in chapters if chapters.count(c) > 1]
    print(f"{vol}: {sz:.0f}KB | {len(chapters)}章 | {'✅' if not missing and not dup else '⚠️ 缺/重'}")

# 格式完整性：缺失钩子的章节
for vol, path in files.items():
    with open(path) as f:
        content = f.read()
    chapters = re.findall(r'### Ch(\d+)[:：](.*?)(?=\n---|\n### |\Z)', content, re.DOTALL)
    missing_hook = [(ch, len(c)) for ch, c in chapters if '钩子' not in c]
    if missing_hook:
        print(f"{vol}缺失钩子: {[(ch, wc) for ch, wc in missing_hook]}")

# 卷间一致性
for vol, path in files.items():
    with open(path) as f:
        content = f.read()
    years = re.findall(r'20\d\d年', content)
    print(f"{vol}年份: {sorted(set(years))[:8]}")

## 伏笔线维护规范

**问题现象**：在爽点节奏表等有顺序编号的表格中，多次 patch 后出现重复序号（如两个21、两个24）。

**原因**：每次 patch 替换的是 old_string 中的序号，但旧序号在 old_string 中仍存在，导致合并后重号。

**解决策略**：
- patch 后立即 read_file 确认当前状态
- 用 `execute_code` 统计所有序号：出现次数 >1 的就是重复项
- 按顺序重新编排（最终版 30 个爽点 = 序号 1-30）

**验证命令**：
```python
import re
with open(path) as f:
    nums = re.findall(r'(?:^||)\s*(\d+)\s*(?:\s|$||)', f.read(), re.MULTILINE)
from collections import Counter
dups = {n:c for n,c in Counter(nums).items() if c>1}
print(dups)  # 空字典=无重复
```

## 伏笔追踪验证流程（标准化）

执行伏笔追踪检查时使用，本流程替代手工逐条核对。

### 步骤1：提取伏笔关键词
使用正则扫描五卷细纲，提取所有含伏笔关键词的章节：
```python
import re
keywords = ["伏笔", "悬念", "第一次", "那天", "倒计时", "宿命", "前世",
            "车祸", "旧伤", "母亲", "身世", "父母", "左臂"]
for vol, fname in [("第一卷","第一卷细纲.md"), ...]:
    content = open(f"细纲/{fname}").read()
    for m in re.finditer(r'### Ch(\d+)[:：]', content):
        ch_start = m.end()
        ch_end = content.find("---", ch_start)
        ch_text = content[ch_start:ch_end]
        hits = [kw for kw in keywords if kw in ch_text]
        if hits:
            hook_m = re.search(r'\*\*章末钩子\*\*[：:](.+?)(?:\n|$)', ch_text)
            print(f"Ch{m.group(1)}: {hits} | 钩子: {hook_m.group(1)[:50] if hook_m else ''}")
```

### 步骤2：校验伏笔回收表（常见章节号错误）
逐一核对已记录伏笔的"首次出现"章节编号是否正确。常见错误：

| 华尔街反向面试 | Ch183 | Ch170 |
| 闪拍降世 | Ch151 | Ch160 |
| 张海文暗线 | Ch156 | Ch163 |

### 步骤3：检测4类常见伏笔漏洞
用关键词扫描检测以下空白：

1. **陈权身体特征伏笔**（左臂旧伤）：搜索"左臂/旧伤/伤疤"，若无→正文需补充
2. **室友在关键情节缺席**：扫描被软禁章节（Ch110-Ch120）是否有室友焦急戏
3. **母亲线断线**：搜索Ch137附近是否展开母亲完整身份
4. **主角父母线空白**：搜索"父母/回家/见家长"，确认陈权父母是否被交代

### 步骤4：输出问题清单并更新文档
- P0（必须补充）：影响核心逻辑的伏笔断线
- P1（建议补充）：影响配角弧光的伏笔空白
- P2（可选补充）：人物设定完整性漏洞
- 更新B_伏笔线.md版本号（v1→v2）、日期、错误记录表

### 伏笔追踪已知坑点
- **占位符干扰**："（约150字）"占位符导致剧情描述不完整，需扩大搜索范围
- **Ch编号歧义**：部分章节有Ch45a/Ch45b子章节，需兼容搜索
- **同义词映射**：同一伏笔在不同章节用不同词（如"车祸"和"宿命"），需建立关键词组

## 三表检测法（冲突识别标准化）

读完细纲后，用三张表系统识别问题，再动手：

### 表一：章号冲突表
扫描所有细纲文件，提取所有章节号，检查：
- 缺号（某章编号不存在）
- 重号（同一编号出现两次）
- 跨卷不一致（同一章节在不同文件编号不同）

```python
import re
files = {"第一卷": "第一卷细纲.md", ...}
for vol, path in files.items():
    with open(path) as f:
        content = f.read()
    chapters = sorted([int(m.group(1)) for m in re.finditer(r'### 第(\d+)章', content)])
    missing = sorted(set(range(chapters[0], chapters[-1]+1)) - set(chapters))
    dup = [c for c in chapters if chapters.count(c) > 1]
    print(f"{vol}: {len(chapters)}章 | 缺号:{missing} | 重号:{dup}")
```

### 表二：情节重复表
用关键词扫描检测情节重复（尤其是相邻两卷）：
- 同一诗词出现多次（如《满江红》三用）
- 同一情节类型重复（两次"进入圣院"、两次"渡劫雷劫"）
- 同一Boss情绪重复（两次"混沌神震怒"）

```python
import re
keywords = ["进入圣院", "渡劫", "雷劫", "满江红", "混沌神震怒", "订婚"]
for vol, path in files.items():
    with open(path) as f:
        content = f.read()
    for kw in keywords:
        hits = [(m.start(), content[max(0,m.start()-20):m.start()+20]) 
                for m in re.finditer(kw, content)]
        if len(hits) > 1:
            print(f"{vol}「{kw}」出现{len(hits)}次")
```

### 表三：时间线矛盾表
用关键词扫描检测人物状态矛盾：
- 身份矛盾（第一卷已是院士，第三卷又以进士身份入学）
- 人物生死矛盾（某卷说死亡/消失，下一卷又出现无解释）
- 关系矛盾（已订婚又订婚、已死又死一次）

```python
contradictions = [

    ("订婚重复", "订婚", "订婚"),

| 级别 | 定义 | 处理方式 |

| P0 | 逻辑硬伤/核心人物矛盾/同名场面重复 | 立即修复，删除或替换重复内容 |

## 章节扩容方法论（261→500章）

当用户要求扩充篇幅时，按以下优先级分配新增章节：

### 扩容优先级

| 优先级 | 内容类型 | 说明 |
|--------|---------|------|
| 1 | 修复重复+矛盾 | 删除重复内容后重新分配章号 |
| 2 | 感情线深化 | 主角与女主的更多互动、名场面 |
| 3 | 世界观扩展 | 修炼体系、势力分布、秘境设定 |
| 4 | 反派阴谋深化 | 顾长风级别反派应有完整阴谋链 |
| 5 | 支线角色弧光 | 配角（赵铁柱、林远）的成长故事 |
| 6 | 新地图/新副本 | 混沌海、多元宇宙等 |

### 扩容后的卷数重新分配

```
原始: Ch1-60 / Ch61-120 / Ch121-180 / Ch181-240 / Ch241-261 (共261章)
目标: Ch1-100 / Ch61-145 / Ch121-200 / Ch181-270 / Ch271-500 (共500章)
     +40章   +25章    +20章     +30章    +130章
```

### 删除重复内容后的替换策略

不要只是删除，要用更有价值的内容替换：
- 删除"进入圣院"重复（Ch101-120）→ 替换为"顾长风完整阴谋链"（+10章）
- 删除重复"渡劫"（第四卷Ch186-190）→ 替换为"天道真相深度展开"（+6章）
- 替换《满江红》第二次使用 → 改为《登鹳雀楼》或《青玉案》

# 大型网文多卷大纲并行创建流程

## 适用场景
万行级/多卷（5卷以上）网文大纲项目，从零开始一次性创建全部细纲。

## 核心问题
`write_file` 对大文件（>12KB）可能写入0字节，需要用execute_code或分块策略解决。

## 标准工作流

### 1. 项目架构规划（execute_code扫描+规划）
```python
# 并行扫描参考项目结构，提取各文件规模/行数
# 扫描原始大纲行数，预估各卷内容量
# 生成7要素格式模板
# 输出：每卷预估KB → 决定是否需要分块写入
```

### 2. 顺序写入（小文件 <12KB）
直接 `write_file`，一般稳定。

### 3. 大文件分块写入策略
当 `write_file` 返回0字节时，用以下任一方案：

**方案A：execute_code写文件**
```python
with open('path/to/volume.md', 'w', encoding='utf-8') as f:
    f.write(content)
```

**方案B：分批创建文件**
将内容拆分为2-3块，分别写入同一路径（覆盖模式）。

### 4. 文件结构（参考《文道至尊》《数学太凶猛》模板）
```
项目名/
├── 项目概述.md            ← 定位/架构/核心原则
├── 主大纲.md              ← 全书章节总览（700章列表）
├── A_人物设定.md          ← 主角+配角+反派+关系网
├── B_伏笔追踪表.md        ← 伏笔编号+埋入/回收章节
├── C_爽点节奏表.md        ← 爽点类型+各卷分布统计
├── D_分卷大纲.md          ← 六卷/八卷分卷总览
├── E_世界观与设定.md      ← 金手指/组织/科技树
├── F_时间线.md            ← 年表
├── 细纲/
│   ├── 第一卷_觉醒（Ch1-100）_7要素版.md
│   ├── 第二卷_锋芒（Ch101-200）_7要素版.md
│   └── ...（每卷独立文件）
└── 优化设计文档/
    └── 综合审核报告.md    ← 四维QA + P0/P1/P2待办
```

### 5. 7要素格式标准（每章/每段）
| 要素 | 内容 |
|------|------|
| 情节主线 | 5个情节点（含起承转合） |
| 金手指应用 | 本段金手指如何触发/升级 |
| 冲突对手 | 对手是谁+筹码+弱点+对策 |
| 爽点位置 | 第X段+技巧类型 |
| 情绪价值 | 主导情绪+爽感模组 |
| 伏笔埋设 | 短线/中线/长线伏笔 |
| 章末钩子 | 留下什么悬念 |

### 6. 交付检查清单
- [ ] 目录结构完整
- [ ] 每卷细纲文件 >0字节
- [ ] 主大纲章节数 = Σ各卷章节数（验证整数性）
- [ ] 综合审核报告包含：完成度/评分/各卷评估/创新点/P0-P2待办
- [ ] 名场面用【】标记（如【Ch127名场面】）

## 已知坑

| 坑 | 现象 | 解法 |
|----|------|------|
| write_file 0字节 | 大文件（>12KB）直接写失败 | 用execute_code的open()写入，或分块写 |
| 路径写错 | agent写到~/.hermes/而非项目目录 | 必须在tool call中用绝对路径 |
| 章节缺口 | 主大纲章节数≠各卷细纲章节数之和 | 写完后用wc -l验证 |
| 批处理子代理写错路径 | subagent写到~/.hermes/ | Tao明确不用subagent，直接执行 |

## 验证命令
```bash
find ~/项目目录 -type f | sort
du -sh ~/项目目录
wc -l ~/项目目录/*.md ~/项目目录/细纲/*.md | tail -1
```

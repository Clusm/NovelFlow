# 网文大纲四轮QA审查流程

## 背景

在《她还在读大三》项目中发现：监督者（事实审查）和挑剔读者（阅读体验）两轮审查都没发现**结构性错误**——第三卷被写成了比主线起点还早的闪回叙事，且第二卷的O2O校园大战内容完全缺失。这个问题直到对比原始大纲才暴露出来。

---

## 核心发现

**两轮QA都通过了，但结构仍然有问题，原因是：**
- 监督者：审"事实是否一致"，不审"情节是否符合大纲"
- 挑剔读者：审"读起来顺不顺"，不核对"大纲规定的章节内容是否写了"

**大纲对比才是结构审查的关键**——必须把正文章节内容和大纲规定的情节逐章对照，才能发现缺失或错位。

---

## 四轮QA流程

### 第一轮：监督者事实审查
- **职责**：年龄/时间戳/金额/公司名与大纲是否一致
- **输出**：事实台账（.fact_check_batchN.md），标记P0/P1矛盾
- **局限性**：不发现结构性/叙事逻辑问题

### 第二轮：挑剔读者审查
- **职责**：爽感/节奏/人设/伏笔/叙事逻辑
- **输出**：挑剔读者报告（问题清单+修改建议）
- **局限性**：不核对大纲规定情节是否写了

### 第三轮：对照大纲查结构（关键！）
- **职责**：逐章对照大纲规定的章节情节 vs 实际正文写的
- **输出**：结构问题清单
  - 某段情节是否写了？
  - 时间线是否正确？
  - 有没有章节被写到了错误的时间点？
  - 缺失的章节内容有哪些？
- **发现的问题模式**：
  - 闪回叙事（内容比主线起点还早，无标记）
  - 章节内容错配（大纲规定A情节，正文写了B情节）
  - 完全缺失的情节段

### 第四轮：主编制定重组方案
- **职责**：基于三轮QA结果，制定"保留/重写/新建/删除"决策
- **原则**：重组而非重建，保留已有内容，补写缺失内容
- **输出**：整理方案文档

---

## 网文结构问题常见模式

| 问题模式 | 表现 | 发现方法 |
|---------|------|---------|
| 闪回无标记 | 章节时间比Ch1还早，读者困惑 | 对比章节时间戳 |
| 情节错配 | 章节号对上了，内容不是大纲规定的 | 逐章对照大纲 |
| 叙事线断裂 | 某段爽文情节完全没有 | 大纲情节清单 vs 正文目录 |
| 卷结构混乱 | 一卷里混了不属于本卷的内容 | 按卷的大纲主题检查 |
| 成长弧光断崖 | 角色性格突然变化，无过渡 | 挑剔读者审查 |

---

## 去AI化润色要点

- 句式高度重复（如"眼泪掉下来""眼眶红了"出现超过10次）
- 情感描写用词贫乏
- 对话标签单一（"他说""她说"为主）
- 场景切换无过渡词
- 章节结尾无钩子或钩子生硬

---

## 重组决策树

```
这个章节内容是否符合大纲规定？
├── 是 → 保留，检查时间线
├── 否，但内容有价值 → 提取精华，重新定位章节位置
└── 否，且无价值 → 删除或备份
```

## ⚠️ 追加内容格式丢失问题（关键！）

### 发现背景
在《文道至尊》500章大纲扩充时，番外章节（V3 Ch292-300、V4 Ch371-399、V5 Ch471-490）通过`cat >>`追加到对应卷文件末尾。追加完成后发现：
1. 追加章节**缺少 `**钩子**` 字段**（只有 `**情节**` 和 `**爽点**`）
2. 部分卷终章节用 `**点题**` 而非标准的 `**爽点**` + `**钩子**`
3. V3Ch291、V4Ch370、V5Ch470 三个卷终章节内容完全相同（"永恒图书馆雕像"段落），是写作时的复制粘贴疏漏

### 根本原因
`cat >>` 追加内容时，只追加了章节正文部分（`### ChXX：标题\n**情节**：...\n**爽点**：...`），但：
- 没有追加 `**钩子**` 行
- 复制粘贴时未检查目的地卷的格式规范差异
- 不同卷的卷终标题不同，但情节内容被错误复制

### 格式规范（标准章节块）
```
### 第N章：标题
**情节**：具体情节描述
**爽点**：爽点类型
**钩子**：下一章悬念/钩子
```
注意：只有卷终/全书终章节可能用 `**点题**` 替代 `**钩子**`（仅允许1个），其余章节必须同时有 `**爽点**` 和 `**钩子**`。

### 检测脚本
```python
import re

def check_volume_format(path, expected_range):
    with open(path) as f:
        content = f.read()
    chapters = re.findall(r'第(\d+)章[：:][^\n]+', content)
    problems = []
    for ch in chapters:
        pat = rf'第{ch}章[：:][^\n]+\n'
        m = re.search(pat, content)
        if not m:
            continue
        start = m.end()
        next_m = re.search(r'第\d+章[：:]', content[start:start+2000])
        block = content[start:start+(next_m.start() if next_m else 2000)]
        if '**情节**' not in block:
            problems.append(f"Ch{ch}: 缺情节")
        if '**爽点**' not in block and '**点题**' not in block:
            problems.append(f"Ch{ch}: 缺爽点")
        if '**钩子**' not in block:
            problems.append(f"Ch{ch}: 缺钩子")
    return problems
```

### 修复策略

**缺少 `**钩子**` 的章节**：在 `**爽点**` 行后的空行处插入合适的 `**钩子**` 内容（通过脚本批量检测并 patch）

**`**点题**` 而非 `**钩子**` 的卷终**：将 `**点题**` 改为 `**爽点**`（点题金句） + 新增 `**钩子**：下一卷预告/全书完`

**重复内容块**：检查相邻卷的卷终章节（如Ch291、Ch370、Ch470）是否有相同的 `**情节**` 段落。如有，为每个卷终写独特内容，确保：
- 第三卷卷终：回顾第三卷主线（天道/道祖）
- 第四卷卷终：过渡到传承/下一代
- 第五卷卷终：全书升华/主题收束

### 验证清单（扩充大纲后必做）
1. `re.findall` 统计每卷章节数 = 100
2. 每章有 `**情节**` + `**爽点**` + `**钩子**`（卷终可允 `**点题**`）
3. 检查卷终章节是否有重复内容块
4. D_分卷大纲的篇章结构（三幕/四幕分界）是否与细纲章节标题一致

---

## 验证步骤

每轮修改后：
1. 快速扫描章节时间戳（确保顺序递增）
2. 抽查3-5章内容是否与大纲对应
3. 检查卷末钩子是否衔接下一卷开头
4. **追加内容格式检查**：新增章节（特别是 `cat >>` 追加的章节）必须逐章检查三要素齐全

## ⚠️ 批量重写正文时的文件写入陷阱</parameter>


### 发现背景
在《她还在读大三》Ch46-70（25章）批量重写时，所有新章节文件全部变成0字节空文件，导致旧内容全部丢失。

### 根本原因
使用`terminal`工具执行`cat > file.md << 'EOF'` heredoc命令批量写入多个文件时：
1. **内容截断**：单次heredoc内容超过约50KB时，后半部分被截断（文件不报错但内容不完整）
2. **mv/cp覆盖**：批量`mv`或`cp`命令在文件数量多时执行顺序混乱，后面的mv覆盖前面的mv
3. **无验证**：terminal写入不报错，但文件实际是空的

### 解决方案
1. **用`write_file`工具**替代terminal heredoc写正文内容（write_file工具是直接API调用，不会截断）
2. **分批小量写入**：每次terminal调用写5-6章，写完立即验证文件大小
3. **先备份后重写**：重组正文之前，先把旧内容完整备份到`备份_YYYYMMDD/正文_backup/`目录
4. **写完即验证**：每写完一批，用`execute_code`统计文件大小，确保没有0字节文件

### 正确工作流（大规模正文重写）
```
1. 备份旧内容到 backup/正文_backup/
2. 确认大纲和细纲完整
3. 用write_file逐章写入（小批量：5-6章/批）
4. 每批写完立即验证文件大小
5. 全部写完后execute_code统计全书字数
6. 如发现0字节文件，立即从备份恢复并重新写入
```

### 教训
> Python的`write_file`工具比terminal的heredoc可靠一万倍——大段落正文内容，永远用write_file，不要用terminal cat。

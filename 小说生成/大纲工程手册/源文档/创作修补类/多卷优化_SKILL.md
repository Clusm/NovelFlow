# 万行级网文大纲多卷优化流程

## 适用场景
对《数学太凶猛》《文道至尊》级别的多卷网文大纲（6卷+、700章+、万行级）进行系统性多轮优化。从"可读大纲"→"商业级细纲"的标准化流程。

## 触发条件
用户要求"优化大纲"、"继续昨天的流程"、或提供了《XX项目优化设计文档》类的交接文档。

## 16步标准流程

### 步骤1-7：审计阶段
1. 扫描全部文件（`find` + `wc -l` + `du -sh`）
2. 建立文件清单（含行数/大小）
3. 识别章节缺口、矛盾、时间线错误
4. 建立爽点总表（B_爽点节奏表.md）
5. 建立伏笔追踪总表（B_伏笔追踪表.md）
6. 建立7要素格式标准（情节主线/金手指/冲突/爽点/情绪/伏笔/钩子）
7. 生成综合审核报告（`优化设计文档/综合审核报告.md`）

### 步骤8：创建五卷优化设计文档
- 并行创建5个卷的优化文档（文件名：`第X卷_名称（ChXX-YY）_优化设计文档.md`）
- 每个文档包含：质量评级、P0/P1/P2问题列表、待补充名场面标记、爽点密度核查、伏笔链核查、执行计划

**⚠️ 已知坑**：write_file对中文路径文件可能静默失败（0字节）。解决方案：用terminal heredoc:
```bash
cat > '/path/to/中文文件名.md' << 'ENDOFFILE'
content here
ENDOFFILE
```

### 步骤9：P0/P1/P2补丁执行
按优先级打patch到细纲文件：

**P0（必须修复）**：
- 时间线矛盾
- 章节缺口
- 节奏断层（连续S级无缓冲）

**P1（建议修复）**：
- S级过密（两S之间加温情缓冲）
- 逝世/情感高潮前铺垫不足
- 配角背景单薄

**P2（可选）**：
- 爽点统计表错误
- 跨卷事件混淆

**温情缓冲标准写法**：
```
| Ch466.5 | 【温情缓冲】 | 殖民队成员的家人视频通话，热泪盈眶 | 情感缓冲 |
```

### 步骤10：补充7要素
确保每章都有"情节主线/金手指/冲突/爽点/情绪/伏笔/钩子"七要素。

### 步骤11：补全名场面标记
各卷≥3个【名场面】，标记格式：`【ChXX名场面】`。

### 步骤12：挑剔读者视角四维审核
- 爽感密度
- 节奏张力
- 人设弧光
- 伏笔回收

### 步骤13：编辑视角市场审核
- 番茄/起点平台适应性
- 对标竞品书

### 步骤14：正文vs细纲一致性审查

### 步骤15：伏笔链逐条验证回收

### 步骤16：最终综合审核报告 + 复审评分

## 关键经验

### 并行文件创建失败处理
- 症状：write_file返回0字节，中文路径静默失败
- 修复：改用`terminal` + heredoc
- 验证：always `wc -l` after creation

### 爽点统计表易出错
- 常见错误：把后卷的事件放在错误的卷里
- 验证方法：检查ChXXX是否在对应卷的章节范围内
- 发现即patch

### S级过密的修复模式
- 问题：连续3+个S级无缓冲 → 读者审美疲劳
- 方案：在S级之间插入"温情缓冲"或"日常场景"
- 标准：S级之间至少1个缓冲场景

### 逝世/情感高潮铺垫法
- 情感高潮前需要3-5个递进式情感缓冲
- 温情缓冲1 → 情感铺垫 → 情感前奏 → S级高潮

### 细纲文件行数参考
| 卷 | 章节数 | 建议行数 |
|----|--------|----------|
| 卷一 | 100章 | 250-320行 |
| 卷二 | 100章 | 230-280行 |
| 卷三 | 150章 | 300-360行 |
| 卷四 | 150章 | 250-320行 |
| 卷五 | 150章 | 260-320行 |
| 卷六 | 50章 | 170-220行 |

低于此区间说明内容偏薄，高于此区间可能有冗余。

## 输出标准
- 总文件数：~20个
- 总行数：3500-4500行
- 总大小：180-250KB
- 每卷都有独立的优化设计文档
- 每次P0/P1修复都有时间戳和修复状态标记

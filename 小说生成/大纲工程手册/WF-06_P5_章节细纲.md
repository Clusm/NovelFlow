**目标**：把剧情块展开为逐章细纲——每一章写完，拿给一个写手，他就能直接写正文。

**AI角色**：逐章细节展开

**耗时**：每批8-10章约3分钟 + 人审每批约5分钟

---

## Step 5.1：AI逐章生成细纲（每批8-10章）

**AI使用的wiki知识**：
- `倒果为因` — 章节级落点设计 + 递进铺垫
- `黄金开篇与节奏控制` — 开头钩子公式（冲突前置/悬念前置/人设前置）+ 章末钩子5类型
- `爽感四大基石` — 爽感段落写法要点（含旁观者反应层次设计）
- `伏笔设计体系` — 每章埋设/回收提示 + 葡萄串链检查
- `去AI化写作` — 禁区词+句式提醒预标注

**产出物**：`06_第N卷细纲_ChXX-ChYY.md`

**每章必须包含的字段**：

| 字段 | 说明 |
|------|------|
| 所属剧情块+块内位置 | 块名（ChX-ChY），第X/N章，当前处于建置期/推进期/高潮期/收束期 |
| 本章落点 | 读者看完这章最想记住的画面（倒果为因） |
| 倒推铺垫 | 本章落点需要的前置条件，标出在前面哪章已完成 |
| 爽感类型 | 公平需求/优越需求/掌控需求/归属需求 |
| **开头钩子** | 类型（冲突前置/悬念前置/人设前置）+ 内容 + 选择理由 |
| **详细情节点** | 起（约500字）+ 承（约800字）+ 转-高潮（约1000字，含爽点写法要点+旁观者反应层次）+ 合（约500字） |
| **章末钩子** | 类型（反转式/选择式/悬念式/情感式）+ 内容 + 读者感受 |
| **伏笔管理** | 埋设：[ID+内容+承诺回收章] / 回收：[ID+回收方式] |
| **状态追踪核对** | □主角状态一致 □无违反铁律 □人设无OOC |
| **去AI味预标注** | 🚫禁用词（本章场景易触发项）/ 🚫句式提醒 |

## Step 5.2：每批细纲生成后做「章节衔接检查」

| 检查项 | ChXX→ChXX+1 | 评级 |
|--------|------------|------|
| 上一章钩子 | [内容] | — |
| 下一章开头 | [内容] | — |
| 钩子承接 | [✅紧密 / ⚠️跳线 / ❌断裂] | — |
| 时间线连续 | [✅ / ⚠️跳跃 / ❌倒流] | — |
| 情绪连续 | [✅自然 / ⚠️断层] | — |

---

## 🚪 门禁5：细纲可写性评审

| # | 检查项 | 不通过的表现 |
|---|--------|------------|
| 1 | 这章给写手能直接写吗？ | 需要再问"这里具体写什么" |
| 2 | 落点有感觉吗？ | 看了无爽感/期待感 |
| 3 | 开头200字钩子抓人吗？ | 平淡 |
| 4 | 章末钩子有追更欲望吗？ | 不想看下一章 |
| 5 | 爽点写法可执行？ | "这里要爽"没写怎么爽 |
| 6 | 伏笔记录完整？ | 埋了什么、回收章写清楚了吗 |
| 7 | 衔接检查 ⚠️/❌ 全部解决？ | 有残留 |
| 8 | 去AI味预标注有用？ | 禁区词无关痛痒 |

**通过标准**：该批次全部 ✅ → 该批次细纲定稿。逐批推进，直到全卷完成。全部卷完成后进入 [Phase 6](WF-06_P6_大纲全维审核.md)。

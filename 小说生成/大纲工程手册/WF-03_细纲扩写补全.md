## 细纲扩写：7要素批量补全

### 步骤 1：扫描实际文本并统计字数

```python
import re

path = '/path/to/细纲文件.md'
with open(path, encoding='utf-8') as f:
    lines = f.read().split('\n')

# 检查每个章节的【情节主线】字数
for target in range(311, 351):
    for i, line in enumerate(lines):
        if re.match(f'^#### Ch{target}：', line.strip()):
            for j in range(i+1, min(i+50, len(lines))):
                if '**【情节主线】**' in lines[j]:
                    content = []
                    for k in range(j+1, len(lines)):
                        stripped = lines[k].strip()
                        if re.match(r'\*\*【\w+】\*\*', stripped):
                            break
                        if stripped.startswith('>'):
                            content.append(stripped[1:].strip())
                    text = ' '.join(content)
                    char_count = len(text)
                    status = 'OK' if char_count >= 150 else 'WEAK'
                    print(f"Ch{target}: {char_count} chars [{status}]")
                    break
            break
```

### 步骤 2：编写批量扩写脚本

每次处理10章，分批进行：

```python
path = '/path/to/细纲文件.md'
with open(path, encoding='utf-8') as f:
    content = f.read()

expansions = [
    # 格式：(章节号, 原文精确文本, 扩写后内容)
    ("Ch311", "情节主线原文...", "扩写后内容（情节具体化）"),
    # ... 最多10条
]

modified = content
for ch, old, new in expansions:
    if old in modified:
        modified = modified.replace(old, new, 1)
        print(f"OK {ch}")
    else:
        print(f"MISSING {ch}")

with open(path, 'w', encoding='utf-8') as f:
    f.write(modified)
```

### 步骤 3：处理MISSING章节

当脚本报告MISSING时：
1. 重新扫描该章节的实际【情节主线】文本
2. 用 `repr()` 打印精确文本
3. 更新脚本中的old字符串，重新运行

### 步骤 4：验证扩写结果

扩写后重新执行步骤1，确认所有章节字数达标。

## 正文扩写：从感官/心理/环境维度扩写

### 核心概念
- **扩写 ≠ 重写**：保留核心情节和节奏，只在关键节点补充细节
- **润色 = 去AI化 + 扩写**：在扩写基础上清理AI痕迹

### 扩写维度（按优先级）

**1. 感官细节（最优先）：**
- 嗅觉：烟味/汽油/汗水/花香/霉味/炸串香
- 触觉：铁锈/黏糊/凉意/烫/扎/硌
- 味觉：泡面调料包/廉价啤酒/烟的苦
- 听觉：喇叭声/键盘声/蝉叫/脚步

**2. 环境深度：**
- 场景中的次要人物（小贩/老人/孩子）
- 天气/光线/时间
- 物品细节（窗帘/门把手/地上的裂缝）

**3. 心理深度：**
- 主角沉默时的内心想法
- 回忆片段（前世vs今生对比）
- 计算/盘算（数字/时间线/资产）

**4. 情绪钩子强化：**
- 章节结尾留悬念或情绪高点
- 避免"他心想/他发誓/他决定"等直白表述

### AI痕迹清理（每章必做）

| 禁区词 | 处理 |
|--------|------|
| 蓦然回首 | 删除或改写 |
| 不由得/不由自主/下意识 | 删掉，保留动作本身 |
| 此刻/与此同时 | 删掉或用具体时间/空间代替 |
| 眼眸中闪过 | 用具体眼神描写替代 |
| 嘴角微微上扬 | 改为"笑了一下/笑了声/笑容很淡" |
| 心中暗暗想着 | 直接删掉，让行动说话 |
| 轻轻地/缓缓地/渐渐 | 删掉"地"，保留动作 |
| 像X一样比喻 | AI爱用比喻，检查并删除 |

### 铁律检查
- 删"像X一样"比喻
- 删解释尾巴（因为/所以/虽然/但是连词且是解释性内容）
- 删过度心理描写（超过3行连续内心独白）

## 大纲文件精准Patch流程

### 适用场景
大纲文件中需要精准替换或插入内容时。

### 步骤 1：读取目标区域

```python
with open('/path/to/大纲.md', 'r') as f:
    content = f.read()
idx = content.find('第XX章：精确标题')
print(repr(content[idx-200:idx+100]))
```

### 步骤 2：构建唯一old_string

**关键要求：**
- 包含章节的精确标题（含标点符号）
- 包含情节段落中独有的句子
- 包含章节之间的分隔符（如`---`）
- 长度至少覆盖一个完整章节块

### 步骤 3：验证marker唯一性

```python
count = content.count(marker)
print(f"Marker found {count} time(s)")
if count != 1:
    print("NOT UNIQUE - need more context")
```

### 步骤 4：执行替换

```python
new_content = content.replace(marker, new_section, 1)
with open('大纲.md', 'w') as f:
    f.write(new_content)
```

### 验证步骤

```bash
# 1. 确认只有1个实例
grep -c "ChXX" 大纲.md

# 2. 确认行数变化
wc -l 大纲.md

# 3. 读取插入点附近确认内容连续无重复
grep -n "ChXX" 大纲.md

# 4. 全面扫描小数章节
grep "Ch[0-9]*\.[0-9]" 大纲.md

# 5. 去重扫描
grep "^#### Ch" 大纲.md | awk '{print $2}' | sort | uniq -c | awk '$1>1{print}'
```

## 插入新内容的优先级规则

| 优先级 | 方式 | 适用场景 |
|--------|------|---------|
| **A（最优先）** | 节末补充 | 新内容在5-10句以内，不构成独立章节结构 |
| **B** | 合并到相邻章节 | 新内容不超过3个情节点，不需要完整7要素 |
| **C（最后手段）** | 全文末尾追加+章号重排 | 新内容需要独立完整章节结构（≥4个情节点） |

**禁止：** 创建小数章节号（如Ch305.5）

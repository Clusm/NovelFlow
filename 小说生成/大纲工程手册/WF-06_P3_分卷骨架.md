## Step 3.1：AI生成「全书结构与开篇策略」

**AI使用的wiki知识**：
- `网文架构师` — 3大故事结构（总分总莲花式/递进阶梯式/并列无限式）
- `网文架构师` — 4种情节结构模型（三幕式/五幕式/起承转合/七点式）
- `起承转合` — 最小叙事单元结构（起：建立场景→承：矛盾积累→转：爆发点→合：余韵+钩子），适用于"单章-单场景-副本"三层
- `黄金三章` / `黄金开篇与节奏控制` — 开篇策略 + 平台关键节点

**产出物**：`03a_全书结构与开篇策略.md`

**必须包含的章节**：

1. **全书整体结构选择**：选择理由 + 阶梯/莲花/并列的图示
2. **每卷内部结构模型选择**：三幕式/五幕式/起承转合/七点式。起承转合作为卷内剧情块的最小组织单元——每个剧情块必须能独立分解为起→承→转→合四阶段
3. **黄金开篇策略**：根据目标平台，标注Ch3/Ch8/Ch15等关键节点必须完成的事项

---

## Step 3.2：AI生成「分卷剧情骨架」

**AI使用的wiki知识**：
- `网文九线写法` — 卷级九线交织
- `倒果为因` — 卷级落点设计（大小高潮递进）
- `爽感四大基石` — 每卷爽感类型分配
- `伏笔设计体系` — 跨卷伏笔因果链

**产出物**：`03b_分卷剧情骨架.md`

**每卷必须包含的字段**：

| 字段 | 说明 |
|------|------|
| 本卷在全书中的位置图 | 承接上卷结局 → 本卷要做的事 → 留给下卷的钩子 |
| 本卷核心信息卡 | 章范围 + 境界变化 + 核心冲突 + 爽感主类型 + 情绪基调 |
| 本卷三级落点 | 小高潮/中高潮/大高潮的：落点画面 + 位置 + 倒推铺垫需求 + 读者感受 |
| 本卷九线状态 | 9条线的：卷初状态 + 卷中推进 + 卷末状态 + 与主线交织点 |
| 本卷新增元素+一事不烦二主 | 每个新角色/设定的首次出现章 + 后续作用 |
| 幕级剧情骨架 | 四幕/三幕结构：每幕的目标 + 落点 + 剧情块列表（块名+章节范围+核心事件） |

## Step 3.3：AI做「跨卷一致性校验」

**AI使用的wiki知识**：
- `节奏模组` — 节奏健康度量化标准（爽感频率：每3-5章小高潮/每10-15章大高潮/持续压抑不超过5章）

**检查维度**：

| 维度 | 检查内容 |
|------|---------|
| 战力膨胀 | 逐卷境界→跨卷跨度→判断后期膨胀/前期太慢/合理均衡 |
| 爽感类型分布 | 逐卷标注4种爽感类型的权重 → 判断是否缺失某种类型 |
| 节奏健康度 | 逐卷检查：爽感频率达标？（小高潮≤5章间隔/大高潮≤15章间隔/无>5章纯平淡段）；节奏曲线是否有连续低谷？；各卷节奏占比是否符合平台偏好？ |
| 伏笔因果链 | 每卷结局钩子 → 下卷是否承接 → 标注✅/⚠️/❌ |
| 九线主线 | 逐卷检查每条线是否在推进 → 标注连续掉线超过2卷的线 |

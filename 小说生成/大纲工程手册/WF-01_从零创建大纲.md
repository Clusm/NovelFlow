# WF-01：从零创建大纲

## 适用场景
- 新项目从零开始搭建
- 适用规模：万行级 / 多卷（5卷以上）/ 500章+

## 入口判断
```
项目规模 < 200章？ → 直接手动创建，无需此流程
项目规模 ≥ 500章，多卷？ → 使用本流程
```

---

## 标准工作流

### 步骤 1：项目架构规划

**执行内容：**
- 扫描参考项目结构，提取各文件规模、行数
- 扫描原始大纲行数，预估各卷内容量
- 生成7要素格式模板
- 输出：每卷预估文件大小 → 决定是否需要分块写入

---

### 步骤 2：建立标准文件夹结构

```
项目名/
├── 项目概述.md              ← 定位/架构/核心原则
├── 主大纲.md                ← 全书章节总览（如700章列表）
├── A_人物设定.md            ← 主角+配角+反派+关系网
├── B_伏笔追踪表.md          ← 伏笔编号+埋入/回收章节
├── C_爽点节奏表.md          ← 爽点类型+各卷分布统计
├── D_分卷大纲.md            ← 多卷分卷总览
├── E_世界观与设定.md         ← 金手指/组织/科技树
├── F_时间线.md              ← 年表
├── 细纲/
│   ├── 第一卷_标题（Ch1-100）_7要素版.md
│   ├── 第二卷_标题（Ch101-200）_7要素版.md
│   └── ...
└── 优化设计文档/
    └── 综合审核报告.md
```

---

### 步骤 3：写入策略

| 文件大小 | 策略 |
|---------|------|
| 小于12KB | 直接 `write_file` |
| 大于等于12KB | 用 `execute_code` 的 `open()` 写入 |

**注意：** 大文件直接用write_file可能写入0字节，用execute_code更可靠。

---

### 步骤 4：7要素格式标准

每章包含：

| 要素 | 内容 |
|------|------|
| 情节主线 | 5个情节点（含起承转合） |
| 金手指应用 | 本段金手指如何触发/升级 |
| 冲突对手 | 对手是谁+筹码+弱点+对策 |
| 爽点位置 | 第X段+技巧类型 |
| 情绪价值 | 主导情绪+爽感模组 |
| 伏笔埋设 | 短线/中线/长线伏笔 |
| 章末钩子 | 留下什么悬念 |

## 交付检查清单

- [ ] 目录结构完整
- [ ] 每卷细纲文件大于0字节
- [ ] 主大纲章节数 = 各卷章节数之和（验证整数性）
- [ ] 综合审核报告包含：完成度/评分/各卷评估/创新点/P0-P2待办
- [ ] 名场面用【】标记

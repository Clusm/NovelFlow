# WF-06-P6：大纲全维审核

> v2.0 大纲生成流水线 · Phase 6
> 上一Phase：[P5 章节细纲](WF-06_P5_章节细纲.md) | 下一Phase：[P7 审核修复闭环](WF-06_P7_审核修复闭环.md)
> 版本：v2.0 | 最后更新：2026-04-30

---

**目标**：在进入正文写作之前，从五个独立视角对完整大纲做最后一轮深度审查。

**核心原则**：不依赖一个AI一次性做所有审核。五个视角独立审查，确保没有盲区。

**参考工具文档**：[WF-04](WF-04_质量审查去重.md)（Python脚本） + [WF-02](WF-02_大纲优化迭代.md)（P0/P1/P2分级修复）

---

## 视角一：监督者事实审查

**AI使用的wiki知识**：
- `WF-04_质量审查去重·第一轮` — 事实一致性核对
- `AI大纲一致性管理` — 时间线/地理/关系一致性

**审查内容**：对全文大纲做"事实一致性"级别的逐项核对。

| 核对表 | 内容 |
|--------|------|
| 时间线事实核对 | 逐章建立"天为单位的时间轴"，检查跳跃/倒流/矛盾 |
| 角色状态链核对 | 逐章追踪主角/关键配角的身体状态、境界，检查不合理变化 |
| 地理一致性核对 | 地点之间的移动耗时是否与世界设定一致 |
| 名称/称呼核对 | 角色名/地名/技能名在全文是否统一 |
| 数量/数字核对 | 灵石/丹药/年龄等数字前后是否一致 |

**产出物**：`07a_监督者事实审查报告.md`（含P0/P1问题清单 + 事实台账）

---

## 视角二：挑剔读者审核

**AI使用的wiki知识**：
- `WF-02_大纲优化迭代·步骤12` — 挑剔读者四维审核
- `爽感四大基石` — 爽感密度评估
- `读者情绪地图` — 情绪曲线检查
- `读者心理学` — 叙事闭合渴望/虚拟危险安全边界/期待感操控
- `读者人格代入创作法` — 六步执行流程（找最吸引点→一句话卖点→开篇钩子验证→章末悬念测试→期待感布局→紧张感检查）
- `黄金开篇与节奏控制` — 钩子质量审核

**审查内容**：

| 审查项 | 方法 |
|--------|------|
| 爽感密度热力图 | 每10章一组，统计4种爽感类型的次数，标注低密度区 |
| 节奏曲线审核 | 逐段评估情绪强度(1-10)，标注连续低情绪段（弃书风险） |
| 章末钩子质量 | 逐章检查钩子类型和质量(1-5)，标注无钩子章和同类型疲劳 |
| 人设一致性 | 逐关键场景检查角色行为是否偏离性格基准线（OOC） |
| 伏笔质量初审 | 统计短线/中线/长线比例，检查悬空伏笔和回收间隔 |
| **读者人格代入测试**（⭐新增） | 切换到纯读者视角，禁止使用分析框架，只问6个问题：1)这个故事最吸引我的是什么？2)我读完能一句话说清楚这是什么故事吗？3)200字内我被拽进去了吗？4)每章结尾我想不想点下一章？5)我期待主角达成什么？6)我担心主角会遇到什么？ |

**产出物**：`07b_挑剔读者审核报告.md`（含热力图+节奏曲线+问题清单）

## 视角三：编辑视角市场适应性审核

**AI使用的wiki知识**：
- `WF-04_质量审查去重·第五轮` — 编辑视角五张清单
- `新人选平台指南` — 平台特性
- `传统升级流vs番茄风` — 风格匹配
- `2025网文十大关键词` — 趋势对齐

**审查内容（五张清单）**：

| 清单 | 检查项 |
|------|--------|
| 故事内核 | 一句话概括清晰？核心卖点在300字内体现？目标读者画像精准？ |
| 节奏与爽点（平台维度） | 前3章完读率保障？小爽点密度达标？章末钩子覆盖率？金手指可视化？大高潮间隔合理？ |
| 人设与感情线 | 主角智商在线？反派智商在线？感情线比例合理？配角有辨识度？ |
| 市场风险 | 三观正确？无毒点（圣母/绿帽/无脑虐主）？有付费点钩子？ |
| 开篇黄金三章 | 第一章500字内出现核心冲突？前3章交代世界观？前5章金手指亮相？有代入感？ |

**产出物**：`07c_编辑视角市场审核报告.md`（含五张清单的逐项评分 + 修改优先级建议）

## 视角四：伏笔全链审计

**AI使用的wiki知识**：
- `WF-05_伏笔追踪审计` — 四阶段审计 + 三重核对 + Python验证脚本
- `伏笔设计体系` — 葡萄串因果链检查 + 回收间隔标准

**审查内容**（参考 [WF-05](WF-05_伏笔追踪审计.md) 的详细脚本）：

| 步骤 | 内容 |
|------|------|
| 读取伏笔记录 | 从各Phase产出中提取所有伏笔信息 |
| 扫描所有卷关键词 | 验证每条伏笔的埋设章和回收章是否在正文中有体现 |
| 验证伏笔链 | 逐条核对：埋点章节号正确？回收点章节号正确？存在因果链？ |
| 识别问题类型 | 章节号错误/缺少因果链/缺少情感回报/连续性断裂 |
| 检查回收间隔 | 核心伏笔≤50章/单元伏笔≤本卷末/关键伏笔≤150章 |

**产出物**：`07d_伏笔全链审计报告.md`（含断裂伏笔清单 + 修复建议）

## 视角五：终审评分

**职责**：汇总前四个视角的发现，给出综合评分和最终建议。

**评分维度**（每项满分5.0）：

| 维度 | 权重 | 评分依据 |
|------|------|---------|
| 事实一致性 | 25% | 视角一的问题数量和严重程度 |
| 爽感与节奏 | 25% | 视角二的热力图和节奏曲线 |
| 市场适应性 | 20% | 视角三的五张清单评分 |
| 伏笔质量 | 15% | 视角四的断裂率和回收率 |
| 创新性/差异化 | 15% | 与同类作品的差异评估 |

**总评**：加权平均分
- ≥4.0：通过，可进入正文写作
- 3.0-3.9：需修复后复审
- <3.0：需回到对应Phase重新设计

**产出物**：`07e_终审评分报告.md`

## 🚪 门禁6：全维审核通过

| # | 检查项 | 不通过标准 |
|---|--------|----------|
| 1 | 终审总评 ≥ 4.0？ | < 4.0 |
| 2 | P0级问题（时间线矛盾/章节缺口/节奏断层）全部修复？ | 有残留P0 |
| 3 | 爽感热力图无连续10章低密度区？ | 有 |
| 4 | 伏笔断裂率 < 10%？ | ≥ 10% |

**不通过怎么办**：进入 [Phase 7](WF-06_P7_审核修复闭环.md) 修复闭环。未达标维度标注修复优先级。

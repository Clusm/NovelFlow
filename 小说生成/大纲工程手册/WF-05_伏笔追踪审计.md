### 阶段 3：验证每条伏笔链

```python
def get_ch_drama(content, ch_num):
    for m in re.finditer(r'### Ch(\d+)[:：]', content):
        if m.group(1) == ch_num:
            ch_start = m.end()
            ch_end = content.find("---", ch_start)
            ch_text = content[ch_start:ch_end]
            drama_m = re.search(r'\*\*剧情\*\*[：:]\s*(.+)', ch_text)
            return drama_m.group(1)[:70] if drama_m else ""
    return ""
```

## 伏笔链三重核对

伏笔追踪表中的章节号**必须与实际细纲章节内容交叉验证**。

### 常见错误模式

1. **埋点章节号错位**（追踪表写Ch5，实际情节在Ch8）
2. **回收点章节号断裂**（预期Ch250，实际Ch250无相关关键词）
3. **多条伏笔共享同一回收章节**

### 验证脚本

```python
for fid, (bury, expect_recover, kws) in foreshadowings.items():
    found_recover = None
    for ch in range(expect_recover-10, expect_recover+11):
        block = get_chapter_block(ch)
        if block and any(kw in block for kw in kws):
            found_recover = ch
            break
    status = "OK" if found_recover and abs(found_recover-expect_recover)<=15 else "断裂"
```

**P0级问题：** 回收断裂（found_recover=None）必须修复

## 常见伏笔漏洞检测

1. **关键身体特征伏笔：** 搜索"左臂/旧伤/伤疤"等身体特征关键词，若无→正文需补充
2. **重要角色在关键情节缺席：** 扫描关键情节章节，确认相关角色有戏份
3. **亲情/感情线断线：** 搜索重要角色关系线（如父母/导师/挚友），确认是否完整展开
4. **主角背景线空白：** 搜索"父母/回家/见家长"等，确认主角背景是否被交代

---

## 伏笔线维护规范

每次大纲修改后，检查并更新伏笔追踪文档：

- **已回收的伏笔：** 保留原位，标注回收状态
- **新种伏笔：** 添加到待回收列表
- **章节号变化：** 同步更新所有相关章节号
- **格式：** 标准Markdown表格，单竖线分隔，|和内容之间有空格

---

## 伏笔回收间隔标准

| 伏笔类型 | 回收标准 |
|---------|---------|
| 核心伏笔（影响主线） | 50章内必须回收 |
| 单元伏笔（单卷内） | 本卷结尾前回收 |
| 氛围伏笔（增加神秘感） | 全文结尾前回收 |
| 关键伏笔（身世/系统来源/核心秘密） | 150章内必须回收 |

## 已知坑点

1. **追踪表章节号常错：** 文档写Ch11，实际在Ch10——必须与正文交叉验证
2. **子章节匹配问题：** 正则模式可能同时匹配Ch45和Ch450，需精确界定
3. **空章节：** 部分章节标题后直接跟分隔符，内容为空，需跳过
4. **Patch失败：** 章节格式不同，使用前用 `repr()` 确认精确文本
5. **相邻章节合并：** 如果两章之间无分隔符，说明原内容可能为空

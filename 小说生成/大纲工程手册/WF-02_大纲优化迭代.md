# WF-02：大纲优化迭代

## 适用场景
- 已有大纲需要系统性优化
- 500章+ / 万行级 / 多卷项目
- 用户说"优化大纲"、"审校大纲"、"升级大纲"时触发

## 核心原则
1. **先审计后动手** — 步骤1-7建立全局视图，步骤8才出优化方案
2. **补丁优先于重写** — 用patch精准修改，不大面积重写
3. **每步交付物明确** — 步骤完成后确认当前状态
4. **P0问题立即修** — 高优先级问题在发现步骤立即修复，不等最后

---

## 两种执行模式

### 模式一：系统优化（全面质量提升）

适用于：输出商业级大纲

```
步骤1-7：初始审计
步骤8：创建分卷优化设计文档
步骤9：执行P0/P1/P2补丁
步骤10：7要素补全
步骤11：名场面标注补充
步骤12：挑剔读者视角四维审核
步骤13：编辑视角市场适应性审核
步骤14：伏笔链审计
步骤15：生成最终综合审核报告
步骤16：最终复审评分交付
```

### 模式二：迭代优化（轻量级）

适用于：针对性改进

```
第1轮：write_file完整优化版 → 审核 → patch修复严重问题
第2轮：复审 → patch修复次要问题
第N轮：重复直到评分达到预期
```

---

## 系统优化16步详解

### 步骤1-7：初始审计

**执行内容：**
- 扫描项目文件结构（find + wc -l + du -sh）
- 识别分卷结构、7要素覆盖、伏笔链、爽点分布
- 建立文件清单（含行数、大小）

**产出：** 综合审核报告（含P0/P1/P2问题清单）

---

### 步骤8：创建分卷优化设计文档

每个文档结构：
- 现状诊断（爽感密度、名场面数量、7要素完整度）
- 核心爽感定位（主导类型 + 情绪曲线）
- 优化设计方案（按章节段）
- 新增名场面设计
- 爽感节奏核查（过密/过疏/正常）
- 伏笔埋设优化（与邻卷联动）
- 主线影响评估（改动点 + 风险等级）

---

### 步骤9：执行P0/P1/P2补丁

**P0（必须修复，立即）：**
- 时间线矛盾
- 章节缺口
- 节奏断层（连续高潮无缓冲）

**P1（建议修复）：**
- 高潮过密（两高潮之间加温情缓冲）
- 情感高潮前铺垫不足
- 配角背景单薄

**P2（可选）：**
- 爽点统计表错误
- 跨卷事件混淆

---

### 步骤10：7要素补全

**识别方法：**
- 简表格式（章节/事件/爽点/情绪）= 缺少完整7要素
- 完整格式（含【金手指应用】【冲突对手】等标题）= 7要素齐全

**操作：** 读取原文件 → patch替换简表格式章节为完整7要素

---

### 步骤11：名场面标注补充

扫描各卷有"名场面"描述但缺少【名场面】标注符的章节，批量patch添加标注符。

---

### 步骤12：挑剔读者视角四维审核

使用 `novel-reader-reviewer` skill：

| 维度 | 检查标准 |
|------|---------|
| **爽感密度** | 每3章1小爽点 / 每10章1大冲突 / 每卷≥5名场面 |
| **节奏与钩子** | 章末钩子质量 / 冲突间隔≤10章 / 持续压抑≤5章 |
| **人设一致性** | 主角智商在线 / 感情线清晰 / 反派智商稳定 |
| **伏笔质量** | 回收间隔合理 / 有仪式感 / 一事不烦二主 |

---

### 步骤13：编辑视角市场适应性审核

主要检查项：
- 快节奏（合理爽点间隔）
- 甜宠（感情线占比适中）
- 金手指可视化（能力说明清晰）
- 名场面（经典台词/场景）

---

### 步骤14：伏笔链审计

**步骤：** 逐条核对伏笔表，检查埋设→回收章节→状态

**识别断裂伏笔：** 埋设后未回收

**产出：** 伏笔链完整性评分 + 问题修复清单

---

### 步骤15：生成最终综合审核报告

汇总前14步所有发现和修复：
- 问题修复清单（高/中/低优先级）
- 各维度质量评分

---

### 步骤16：最终复审评分交付

**最终评分卡（各维度满分5.0）：**
- 爽感密度
- 节奏把控
- 人设一致
- 伏笔质量
- 市场适应性

**交付确认：** 文件数、行数统计

---

## 问题优先级定义

| 级别 | 定义 | 处理 |
|------|------|------|
| P0 | 影响留存/完读率，立即修复 | 立即patch |
| P1 | 影响精品率，30天内修复 | 计划patch |
| P2 | 锦上添花 | 记录待办 |

---

## 常见高优先级修复

| 问题类型 | 典型表现 | 修复方式 |
|---------|---------|---------|
| 高潮过密 | 某区间高潮密集 | 插入温情缓冲章节 |
| 情感铺垫不足 | 重要情感场景缺乏铺垫 | 增加铺垫章节 |
| 失败场景缺失 | 主角只有困难→解决，无真正失败 | 增加失败场景 |
| 配角背景不完整 | 配角有故事但未揭示 | 在相关情节补充背景 |
| 伏笔断裂 | 埋设后未回收 | 在相关章节补充揭示 |
| 开篇爽点不够 | 前50章爽点稀疏 | 增加小爽场景 |
| 时间线不完整 | 年份/时间跳跃 | 补充时间线文档 |

# WF-06：大纲生成流水线 v2.0

> 从"一次生成"到"逐层收敛"的7阶段渐进式大纲生成流水线。
> 适用规模：万行级 / 500章+ / 多卷（5卷以上）网文项目。
> 版本：v2.0 | 最后更新：2026-04-30

---

## 核心原则

1. **逐层收敛** — 每一层只做一件事，人确认后解锁下一层。AI不会因为你多问一轮就"自动发挥"，而是越问越精确。
2. **AI不替人做创意决策** — AI负责结构化扩展（1→100），人负责核心创意触发（0→1）和价值判断。
3. **wiki知识按需注入** — 每个Phase只注入该Phase最需要的wiki知识卡片，不一股脑全塞。
4. **回退范围最小化** — 门禁失败只回退到对应Phase，不波及已完成部分。

---

## 流水线总览

```
Phase 1: 素材征集     Phase 2: 内核定型       Phase 3: 分卷骨架
（人机交互收集）       （AI生成+人拍板）        （AI生成+人审）
      ↓门禁1               ↓门禁2                 ↓门禁3

Phase 4: 剧情块落地    Phase 5: 章节细纲       Phase 6: 大纲全维审核
（AI填充+人审细节）     （AI逐章展开+人审）       （五视角深度审查）
      ↓门禁4               ↓门禁5                 ↓门禁6

Phase 7: 审核修复闭环
（修复→复审→归档）
      ↓门禁7
  大纲定稿，移交正文写作
```

## 附录A：各Phase回退策略

| 门禁失败位置 | 回退到 | 不回退到更早阶段的原因 |
|------------|--------|---------------------|
| 门禁1（理念不清） | Phase 1 Step 1.1 | 重新回答问卷中不清的问题 |
| 门禁2（方向不对） | Phase 2 对应Step | 只修改创作纲领/题材定位/世界观中不对的维度 |
| 门禁3（节奏不对） | Phase 3 对应卷 | 只调整那一卷的结构，不重来全部 |
| 门禁4（细节看不到） | Phase 4 对应幕/块 | 只补充看不清的那一/几块 |
| 门禁5（某章不可写） | Phase 5 对应章 | 只重写这一章，不影响同批次其他章 |
| 门禁6（审核不通过） | Phase 7 | 修复具体问题，不重来整个大纲 |
| 门禁7（修复未达标） | Phase 7 Step 7.3 | 执行未达标的修复项 |

**注入层说明**：
- **Layer 0：永久约束** — 全程生效（去AI化禁区词表、世界观铁律）
- **Layer 1：本章约束** — 生成前注入，生成后可移除（wiki方法论）
- **Layer 2：段落级约束** — 写到特定段落时注入（审核清单）
- **Layer 3：工具调用** — 需要时调用（Python脚本）

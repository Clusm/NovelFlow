# WF-04：质量审查与去重修复

## 适用场景
- 大纲/细纲完成后需要系统性诊断
- 发现章节重复、内容错位、格式不一致等问题
- 用户要求"检查大纲"、"审查大纲"时触发

---

## 四轮QA审查流程

### 第一轮：监督者事实审查

**职责：** 年龄/时间戳/金额/名称等事实与大纲是否一致

**输出：** 事实台账，标记P0/P1矛盾

**局限性：** 不发现结构性/叙事逻辑问题

---

### 第二轮：挑剔读者审查

**职责：** 爽感/节奏/人设/伏笔/叙事逻辑

**使用skill：** `novel-reader-reviewer`

**输出：** 挑剔读者报告（问题清单+修改建议）

**局限性：** 不核对大纲规定情节是否写了

---

### 第三轮：对照大纲查结构

**职责：** 逐章对照大纲规定的章节情节与实际正文

**输出：** 结构问题清单
- 某段情节是否写了？
- 时间线是否正确？
- 有没有章节被写到了错误的时间点？
- 缺失的章节内容有哪些？

**发现的问题模式：**
- 闪回叙事（内容比主线起点还早，无标记）
- 章节内容错配（大纲规定A情节，正文写了B情节）
- 完全缺失的情节段

---

### 第四轮：主编制定重组方案

**职责：** 基于三轮QA结果，制定"保留/重写/新建/删除"决策

**原则：** 重组而非重建，保留已有内容，补写缺失内容

**输出：** 整理方案文档

---

### 第五轮：编辑视角审核（平台适应性检查）

**职责：** 从商业化角度评估大纲是否符合目标平台读者偏好

**使用skill：** `挑剔读者审核`、`多视角联审`

**产出：** 市场适应性评估报告（含修改优先级建议）

---

#### 编辑审核清单（逐项检查）

**一、故事内核评估**

| 检查项 | 达标标准 | 未达标表现 |
|--------|---------|-----------|
| 一句话概括 | 能用一句话说清：谁+想做什么+遇到什么阻碍 | 需要三句以上才能说清 |
| 核心卖点 | 明确1-2个核心卖点（打脸/系统/甜宠等） | 没有明显卖点，什么都有但都不突出 |
| 目标读者 | 明确对标人群（学生/职场/小白等） | 定位模糊，男女频不明 |

**二、节奏与爽点设计**

| 检查项 | 达标标准 | 未达标表现 |
|--------|---------|-----------|
| 小爽点密度 | 每3-5章一个小爽点（打脸/收获/认可是） | 连续10章以上无爽点 |
| 大冲突间隔 | 每10-15章一个大冲突/高潮 | 冲突稀薄，流水账感 |
| 章末钩子 | 每章末尾有悬念或情绪高点 | 平坦结尾，读者随时可停止 |
| 金手指可视化 | 系统/能力有可见升级路径 | 金手指模糊，读者不知道强在哪 |

**三、人设与感情线**

| 检查项 | 达标标准 | 未达标表现 |
|--------|---------|-----------|
| 主角智商 | 主角决策合理，不愚笨不圣母 | 经常需要配角救场 |
| 反派智商 | 反派有逻辑、有威胁、有弱点 | 反派弱智，碾压无悬念 |
| 感情线比例 | 不喧宾夺主（辅助线≤30%） | 大段感情描写拖慢节奏 |
| 配角辨识度 | 主要配角有独特记忆点 | 配角面目模糊，读者记不住 |

**四、市场风险检查**

| 检查项 | 达标标准 | 未达标表现 |
|--------|---------|-----------|
| 三观正确 | 无违规内容，平台无敏感词 | 涉及政治/血腥/擦边内容 |
| 毒点排查 | 无追读率毒点（圣母/绿帽/无脑虐主） | 主角被动、憋屈、被绿 |
| 付费点 | 章节末尾有钩子，引导读者付费 | 没有情绪高点，读者无追更动力 |

**五、开篇黄金三章**

| 检查项 | 达标标准 | 未达标表现 |
|--------|---------|-----------|
| 第一章钩子 | 前500字内出现核心冲突或悬念 | 平淡开局，无亮点 |
| 世界观交代 | 前3章内交代清楚背景设定 | 10章后还在解释世界观 |
| 金手指亮相 | 主角能力/系统在前5章亮相 | 金手指藏太深，读者无期待感 |
| 代入感 | 有明确的代入感（身份/处境/目标） | 读者无法理解主角处境 |

---

#### 平台适应性速查

不同平台有不同偏好，调整大纲时请对照：

| 平台 | 节奏偏好 | 感情线 | 金手指要求 | 日更压力 |
|------|---------|--------|-----------|---------|
| **起点中文网** | 偏慢热，需要耐心铺垫 | 可以很重（都市/仙侠） | 需要有深度和特色 | 3000字/日 |
| **番茄小说** | 快节奏，飞卢风优先 | 偏轻（无女主或女主淡） | 系统要可视化、面板化 | 4000字/日 |
| **飞卢小说网** | 最快节奏，章章有爽点 | 很轻或无女主 | 系统直接、收益明确 | 6000字/日 |
| **晋江文学城** | 感情线为核心 | 感情线≥60% | 较弱，重人设 | 3000字/日 |
| **七猫免费小说** | 节奏快，新媒体风 | 甜宠为主 | 噱头明确 | 4000字/日 |

**快速判断：** 如果你的大纲在起点能过稿，不一定能在番茄/飞卢火——节奏和爽点设计差异显著。

## 章节完整性核查

### 检测脚本

```python
import re, os

files = {
    "第一卷": "细纲/第一卷细纲.md",
    "第二卷": "细纲/第二卷细纲.md",
    "第三卷": "细纲/第三卷细纲.md",
    "第四卷": "细纲/第四卷细纲.md",
    "第五卷": "细纲/第五卷细纲.md",
}

for vol, path in files.items():
    with open(path, encoding='utf-8') as f:
        content = f.read()

    # 章节覆盖检查
    chapters = [int(m.group(1)) for m in re.finditer(r'### Ch(\d+)', content)]
    expected = set(range(chapters[0], chapters[-1]+1))
    missing = sorted(expected - set(chapters))
    dup = [c for c in chapters if chapters.count(c) > 1]

    print(f"{vol}: 章节{chapters[0]}-{chapters[-1]}({len(chapters)}章) | 缺失:{missing} | 重复:{set(dup)}")
```

## 去重修复流程

### 检测重复章节

```bash
# 统计所有章节号，找出缺失和重复
grep -oP '^\*\*第\K\d+(?=章：)' 文件.md | sort -n | uniq -c | sort -k2 -n
```

```python
import re
from collections import defaultdict

with open('大纲.md', 'r') as f:
    lines = f.read().split('\n')

chapter_positions = []
for i, line in enumerate(lines):
    m = re.match(r'\*\*第(\d+)章：(.+?)\*\*', line.strip())
    if m:
        chapter_positions.append((i+1, int(m.group(1)), m.group(2)))

by_chapter = defaultdict(list)
for line_num, ch_num, title in chapter_positions:
    by_chapter[ch_num].append(line_num)

duplicates = {ch: entries for ch, entries in by_chapter.items() if len(entries) > 1}
for ch_num in sorted(duplicates.keys()):
    print(f"第{ch_num}章: {duplicates[ch_num]}")
```

### 修复策略A：保留最新版本（推荐）

**重要发现：** 行号较大的版本（后追加的）内容通常更完整。行号较小的版本往往是简略版。

**正确算法：**
1. 对每个重复章节号，按行号升序排序
2. 保留行号最大的版本（最新/最完整）
3. 删除其他所有旧副本的整段内容（从该章节标题行到下一章标题行前）

```python
import re
from collections import defaultdict

with open('大纲.md', 'r', encoding='utf-8') as f:
    lines = f.read().split('\n')

# 找所有章节位置
chapter_positions = []
for i, line in enumerate(lines):
    m = re.match(r'\*\*第(\d+)章：(.+?)\*\*', line.strip())
    if m:
        chapter_positions.append((i, int(m.group(1)), m.group(2)))

# 按章节号分组
by_chapter = defaultdict(list)
for line_num, ch_num, title in chapter_positions:
    by_chapter[ch_num].append(line_num)

# 找出需删除的行区间
lines_to_delete = set()
all_chapter_lines = sorted(set(pos for pos_list in by_chapter.values() for pos in pos_list))

for ch_num, pos_list in by_chapter.items():
    if len(pos_list) > 1:
        pos_sorted = sorted(pos_list)
        keep_pos = pos_sorted[-1]
        for del_pos in pos_sorted[:-1]:
            # 找到下一个章节的位置
            next_pos = None
            for i, pos in enumerate(all_chapter_lines):
                if pos == del_pos and i + 1 < len(all_chapter_lines):
                    next_pos = all_chapter_lines[i + 1]
                    break
            if next_pos is None:
                next_pos = len(lines)
            for i in range(del_pos, next_pos):
                lines_to_delete.add(i)

# 构建并写入新内容
new_lines = [line for i, line in enumerate(lines) if i not in lines_to_delete]
with open('大纲.md', 'w', encoding='utf-8') as f:
    f.write('\n'.join(new_lines))
```

### 修复策略B：内容行数比较

当文件结构严重损坏时，完整重建比逐个修补更快：

```python
import re
from collections import defaultdict

with open('大纲.md', 'r', encoding='utf-8') as f:
    lines = f.read().split('\n')

# 提取所有章节块
chapter_blocks = {}
for i, line in enumerate(lines):
    m = re.match(r'\*\*第(\d+)章：', line.strip())
    if m:
        ch_num = int(m.group(1))
        end_pos = None
        for j in range(i + 1, len(lines)):
            if re.match(r'\*\*第\d+章：', lines[j].strip()):
                end_pos = j
                break
        if end_pos is None:
            end_pos = len(lines)

        block_lines = lines[i:end_pos]
        valid_lines = sum(1 for l in block_lines if l.strip() and l.strip() != '---')

        # 保留内容最丰富的版本
        if ch_num not in chapter_blocks or valid_lines > chapter_blocks[ch_num]['lines']:
            chapter_blocks[ch_num] = {
                'lines': valid_lines,
                'content': '\n'.join(block_lines)
            }

# 重建文件（按章节号顺序）
final_lines = []
for ch_num in range(1, max(chapter_blocks.keys()) + 1):
    if ch_num in chapter_blocks:
        final_lines.append(chapter_blocks[ch_num]['content'].strip())
    else:
        final_lines.append(f"**第{ch_num}章：章节待补充**")

with open('大纲.md', 'w', encoding='utf-8') as f:
    f.write('\n'.join(final_lines))
```

## 正文多文件章节去重（跨目录）

### 典型症状
同一章节号出现多个不同标题的文件，如：`Ch74_身世揭露.md`、`Ch74_大学生活.md`、`Ch74_出海计划.md` 同时存在。

### 检测命令

```bash
cd "正文/第四卷/"
for ch in 71 72 73 74 75 76 77 78 79 80; do
  count=$(ls Ch${ch}*.md 2>/dev/null | wc -l)
  if [ "$count" -gt 1 ]; then echo "Ch${ch}: ${count}个文件"; fi
done
```

### Python检测脚本

```python
import os
from collections import defaultdict

vol_path = "/path/to/正文/第四卷/"
chapter_versions = defaultdict(list)

for f in os.listdir(vol_path):
    if f.startswith("Ch") and f.endswith(".md"):
        ch_num = f.split("_")[0].replace("Ch","")
        path = os.path.join(vol_path, f)
        size = os.path.getsize(path)
        with open(path, encoding='utf-8') as fh:
            lines = fh.readlines()
        title = lines[0].strip() if lines else "(空)"
        chapter_versions[ch_num].append({
            'file': f, 'size': size, 'title': title
        })

for ch, versions in sorted(chapter_versions.items(), key=lambda x: int(x[0])):
    if len(versions) > 1:
        print(f"\nCh{ch} 有 {len(versions)} 个版本:")
        for v in sorted(versions, key=lambda x: -x['size']):
            print(f"  {v['size']:>6}字 | {v['file']}")
```

## 追加内容格式检查

### 标准章节块格式

```
### 第N章：标题
**情节**：具体情节描述
**爽点**：爽点类型
**钩子**：下一章悬念/钩子
```

**注意：** 只有卷终/全书终章节可能用 `**点题**` 替代 `**钩子**`，其余章节必须同时有 `**爽点**` 和 `**钩子**`。

## 章节内容充实度判定

| 字数范围 | 判定 | 说明 |
|---------|------|------|
| 小于20字 | 几乎空白 | 细纲骨架，正常 |
| 21-50字 | 标题级 | 细纲骨架，正常 |
| 51-100字 | 简短 | 细纲骨架，正常 |
| 101-200字 | 中等 | 可接受 |
| 200字以上 | 充实 | 优秀 |

**重要：** 如果文件名含"细纲"或"7要素"，则低字数是正常现象，不代表需要重写。

## 常见逻辑矛盾修复（去重后必检）

| 问题类型 | 检测方法 | 修复思路 |
|---------|---------|---------|
| **角色时间线重复** | 同一能力/事件在不同章节多次出现 | 确认各章各有时序定位，区分"发现→成长→飞跃" |
| **名称前后不一** | 同一角色在不用章节名字不同 | 统一为第一次出现的名字，全文替换 |
| **反派打脸节奏不当** | 反派长期存在后才被打脸 | 重排节奏：登场→刁难(3章内)→当众打脸→彻底败露 |

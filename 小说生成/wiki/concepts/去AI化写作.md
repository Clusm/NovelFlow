# 去AI化写作

> **核心目标**：让AI生成的文字摆脱"AI味"，回归人类写作的真实感。
>
> **核心心法**：AI处理"怎么说"，人类决定"说什么"。当核心思想真正属于你时，AI痕迹自然降到最低。
>
> **本版更新**：新增24项量化检测体系 + 量化约束规则 + 段落呼吸节奏控制，来源为 OUBIGFA/De-AI-Writing-SKILL 的 ai-trace-detector 和 good-writing 的 style-base 框架。

---

## 一、AI味检测原理：知己知彼

在动手修改之前，先弄清AI写作为什么会被识别出来。检测系统（知网AIGC检测、Turnitin、Masterpass等）本质上是在比对以下六个维度的统计特征。

### 1.1 六大检测维度

| 维度 | AI特征 | 人类写作特征 |
|------|--------|------------|
| **词频** | 高频使用"显著""关键""然而""因此" | 用词分散，有个人偏好词 |
| **句长** | 句子长度高度一致（15-25词），均匀得像机器 | 长短交替，有明显节奏变化 |
| **句式结构** | 高度规整的并列结构，"首先/其次/最后" | 依靠内在逻辑关联，不依赖显性标记词 |
| **段落长度** | 每个段落150-200字，三段式结构高度一致 | 段落长短参差不齐 |
| **信息密度** | 过于均匀，每个句子信息量相同 | 有密有疏，有强调有留白 |
| **风格一致性** | 全篇风格恒定，像一个人写的所有段落 | 有语气/节奏的个人风格起伏 |

### 1.2 AI写作的"基因指纹"

**句式层面**：
- 大量使用"首先/其次/再次/然后/最后"机械递进
- 频繁使用"然而""因此""值得注意的是""毋庸置疑""综上所述"等过渡套话
- 句子以逗号分隔的长复合句为主，节奏单调

**用词层面**：
- 偏好"璀璨""熠熠生辉""宛如""深邃""凛冽"等书面大词
- 抽象形容词密集（"深邃的眼神""复杂的内心"）
- 情绪词直接外露而非通过动作/生理反应传达

**逻辑层面**：
- 每个段落都有标准主旨句，论点间过渡平滑到不自然
- 缺乏人类特有的"思维跳跃"——论述A时突然插入B的括号补充
- 论证结构过于"完美"，线性单向

**内容层面**：
- 擅长抽象概括，缺乏具体细节
- 缺少个人化的一手材料（经历/观察/感受）
- 引用非标准材料的能力弱

## 二、词汇层面改造：建立个人禁区词表

### 2.1 AI高频禁区词表（扩充版）

#### 形容词/副词禁区

| 禁用词 | 问题 | 改写方向 |
|--------|------|---------|
| 璀璨/熠熠生辉 | 过于书面，AI标配 | 用光线本身描述：阳光落在金属上刺眼 |
| 宛如/宛若/犹如 | 情绪类比，AI最爱 | 删除类比，直接写行为/反应 |
| 好似/仿佛/好像 | 同上 | 删除，让事实说话 |
| 深邃 | 眼睛描写标配 | 换成具体眼神或聚焦动作 |
| 凛冽/凛然 | 古风/仙侠标配 | 用物理感受替代：风吹在脸上像刀割 |
| 微微/轻轻/缓缓 | AI副词三件套 | 删"地"，保留动作本身 |
| 蓦然回首 | AI古风标配 | 要么删，要么写具体动作 |
| 心中暗暗想着 | 解释尾巴 | 直接删掉，让行动说话 |

#### 连接词/过渡词禁区

| 禁用词 | 问题 | 替代方案 |
|--------|------|---------|
| 首先/其次/再次/然后/最后 | 机械递进，AI第一特征 | 依靠句间内在逻辑关联 |
| 因此/所以/因而 | 解释强迫症 | 删掉结论，让读者自己推断 |
| 然而/但是/不过 | AI转折过于标准 | 用沉默/动作/反问替代 |
| 同时/与此同时 | 时间标记词 | 用具体时间/事件替代 |
| 值得注意的是 | 套话过渡 | 删掉，直接写内容 |
| 毋庸置疑/显而易见 | 过度确定性 | 删掉，显得更真实 |
| 综上所述/总之/可见 | 总结套话 | 删掉或用具体结果替代 |

#### 情绪表达禁区

| 禁用写法 | 问题 | 改写方案 |
|---------|------|---------|
| "他心中涌起一股……" | 情感悬浮 | 附着在具体生理反应或物体上 |
| "眼神中透露着……" | 上帝定义 | 用具体眼神动作替代 |
| "嘴角勾起一抹冷笑" | AI表情标配 | 用"他笑了一下"或干脆不笑 |
| "他感到非常愤怒" | 零冲击力 | 用生理反应：咬肌发紧/手指发抖 |
| "这表现了他的……" | 解释尾巴 | 删掉，读者自己能推断 |

### 2.2 词汇替换策略

**同义词替换**：建立个人替换库，主动替换AI高频词

**个人词汇指纹植入**：有意识地把你自己偏爱的词/表达方式植入AI文本。例如你习惯用"硌"就不用"咯"，用"撕"就不用"扯"。

**适度保留口语痕迹**：在正式书面语中保留1-2处口语化表达，立刻增加人味。

## 三、句式层面改造：打破机械感

### 3.1 长短句交替策略

AI生成的句子长度高度一致（15-25词）。人类写作天然有节奏变化——长句讲完复杂逻辑，紧跟一个短句强调核心。

| AI写法（等长） | 人类写法（长短交替） |
|--------------|-----------------|
| "他推开门走进房间，环顾四周，看到墙上挂着一幅画。" | "他推开门。墙上挂着一幅画。" |

### 3.2 打破"首先其次最后"魔咒

AI最喜欢用显性逻辑标记词，而人类更多依靠句间的内在逻辑关联。

**改造方法**：
- 用时间顺序/空间转移替代逻辑标记词
- 用动作串联替代"首先/其次/最后"
- 在关键位置用短句独立成段，打断机械节奏

### 3.3 倒装、省略、插入

适当使用非常规句式，能有效破坏AI的规整感。

**示例**：
- 倒装："真香啊，这碗面。"（AI不会这样写）
- 省略："他拿起杯子，喝了一口。放下。拿起筷子。"（动作串联）
- 插入："他——还是算了——没有说出口。"

### 3.4 段落长短参差

AI生成的段落长度高度一致（150-200字）。人类写作的段落参差不齐——有的只有一行（强调），有的长达一页（深度论证）。

**操作**：
- 每个场景单元内，故意制造段落长短变化
- 重要情绪点用短句独立成段
- 铺垫部分可以用稍长的段落

### 3.5 标点符号的节奏功能

- "咚"/"啪"/"嗒"单独成行，充当节奏鼓点
- 问号、感叹号单独成行，增强语气
- 省略号（……）替代"沉默"类描述

## 四、逻辑层面改造：增加"人"的思维痕迹

### 4.1 斩断解释欲

写完一个动作或场景后，**克制住补充说明的冲动**，立刻停笔切到下一段。

检验标准：删掉解释句后，读者还能 get 到原意吗？能 → 解释是多余的。

| AI写法（解释过剩） | 人类写法（斩断解释欲） |
|-----------------|-----------------|
| "他把门轻轻关上，仿佛怕吵醒什么人" | "他把门轻轻关上" |
| "她看着窗外的雨，内心感到孤独" | "她看着窗外的雨。手指在玻璃上画了一个圈，又抹掉" |

### 4.2 杜绝上帝表达

用叙事者的上帝视角替角色下定义/做判断，是AI的典型问题。

| 上帝表达（AI味） | 人类表达（演绎） |
|---------------|--------------|
| "这是一个残酷的世界" | "城门口的饿殍排了三丈远，野狗在啃骨头。巡卫走过，连看都没看" |
| "他是天下最狠毒的人" | "他把刀从对方喉咙里抽出来的时候，还笑了一下" |
| "她感到非常委屈" | "她的眼泪在眼眶里打转。仰起头，眨了眨眼。眼泪还是掉下来了" |

### 4.3 制造思维跳跃

人类写作不是线性的——会在论述A时突然想到B，加一个括号或破折号补充说明，然后再回到A。AI很少这样写。

**示例**："他走出门。（街上比想象中安静。路灯坏了一半。）站在台阶上点了根烟。"

### 4.4 加入适度的"逻辑瑕疵"

真正的人类论文/写作往往有跳跃式思维、旁逸斜出的例证。刻意制造1-2处不完美的"思维跳跃"，反而增加人味。

## 五、情感表达技巧：生理反应 > 心理描写

### 5.1 生理反应替代抽象情感

| 情感 | AI写法（抽象） | 人类写法（生理） |
|------|-------------|--------------|
| 愤怒 | "他感到非常愤怒" | "他的咬肌发紧。指甲抠进掌心，留下了四个月牙印" |
| 恐惧 | "他感到恐惧" | "胃突然抽搐了一下。后颈的汗毛全部竖了起来。他发现自己屏住了呼吸" |
| 悲伤 | "他非常悲伤" | "他的喉咙发紧。眼眶发酸，但没哭出来" |
| 震惊 | "他被震惊了" | "他张了张嘴，发不出声音。手里的杯子滑落，摔在地上" |
| 紧张 | "他很紧张" | "他的手指不自觉地绞在一起。茶水洒了半杯都没发现" |

### 5.2 情感物化锚点表

情感必须绑定一个具体物/气味/声音，不能悬浮。

| 情感 | 物化锚点（具体） |
|------|--------------|
| 思念 | 饺子（包了一碗，吃不完，放着） |
| 遗憾 | 旧日历（停在某一页，再翻不动了） |
| 父爱 | 鱼腥味（渔夫父亲身上的味道） |
| 初恋 | 背影（那个消失在人海中的身影） |
| 愤怒 | 砸碎的杯子（用钱能赔的东西发泄） |
| 绝望 | 熄灭的烟（灰落在鞋面上，烫了一个洞都没发现） |

### 5.3 具体化原则

抽象 → 具体的替换，是去AI味最核心的操作。

| 抽象（AI味） | 具体（人类味） |
|------------|-------------|
| "他很穷" | "他碗里的粥稀得能照出人影" |
| "天气炎热" | "石板路上的鸡蛋自己就能煎熟" |
| "战场很惨烈" | "断臂的士兵在泥水里爬，肠子拖了三尺远。没人扶他" |
| "她很美" | "她穿着一件洗得发白的红嫁衣，嘴唇涂得鲜红，脚趾甲也涂了同样的红" |
| "主角很厉害" | "他一拳下去，沙袋裂了。旁边的人下意识地往后退了半步" |

## 六、角色塑造技巧：不完美才是真人

### 6.1 给角色加缺点/异味

完美无缺的角色是AI标志之一。真人有缺点、有异味、有尴尬。

**规则**：
- 给每个重要角色至少一个缺点或异味
- 缺点不能是"太善良"这种假缺点
- 主角的缺点最好是剧情相关功能性缺点

**示例**：
- "他凑过来的时候，一股大蒜混着烟草的臭味从嘴里喷出来"
- "她每次撒谎都会不自觉地摸耳垂"
- "他打呼噜声音像拉锯，整间客栈都能听到"
- "他的腋下有一股汗味，穿了三天没换的衣服领口已经发硬"

### 6.2 道具绑定

角色出场时手里必须有东西拿着/做着——"手不空"让角色动作自然。

**规则**：
- 手里动作反映心理状态（紧张→捏紧/放松→转动）
- 道具可以成为后续伏笔

**对比**：
- AI味："他坐在椅子上看着我"
- 人类味："他坐在椅子上，用指甲刀一点点剪指甲。剪完一个，举在眼前看了看，才抬眼看我"

### 6.3 差异化反应

一场戏收尾不能所有人反应一致。简单者被说服，最聪明的一言不发。

## 七、对话技巧：潜台词 > 表面意思

### 7.1 潜台词对话

对话的每句话都有表面意思和潜藏意思两个层次。

**规则**：
- 角色说的话 ≠ 角色真正想表达的意思
- 用动作/停顿/语气变化暗示潜台词
- 好的对话让读者"听懂了角色没说的话"

**示例**：
```
"你最近很忙。"
→ 表面是观察。潜台词是"你背着我做了什么"

他不答。低头把茶碗转了半圈。
→ 潜台词是"是的，被你说中了"

"……那就好。"她站起来走了。
→ 潜台词是"你骗不了我"
```

### 7.2 对话去AI化检查

- 是否每句台词都有潜台词？
- 是否删掉了"XX说道"这种AI标准对话格式？
- 是否用动作/停顿替代了直接的情绪描述？

## 八、感官描写强化：五感写作

### 8.1 五感优先级

| 感官 | 冲击力 | 适用场景 |
|------|--------|---------|
| **触觉** | ★★★★★ | 恐怖/亲密/疼痛/温度 |
| 嗅觉 | ★★★★ | 生活感/记忆/环境 |
| 听觉 | ★★★★ | 节奏/紧张/空间感 |
| 味觉 | ★★★ | 特定场景（吃饭/喝酒/药物） |
| 视觉 | ★★★ | 锚点描写（具体物品>抽象形容） |

**核心原则**：触觉比视觉更深地触发本能反应，是去AI味最有力的武器。

### 8.2 各感官具体化

| 类型 | 示例 |
|------|------|
| 嗅觉 | "烟味/汽油/汗水/花香/霉味/炸串香/廉价洗发水" |
| 触觉 | "铁锈/黏糊/凉意/烫/扎/硌/粘腻/发麻/刺痛" |
| 味觉 | "泡面调料包/廉价啤酒/烟的苦/血腥味" |
| 听觉 | "喇叭声/键盘声/蝉叫/脚步/心跳/呼吸声" |

## 九、核心铁律（必须遵守）

### 铁律一：零"像"字

严禁使用"像/仿佛/宛如/好似/犹如"等情绪类比和比喻类比。

| 禁用 | 改用 |
|------|------|
| "他的眼神像刀一样锋利" | "他盯着对方的眼睛。对方的喉结动了一下" |
| "她笑得宛如春风" | "她笑的时候，眼角的皱纹把眼框撑成了两道弯" |

**关键**：不说"他像一头猛狮"——写他的行为（咆哮、猛扑、双眼充血）。

### 铁律二：叙事自信

人类大神相信"事情"本身就有力量，AI生怕读者看不懂必跟解释。

**检验标准**：删掉解释句后，读者还能 get 到原意吗？能 → 解释是多余的。

### 铁律三：零"地"字副词

"轻轻地""缓缓地""渐渐地"——AI最爱。删掉"地"，保留动作本身。

| 禁用 | 改用 |
|------|------|
| "他缓缓地站起身" | "他站起身" |
| "她轻轻地笑了笑" | "她笑了一下" |
| "他渐渐地平静下来" | "他平静下来" |

### 铁律四：动作独立成段

一个动作/细节独立成段，产生视觉冲击。手机屏幕不超过3行。

## 十、24项AI痕迹量化检测体系（核心升级）

> 本节为 OUBIGFA/De-AI-Writing-SKILL 的 ai-trace-detector 框架的系统整合，含硬性量化阈值。

### 10.1 检测优先级与执行原则

**优先级声明**：
- 只抓 **Top 5-10 个最致命命中**（按"读者是否立刻觉得你在跑提示词"排序）
- 发现冲突时：**保风格，删路标**。宁可保留偏见与锋利，也不要改成"干净中性文"
- 每次只做"最小改写"：删掉模板句式/空泛拔高，不把单点修补扩散成整段重写

**最小改写范式**：
- 删"值得注意的是/总体而言/换句话说"等路标词，不重写后半句论证骨架
- 把"象征着/标志着/体现了"改成一个落地判断或事实短句
- 把"专家认为/研究表明"改成可追溯来源；无来源就降级或删除

**平庸化症状（发现即回滚）**：
- 段落明显变薄，信息密度下降
- 句子整体变短且均匀，长句链断裂
- 作者视角与锋利判断被抹平成中性说明书

---

### 10.2 量化约束速查表

以下为全文级别的硬性约束，标注"默认清零"的为最严格要求：

| 编号 | 检测项 | 量化阈值 | 严重程度 |
|------|--------|---------|---------|
| 1 | `"而是"` | **= 0**（逐字引文除外） | 极高 |
| 2 | `"你会"/"你应"` | **= 0**（仅允许1次用于归谬推演） | 高 |
| 3 | 全角冒号`：` | **≤ 2**，仅直接引语提示语可用 | 高 |
| 4 | 路标词合计 | **≤ 2**（更关键/更要命/换句话说/事实上/值得注意/总之/与此同时） | 中 |
| 5 | AI高频隐喻词（比喻义） | **= 0**，原文逐字引用除外 | 高 |

### 10.3 高危检测项详解（命中即失败）

#### 项1：`"而是"` → 默认清零

**信号**：`不是A而是B`、`并非A而是B`、`不在A而在B`、`不只是A更是B`。

**豁免条件**：原文本身包含且用法自然，但全文总量≤1。

**最小改写**：
- 改为"Ａ固然……；但/而……"（不对称对照，更像原作者）
- 或删掉"不是/并非"这一半，把A改成插入语/从句限定

**正误对照**：
- ❌ "这不是口味偏好，而是系统性失败。"
- ✅ "口味偏好固然存在；更要紧的是训练、稳定性与审美体系这些硬约束。"

---

#### 项9：分析师讲解语姿（最隐蔽的AI痕迹之一）

这是逐词黑名单**无法根治**的深层模式，必须从句式模板层面拦截。

**信号**：
- "把X拆完/说完/看完/理完/弄清/厘清/讲完/聊完，再/便/就/然后Y"
- "先看A""接着看B""看完了A再说B""把A理清之后""厘清了背景"
- "上面说的是X，下面说Y""前面分析了A，现在转向B"

**根因**：LLM训练语料中教程/讲义/分析报告占比极高，段落过渡处自动生成"步骤总结→步骤预告"桥接句。

**修法**：
1. 删掉步骤承接的前半句，直接从新段落的事实或判断起笔
2. 如确有因果关系，用"于是/而/不过/可/其实"等轻量黏合词
3. 最强写法：新段落直接给事实或判断，让读者自己感受转折

**正误对照**：
- ❌ "把这些段子拆完，回头看自己这边的数字，便很难继续笑得心安。"
- ✅ "自己这边的数字其实更不好看。"
- ❌ "说完了美国的情况，再来看看中国。"
- ✅ （直接写中国的事实）

---

#### 项11：戏剧化揭露修辞（"扯遮羞布/撕下面具"）

**信号**：
- 遮羞布/面具/画皮/伪装/外衣/幌子/烟幕弹（用于揭露比喻框架）
- 扯下/撕下/揭下/掀开 + 遮蔽物；剥开/戳穿/揭穿 + 表象/真面目/本质/真相

**密度阈值**：正文**默认清零**；逐字引文除外。

**修法**：
- 删除戏剧化动作隐喻，直接给事实和判断——"扯遮羞布"能做到的，一组数据或一段引文做得更好
- 若确需表达"虚伪被揭露"语义，用落地判断替代："说到底是/不过是"、"数字摆在这里"、"事实很简单"

**正误对照**：
- ❌ "这不过是给利益输送披了一层遮羞布。"
- ✅ "利益输送的链条很清楚：A 给 B 拨款，B 给 A 的亲属发合同。"
- ❌ "撕下温情面具之后，剩下的只有冷冰冰的算计。"
- ✅ "温情是给公众看的，内部备忘录里写得很直白：削减成本，裁员 30%。"

---

#### 项12：极值判断句式（"最残酷的地方在于"）

**信号**：
- `"最……的地方在于/是"`：最残酷的地方在于/最可怕的是/最讽刺的是/最荒谬的是/最吊诡的是
- `"真正……的是"`：真正可怕的是/真正残酷的是/真正讽刺的是
- `"……之处在于"`：残酷之处在于/讽刺之处在于/荒谬之处在于
- `"更……的是"`：更残酷的是/更可怕的是/更讽刺的是

**密度阈值**：全文合计 **≤ 1**；超出即为堆叠。

**修法**：
- 删掉极值判断框架，直接给事实和判断——"最残酷的地方在于"不如一组具体数字有杀伤力
- 若只出现1次，必须紧跟可核对事实；无事实支撑的极值判断就是空泛拔高

**正误对照**：
- ❌ "最残酷的地方在于，这些人根本没有选择的余地。"
- ✅ "这些人没有选择的余地——户籍绑定社保，离开就断保，不离开就继续吃亏。"
- ❌ "真正可怕的是，没有人意识到问题的严重性。"
- ✅ "问题在文件里写了三年，每年换一个措辞，没有人动过执行条款。"

---

#### 项19：二分对照壳（"不是A而是B"）

这是项9（二分对照型）中最常见也最隐蔽的一类：读感整齐得像模板，连着出现两三次读者立刻识别。

**信号**（命中其一即可）：
- `不是……而是……` / `并非……而是……` / `不是A而B`
- `不在于……而在于……` / `不在……而在……`
- `不是因为……而是因为……`

**密度阈值**：正文**默认清零**；逐字引文可保留但必须标注，同段不叠加第二个同类壳。

**最小改写（优先级从高到低）**：
1. 不对称对照：`"不是A而是B"` → `"Ａ固然……；但/而……"`
2. 把重心塞回语流：删掉"不是/并非"，把A改成插入语/从句限定
3. 用判断替代对照：直接下判断句，再补一句条件/范围限定

**正误对照**：
- ❌ "这不是口味偏好，而是系统性失败。"
- ✅ "口味偏好固然存在；更要紧的是训练、稳定性与审美体系这些硬约束。"
- ❌ "问题不在于米其林不懂，而在于我们被偏见误解。"
- ✅ "米其林懂不懂倒在其次；我们自己的训练与标准化先站不住。"

---

#### 项13+14+15+16+17+18：伪学术腔系列（"场/叙事/摊开来/初登台/面子里子"）

**"场/场域"抽象化合物** → **= 0**：
- 舆论场/话语场/权力场/博弈场/公共场域/意义场域
- 修法：`"舆论场"` → `"舆论"`，`"话语场"` → `"讨论"`

**"叙事"做万能标签** → **= 0**（文学批评语境除外）：
- 宏大叙事/国家叙事/官方叙事/主流叙事/元叙事
- 修法：`"官方叙事"` → `"官方说法"` / `"口径"`

**"摊开来"伪随意分析姿态** → **= 0**：
- 摊开来看/摊开来说/把XXX摊开来/摊开来讲
- 修法：删引导动作，直接给事实和判断

**剧场隐喻** → **= 0**（非戏剧语境）：
- XX初登台/正式登台/粉墨登场
- 修法：`"初登台"` → `"首次出现"`

**面子/里子二元套壳** → **= 0**：
- 面子上是X，里子是Y / 给足面子、掏空里子
- 修法：`"表面看/实际上"` 或直接铺事实

**伪学术/管理咨询高频词** → **全文合计 ≤ 1**：
- 底层逻辑/顶层设计/认知升级/降维打击/赛道/红利/抓手/闭环/存量/增量/博弈（非博弈论语境）/范式/路径依赖（非经济学语境）/权柄/话语权（非学术引用）

### 10.4 段落呼吸节奏控制（量化版）

> 适用信息/科技类文章；评论/散文类另有基线规则。

**总段落数目标**：1500-3000字正文 → **14-18段**

| 段落类型 | 字数范围 | 目标段数 | 功能 |
|---------|---------|---------|------|
| 中段 | 90-180字 | 5-8段 | 事实铺陈与机制解释（主力） |
| 厚段 | 180-300字（上限350） | 3-5段 | 核心对比与深度分析 |
| 薄段 | 30-90字 | 2-4段 | 转折/过渡/强调 |
| 一句成段 | 单独一句 | 2-4次 | 隐性分节符（节铰） |

**一句成段 = 节铰（section hinge）**：
- 开篇后的论点钩子（"为什么值得细说"）
- 两组事实对比后的总结
- 进入分析/推演前的话题切换
- 进入实用建议前的话题切换
- **必须承载判断或转折功能**，不能是路标句（"接下来看看"）
- 一句成段之间至少隔 **2段** 正文

**反砖墙自检**：
- 连续 2段 >250字 → 审视能否在话题转换处插入中段或一句成段
- 厚段平均字数 >300字 → 检查最长段能否拆分
- 连续 3段 <60字 → 必修复

**禁止连续2段厚段**：厚段前后必须有中段、薄段或一句成段做缓冲。

**反模板段检测**：
- 是否有"挑战/机遇/展望未来/未来可期"这种模板段？→ 删
- 是否有"让我们/接下来我们将/我们先来看"？→ 删

### 10.5 执行工作流

```
1. 按风格底座完成主体骨架（厚段+长句链+事实锚点）
2. 按本节24项检测扫描，抓Top 5-10最致命命中
3. 执行"反模板快扫"：
   - "而是"命中 = 0
   - "你会"命中 = 0
   - 二人称总量 = 0（默认）
   - 路标词总量 ≤ 2
   - 冒号 ≤ 2（仅直接引语提示语）
4. 复核保真改写：不增删核心信息
5. 回读检查：是否变平/变薄/变说明书？若是则回滚
```

## 十二、写作阶段预防：减少AI味的根本

### 12.1 源头控制 > 末端修改

去AI味最有效的方式不是在AI写完后拼命改，而是在**写作阶段**就控制AI的使用方式：

1. **不要让AI写核心情节**：AI写的情节缺乏独特性，容易同质化
2. **不要让AI写关键对话**：对话是最体现个人风格的部分
3. **让AI只做辅助**：资料查询/场景描写补充/文字润色，不做主导
4. **强制加入个人观察**：让AI扩写后，手动加入你自己的一个真实细节

### 12.2 多模型协作降AI味

用不同模型处理不同环节，避免单一模型的风格固化：

- **DeepSeek/GPT-4**：创意发散、大纲设计
- **通义千问/DeepSeek**：中文润色（中文语义理解好）
- **人工**：核心情节、关键对话、个人化表达

---

## 十三、终极心法

> **所有技术手段都无法替代真诚的写作。**
>
> 如果完全依赖AI生成，只在表面做"化妆式修改"，面对真实读者时必然露馅。
>
> 最好的去AI味方法，是用AI处理自己不擅长的机械性工作（资料整理、初稿生成、语言润色），而将核心观点、论证逻辑、案例细节、情感表达留给自己完成。
>
> **AI处理"怎么说"，人类决定"说什么"。**
>
> 当文字的核心思想真正属于你时，AI痕迹自然会降到最低。

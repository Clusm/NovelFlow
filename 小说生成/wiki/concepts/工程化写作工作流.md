## 工程化写作进阶框架

### 七阶段全流程

| 阶段 | 产物 | 核心问题 |
|------|------|----------|
| 1. 素材设计 | 核心设定集 | 这个故事凭什么吸引人？ |
| 2. 大纲构建 | 三层大纲 | 爽点如何分布？ |
| 3. 章节约稿 | 每章细纲 | 这一章的情绪曲线是什么？ |
| 4. 审阅润色 | 修改稿 | 有没有AI味？ |
| 5. 多平台改写 | 平台适配版 | 平台读者偏好是什么？ |
| 6. 质量门禁 | 过审清单 | 数据指标达标了吗？ |
| 7. 分发落盘 | 定时发布 | 发布节奏最优吗？ |

### 分层资产结构

```
/
├── README.md          # 通用设计原则
├── 工程化写作流程.md  # 七阶段方法论
├── 质量门禁清单.md    # 自检标准
└── [题材名]/
    ├── 核心设定.md     # 世界观、力量体系
    ├── 主角档案.md     # 人设、金手指
    ├── 爽点地图.md     # 各阶段爽点分布
    ├── 章节细纲/       # 每章详细大纲
    ├── 正文/           # 各卷正文
    └── 平台改写/       # 多平台适配版本
```

### 质量门禁清单

- [ ] 黄金三章六要素齐全
- [ ] 每章有明确情绪变化
- [ ] 无AI味（去AI化检测）
- [ ] 伏笔有回收
- [ ] 主角每20章有成长
- [ ] 反派智商在线
- [ ] 无敏感词/违规内容

## 来源

> **核心理念**：把"AI写小说"从灵感型操作推进到工程化写作流程。
>
> 核心目标：让AI写作能力像工程资产一样不断复利，而非写一次爽一下。

## 七阶段工作流

```
题材设计 → 大纲搭建 → 章节创作 → 审阅润色 → 多平台改写 → 质量门禁 → 分发落盘
```

| 阶段 | 核心产出 | 关键工具 |
|------|----------|----------|
| **题材设计** | 题材定位、世界观、卖点梳理 | 题材研究文档 |
| **大纲搭建** | 章节大纲、情节点、伏笔线 | 大纲模板 |
| **章节创作** | 正文初稿、控制卡 | 章节创作闭环 |
| **审阅润色** | 修改稿、文字质量提升 | 去AI化自检 |
| **多平台改写** | 平台适配版本（起点版/番茄版等） | 母稿提纯 |
| **质量门禁** | 合规检测、相似度检查 | 脚本质检 |
| **分发落盘** | 定时发布、存档管理 | 分发方案 |

## 与普通提示词仓库的区别

| 普通仓库 | 工程化写作 |
|----------|-----------|
| 终点是"一段Prompt文本" | 更像一个**写作工作台** |
| 一次性聊天结果 | 强调可复用资产 |
| 灵感驱动 | **结构化/可迁移/可持续迭代** |
| 不关注质量把控 | 包含**质量门禁** |

## 分层资产结构

```
通用skills/              # 跨题材稳定复用的通用Skill能力本体
题材目录/.github/skills/ # 题材专属Skill
题材目录/.github/prompts/# 题材专属Prompts
scripts/                  # 门禁、质检、格式化脚本
```

### 通用能力层

可跨题材复用的核心能力：

| 能力 | 用途 |
|------|------|
| 通用-创建小说正文 | 从大纲生成完整章节 |
| 通用-章节创作闭环 | 创作→反馈→修改的工作循环 |
| 通用-生成章节控制卡 | 章节节奏/情绪标注 |
| 通用-提纯多平台母稿 | 统一内容源适配多平台 |
| 通用-多平台小说适配 | 各平台的风格/节奏差异 |
| 通用-输出编排 | 发布格式/定时/存档 |

### 题材专属层

每个题材目录包含：

- `写作研究/` — 题材专项研究
- `.github/skills/` — 题材专属技能定义
- `.github/prompts/` — 题材专属提示词
- `.github/instructions/` — 题材专属指令集

## 四种使用模式

### 1. 个人写作资产库

将写作资产按分层方式组织，让仓库越写越"会写"：

- Prompt模板 → Skill定义 → 题材研究成果
- 每写完一本书，积累的资产可以复用

### 2. 团队化小说生产

分工明确：
- 有人负责题材设计
- 有人负责大纲搭建
- 有人负责章节创作
- 有人负责质量门禁

### 3. 多平台分发

一套母稿 → 多平台适配版本：

| 平台 | 风格特点 |
|------|----------|
| 起点 | 老白向，信息量大，慢热可接受 |
| 番茄 | 小白向，节奏快，情绪到位，2000-3000字/章 |
| 飞卢 | 高强度，前3-10章定追否 |
| 晋江 | 女频，感情线重要 |

### 4. 质量门禁体系

初稿完成后过门禁：

| 检查项 | 说明 |
|--------|------|
| 文字质量 | 去AI化检测、违规词检查 |
| 相似度 | 与前文/外书的重复度 |
| 逻辑一致性 | 与大纲/设定的符合度 |
| 平台合规 | 各平台内容规范 |

## 推荐上手路径

### 从正文生成入手

优先关注：
- `通用-创建小说正文`
- `通用-章节创作闭环`
- `通用-生成章节控制卡`

### 从多平台输出入手

优先关注：
- `通用-提纯多平台母稿`
- `通用-多平台小说适配`
- `通用-多平台输出编排`

## 设计原则

| 原则 | 说明 |
|------|------|
| **结构化** | 分层明确，从通用到题材层层递进 |
| **可迁移** | 资产不绑定特定场景 |
| **可终检** | 强调"会收口"，有质量门禁 |
| **可持续迭代** | 资产像工程资产一样不断复利 |

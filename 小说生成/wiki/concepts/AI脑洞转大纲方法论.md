# AI脑洞转大纲方法论

> **核心课题**：如何让AI把人的想法和脑洞，最好地转换成一本小说大纲。
>
> **核心结论**：AI不擅长"无中生有"，但极其擅长"结构化扩展"。人类做0-1的创意触发，AI做1-100的结构填充。两者接力，才是正确姿势。

---

## 一、重新认识AI在大纲生成中的定位

### 1.1 AI最擅长什么

AI是**结构化扩展**的超级高手：

- 给定一个规则体系，能快速生成大量衍生内容
- 给定一个框架，能迅速填充细节
- 给定一段文字，能生成多个方向的变体
- 能同时考虑大量变量的相互作用
- 能保持跨章节的信息一致性

### 1.2 AI最不擅长什么

- **从零生成真正独特的核心创意**——AI只能组合已有模式，真正的"哇"点必须人来
- **判断哪个创意方向最适合本书**——需要人类主编的审美决策
- **理解"味道"和"气质"**——这些抽象的、感性的、一两句话说不清的东西
- **替作者做价值判断**——什么要保留、什么要放弃、什么值得放大

### 1.3 正确的人机分工

```
人类（主编角色）：            AI（执行角色）：
─────────────────           ─────────────────
✓ 核心创意触发（0→1）        ✓ 结构扩展（1→100）
✓ 核心理念定性                ✓ 规则体系下的填充
✓ 判断哪个方向值得做          ✓ 多方案并行生成
✓ 决定保留什么/放弃什么        ✓ 技术细节补全
✓ 做价值排序和取舍            ✓ 一致性校验
✓ 把控全书的气质和调性         ✓ 批量生成变体
```

**记住**：大纲是"编辑决策"，不是"执行动作"。AI可以做执行，但方向必须人定。

## 二、从脑洞到大纲：五阶段工作流

整体流程：

```
阶段一：核心触发   →   阶段二：理念定性   →   阶段三：结构设计
（人的创意）            （人机共创）            （AI主力）
                                              ↓
阶段五：章节展开   ←   阶段四：设定补全   ←
（AI+人）                （AI主力）
```

---

## 三、阶段详解

### 阶段一：核心触发（人类独立完成）

**目标**：用最简单的话，说清楚你的"哇点"是什么。

这不是写大纲，是问自己三个问题：

#### 3.1.1 必答三问

**问题1：这个故事最让你兴奋的一点是什么？**
（不是"讲什么"，是"什么让你觉得有意思"）

**问题2：读者看完后最可能记住哪个场景/哪个设定？**
（如果这个问题答不上来，说明核心不够突出）

**问题3：和同类作品比，你的差异点是什么？**
（没有差异点，AI做出来的大纲就是同质化产品）

#### 3.1.2 脑洞分类与转换策略

不同类型的脑洞，转换难度不同：

| 脑洞类型 | 描述 | 转换难度 | 转换策略 |
|---------|------|---------|---------|
| **金手指型** | "如果有个XXX系统会怎样" | ★★（较易） | 先设计系统规则，再推导社会影响 |
| **世界型** | "如果这个世界存在XXX" | ★★★（中等） | 先设定世界规则，再找代表性人物故事 |
| **人物型** | "如果有这样一个人" | ★★★（中等） | 先明确人物弧光，再设计世界背景 |
| **冲突型** | "如果面对XXX困境" | ★★（较易） | 先设计困境，再构建解决路径 |
| **结局型** | "我想写一个XXX结局" | ★★★★（较难） | 倒推法，从结局倒推前因 |
| **感觉型** | "我要写一个有XXX味道的故事" | ★★★★★（最难） | 先找同类参照作品，再提炼感觉元素 |

#### 3.1.3 核心触发Prompt（给自己用的，不是给AI的）

如果你连这三个问题都答不上来，用下面这个Prompt帮自己梳理：

```
我有一个脑洞：
[在这里写下你的脑洞]

请用以下框架帮我分析这个脑洞：
1. 这个脑洞的"核弹头"是什么？（最让人印象深刻的一点）
2. 这个脑洞最怕什么？（写跑偏的风险是什么）
3. 要让这个脑洞成立，最需要先确定什么？（第一个要做的决定）
```

### 阶段二：理念定性（人机共创，大量人审）

**目标**：把模糊的兴奋感，变成清晰的创作纲领。

#### 3.2.1 定什么

| 维度 | 问题 | 产出物 |
|------|------|--------|
| **题材定位** | 卖给哪个平台？小白文还是老白文？ | 起点向/番茄向/飞卢向/晋江向 |
| **核心卖点** | 读者为什么点进来？ | 一句话卖点（20字以内） |
| **基调气质** | 读起来是什么味道？ | 参考作品+形容词（"像《诡秘之主》但更黑暗"） |
| **禁区边界** | 什么绝对不写？ | 3条红线 |

#### 3.2.2 理念定性Prompt

```
## 创作纲领确认

我的核心脑洞是：[一句话描述]

我最喜欢的参照作品是：[1-2本]
我最想呈现的感觉是：[3-5个形容词]

请根据以上信息，帮我明确以下五点：

1. **题材定位**：起点中文网/番茄小说/飞卢小说/晋江文学
2. **核心卖点**：用一句话（20字内）描述本书最核心的看点
3. **目标读者**：谁会最喜欢这本书？他们的爽点是什么？
4. **基调关键词**：本书的语言风格（简洁/华丽/幽默/沉重…）
5. **绝对禁区**：我不写什么？（至少3条）

注意：这些是你的建议，最终决定权在我。我会逐条确认或修改。
```

**重要**：AI给出的这5点，**必须逐条和人确认**，不能直接接受。这是主编职责。

---

### 阶段三：结构设计（AI主力，人做关键决策）

**目标**：生成完整的大纲骨架——五卷结构、章数分配、爽点节奏。

#### 3.3.1 世界观与力量体系

**先设计规则，再推导剧情**。规则越清晰，剧情越有力。

Prompt模板：

```
## 世界观设计任务

### 基础设定
类型：[玄幻/都市/仙侠/科幻/历史/悬疑等]
核心脑洞：[一句话描述本书最核心的世界设定]

### 请帮我设计以下内容：

**1. 力量体系**
- 修炼/能力等级名称（至少6级，有层次感）
- 升级规则（怎么变强？努力/天赋/资源/机遇？）
- 升级代价（变强需要付出什么？）
- 力量上限（有没有天花板？打破天花板的路径？）

**2. 势力分布**
- 主要势力（至少3个，各自有立场和目标）
- 势力间关系（谁和谁是敌对/盟友/竞争？）
- 势力内部矛盾（每个势力内部有没有分裂？）

**3. 世界规则**
- 这个世界的核心规则是什么？（限制/法则/铁律）
- 普通人如何生存？
- 最被忽视的机会在哪里？

**4. 核心冲突**
- 世界层面的主要矛盾是什么？
- 这个矛盾为什么会在主角出现的时机爆发？

请输出结构化格式，我后续会逐条确认。
```

#### 3.3.2 金手指设计

**核心原则**：
- 金手指是工具，主角才是驱动力
- 金手指要有**限制**，没有限制的金手指等于没有
- 金手指最好有**成长性**，随主角成长而进化

Prompt模板：

```
## 金手指设计任务

### 主角基本信息
姓名：[名字]
性格标签：[3个关键词]
核心欲望：[最想要什么]
核心恐惧：[最害怕什么]

### 请设计金手指

**类型选项**（可以组合）：
A. 系统/面板类（数值化反馈）
B. 知识/情报类（信息差）
C. 物品/能力类（外物赋予）
D. 特殊体质类（天生异能）
E. 错位认知类（你和世界认知不同）

**请明确以下六点**：

1. **金手指是什么**：（一句话描述）
2. **金手指的来源**：（从哪里来？为什么是主角？）
3. **金手指的限制**：（不能用的时候是什么情况？使用代价是什么？）
4. **金手指的成长性**：（如何从弱变强？升级路径？）
5. **金手指的第一个"爽点"**：（主角第一次用它解决问题的场景）
6. **金手指的"最大风险"**：（什么情况下金手指会成为双刃剑？）

### 设计约束
- 主角必须主观能动，金手指不能替主角做决策
- 90%篇幅必须有主角的感知→判断→决策→行动闭环
- 金手指的限制必须与核心冲突挂钩
```

#### 3.3.3 人物体系设计

**核心原则**：
- 主角必须有缺陷（不是假缺陷）
- 每个重要配角必须有独特的功能（情绪对照/打脸参照/成长参照/剧情推进）
- 反派智商必须在线，反派有自己的逻辑和目标

Prompt模板：

```
## 人物体系设计

### 主角
- 姓名：
- 核心缺陷：（必须是一个真实的功能性缺陷）
- 行为标签：（3个高频动作，体现性格）
- 语言标签：（口头禅/说话方式）

### 核心配角（3-5个）
请为每个配角明确：
- 与主角的关系（对手/队友/导师/情人/路人）
- 他们的核心冲突是什么？
- 他们的存在如何推动主角成长？

### 反派体系
- 主线反派是谁？他的核心目标是什么？他为什么与主角为敌？
- 反派的智慧体现在哪里？（不是弱智反派）

请参考：[[人物塑造方法]] 中的"标签组合法"和"语言DNA"
```

#### 3.3.4 五卷结构大纲

**这是整个大纲的核心骨架**，也是AI最容易填充的部分，但人必须做关键决策。

Prompt模板：

```
## 五卷结构大纲生成

### 基础参数
- 目标总字数：100万字
- 目标总章数：300-400章
- 卷数：5卷

### 请为每一卷设计以下内容：

**卷名**：（这一卷的名字，要体现这一卷的核心主题）
**章数范围**：（例如：第1-60章）
**核心事件**：（这一卷主要讲什么故事）
**情感弧光**：（主角这一卷的情感/认知有什么变化）
**核心爽点**：（这一卷最重要的爽点是什么？什么时候释放？）
**伏笔回收**：（这一卷要回收哪些前面积累的伏笔？同时要埋下哪些新伏笔？）

### 五卷节奏设计原则

卷1（开局）：建立期待，展示金手指，建立核心关系网
卷2（发展）：扩大世界，第一次大困境，金手指初次受限
卷3（深入）：核心冲突浮现，盟友分化，第二次大困境
卷4（高潮前）：所有矛盾汇聚，主角经历最大挫折，金手指新突破
卷5（终局）：终极大对决，伏笔全回收，结局

### 输出格式
请用以下表格格式输出：

| 卷 | 卷名 | 章数 | 核心事件 | 情感弧光 | 核心爽点 | 伏笔状态 |
|---|------|------|---------|---------|---------|---------|
| 一 |  | 1-60 |  |  |  |  |
| 二 |  | 61-140 |  |  |  |  |
| 三 |  | 141-220 |  |  |  |  |
| 四 |  | 221-280 |  |  |  |  |
| 五 |  | 281-300+ |  |  |  |  |

### 重要提示
- 卷3和卷4之间必须有全书最大挫折点
- 每一卷结尾必须有一个"钩子"，让读者不得不追下一卷
- 伏笔分布：短线（3章内）、中线（卷内）、长线（跨卷）
```

#### 3.3.5 爽点地图设计

参考 [[爽感四大基石]] 和 [[读者情绪地图]]，明确每个卷的爽点类型。

Prompt模板：

```
## 爽点地图

### 每卷必问的五个问题

对每一卷，请回答以下五个问题：

1. **打脸爽点**：这一卷最重要的打脸场景是什么？打谁的脸？用什么方式？
2. **升级爽点**：主角在哪一刻变强了？升级的契机是什么？
3. **情报爽点**：主角发现了什么别人不知道的信息？
4. **反差爽点**：什么预期被打破了？（越意外越好）
5. **收获爽点**：主角得到了什么？（能力/资源/关系/地位）

### 爽点密度设计
- 第1卷：每20章至少2个大爽点
- 第2-4卷：每15章至少2个大爽点
- 第5卷：爽点连发，每10章至少1个超大爽点

### 爽点节奏
- 爽点之前必须有压抑（困境+希望）
- 连续爽点之间必须有拉扯（延迟满足）
- 参考：[[叙事技巧体系]] 中的"爽感层"技法
```

### 阶段四：设定补全（AI主力）

**目标**：把阶段三设计的骨架填充成完整的章节目录（章纲）。

#### 3.4.1 章节细纲生成策略

**核心原则**：先粗后细，迭代扩展。

**第一轮：粗纲（每章一句话）**
用Prompt批量生成300-400章的一句话简纲。

```
## 第一轮：章节目录生成

请为[具体卷名]生成章节简纲。

格式要求：
第X章：[章名]——[一句话描述本章核心事件]

规则：
- 每章必须有明确的核心事件（不是"发展"）
- 每章结尾必须有钩子（为下一章留悬念）
- 连续3章不能都是铺垫，必须有起伏
- 章数：[例如61-80章]

请按以下顺序生成：先有事件，再有章名。章名可以后补。
```

**第二轮：细纲（每章200-500字）**

从第一轮中挑选关键章节（有重要爽点/重要转折的章节），生成详细细纲。

```
## 第二轮：重点章节细纲

以下章节请生成详细细纲（每章300-500字）：

章节列表：[例如：第3章、第7章、第15章、第20章……]

每章细纲格式：
- **本章核心事件**：（一句话）
- **本章爽点/虐点**：（有没有？在哪里？）
- **人物状态变化**：（主角这章结束了有什么变化？）
- **伏笔状态**：（本章埋下了什么？回收了什么？）
- **章末钩子**：（本章结尾如何勾住读者？）
- **对话重点**：（本章最关键的一场对话，写出核心潜台词）
```

**第三轮：扩充至完整细纲**

对最关键的30-50章，生成包含具体场景描写的完整细纲。

#### 3.4.2 伏笔分布设计

参考 [[伏笔设计体系]] 的三层伏笔法（短线/中线/长线）。

Prompt模板：

```
## 伏笔设计任务

### 伏笔登记表
请为全书设计伏笔追踪表：

| 伏笔ID | 伏笔内容 | 埋设位置 | 回收时机 | 回收方式 |
|--------|---------|---------|---------|---------|
| F001 |  |  |  |  |
| F002 |  |  |  |  |

### 伏笔设计原则
- 全书至少20个伏笔（10短线+7中线+3长线）
- 短线伏笔3章内必须回收
- 中线伏笔在同一卷内回收
- 长线伏笔至少跨越3卷
- 每个伏笔必须有"触发条件"（什么情况算伏笔被想起？）

### 伏笔"葡萄串"设计
请找出3-4个可以串联的伏笔，设计成"葡萄串"结构：
引线A → 触发点B → 连锁反应C → 核心爆发D

参考：[[伏笔设计体系]] 中的"葡萄串设计"
```

**目标**：把大纲细纲变成正文初稿，或直接使用细纲指导写作。

参见：[[web-novel-chapter-expansion]]（扩写润色workflow）和 [[去AI化写作]]。

## 四、不同输入形式的转换策略

### 4.1 只有一句话脑洞

**Prompt**：

```
## 脑洞扩展任务

我只有一个想法：
"[你的脑洞一句话]"

请帮我做以下扩展（不需要全部采纳，选择最好的3-5个方向）：

1. **核心冲突扩展**：如果这个脑洞成立，最大的戏剧冲突是什么？
2. **世界规则扩展**：什么样的世界规则能让这个脑洞发挥最大效果？
3. **人物弧光扩展**：什么样的主角最能让这个脑坑的故事激动人心？
4. **爽点方向**：读者可能对哪个场景最买账？
5. **3个不同方向**：请给出3个完全不同的发展方向（我可以选）

请用选择题形式呈现，附上每个方向的优缺点分析。
```

### 4.2 有一个经典场景画面

**Prompt**：

```
## 场景逆向还原

我想写一个这样的场景：
"[描述你脑海中的画面]"

请帮我反向工作：

1. **倒推前因**：要使这个场景发生，前提是什么？主角经历了什么才能到这里？
2. **这个场景的"功能性"**：这个场景让读者感受到什么？（爽/虐/燃/甜/虐心）
3. **这个场景的"不可妥协点"**：哪个细节是这个场景的灵魂，绝不能改？
4. **扩展为系列场景**：除了这个场景，这个故事还需要哪些配套场景？（打脸/升级/关系进展）
5. **设计一个起点**：以这个场景为终点，请设计起点和中间的关键节点

注意：这个场景是"[你最喜欢的]"，请优先保护它，在扩展时不要牺牲它的核心要素。
```

### 4.3 有多个人物设定，但没有情节

**Prompt**：

```
## 人物关系驱动的情节生成

### 人物设定
[主角]
- 性格：
- 欲望：
- 恐惧：

[配角A]
- 与主角关系：
- 性格：
- 与主角的冲突点：

[配角B]
……

### 生成任务

请设计一个[数字]章的故事框架，以人物关系为驱动：

1. **关系起点**：主角和每个重要角色的初始关系是什么？
2. **第一次冲突**：主角和[某个角色]的第一个矛盾是什么？发生在什么时候？
3. **关系转折点**：哪些关系在哪个节点发生了质变？
4. **终局关系**：故事结束时，每个角色的关系变成了什么？
5. **章节情节**：请为[数字]章生成一句话简纲，确保每章有至少一对人物关系的变化

### 核心原则
- 没有冲突就没有故事——每个重要角色必须有至少一次与主角的正面冲突
- 关系变化必须有"触发的具体事件"，不能是"自然而然"
```

### 4.4 有一个想写的类型/题材，不知道具体写什么

**Prompt**：

```
## [题材]网文创作方向分析

我想写[题材]文，但还没有具体想法。

请从以下维度帮我打开思路：

1. **当前[题材]文的市场现状**：
   - 什么套路最受欢迎？
   - 什么套路已经写烂了，读者已经不买账了？
   - 最近有什么新趋势？

2. **"反套路"方向**：
   - 如果把[题材]文的某个经典设定反过来，会发生什么？
   - [题材]文里哪些元素几乎没人写过，但逻辑上完全可行？

3. **3个差异化方向**：
   - 方向A：[一个完全不同的切入角度]
   - 方向B：[一个有趣的组合]
   - 方向C：[一个被忽视的人群/视角]

4. **每个方向的爽点设计**：
   - 每个方向的读者爽点是什么？
   - 每个方向的"黄金三章"怎么写？

请选择一个最有潜力的方向，帮我生成一份详细的创作纲领。
```

## 五、大纲生成质量自检清单

当AI生成完大纲后，用以下清单逐项检查：

### 5.1 结构性检查

```
□ 五卷结构是否完整？（起-承-转-合四段式是否清晰）
□ 每卷是否有明确的情感弧光？（主角认知/情感有变化吗？）
□ 卷与卷之间是否有明确的递进关系？（矛盾是否在升级？）
□ 章节数量是否与目标字数匹配？（300-400章≈100万字）
□ 每一卷结尾是否有强钩子？
```

### 5.2 爽点检查

```
□ 第1卷最晚第10章是否出现第一个大爽点？
□ 每20章是否至少有一个"爆点"？（爽/虐/燃/甜）
□ 是否有连续三章以上没有任何情绪起伏的"水章节"？
□ 打脸场景是否具体？（有具体的人、具体的事、具体的打脸方式）
□ 爽点是否有前置铺垫和压抑？
```

### 5.3 伏笔检查

```
□ 全书是否有至少20个伏笔？（短线+中线+长线）
□ 短线伏笔是否3章内回收？
□ 每个长线伏笔是否跨越至少3卷？
□ 是否有"埋了但没有回收"的伏笔？
□ 伏笔的触发条件是否明确？
```

### 5.4 金手指检查

```
□ 金手指是否有限制？（没有限制的金手指会毁掉故事）
□ 金手指是否随主角成长而进化？
□ 主角是否主动使用金手指解决问题？（不是金手指推着主角走）
□ 金手指的"爽点"是否有变化？（不能一个模式用到死）
```

### 5.5 人物检查

```
□ 主角是否有真实的功能性缺陷？
□ 主角是否有至少3个高频行为动作标签？
□ 每个重要配角是否与主角有明确的利益冲突？
□ 反派是否有自己的逻辑和目标？（不是单纯为了"坏"而坏）
□ 是否所有人对同一事件反应都不同？（差异化反应）
```

## 六、提示词工程核心原则

### 6.1 角色设定原则

让AI扮演具体的"人"，而不是泛泛的"写作助手"：

| 低效设定 | 高效设定 |
|---------|---------|
| "你是一个AI写作助手" | "你是一位有10年网文写作经验的资深编辑，熟悉起点白金作者的风格" |
| "请帮我写一个大纲" | "请以《诡秘之主》的叙事节奏和《全职高手》的人物密度为参照" |
| "写得有趣一点" | "每章结尾必须有具体的钩子动作，不能是'欲知后事如何'" |

### 6.2 分步输出原则

**不要让AI一次性生成完整大纲**——会让它进入"泛泛而谈"模式。

正确方式：分段明确，每段只做一个任务：

```
正确：分7次调用AI
1. 生成力量体系
2. 确认力量体系（人审）
3. 生成势力分布
4. 确认势力分布（人审）
5. 生成五卷结构
6. 确认五卷结构（人审）
7. 生成章节目录

错误：一次调用"帮我生成完整大纲"
```

### 6.3 格式约束原则

给AI具体的输出格式要求，它会做得更准确：

| 低效要求 | 高效要求 |
|---------|---------|
| "写一个详细的大纲" | "请用Markdown表格输出，包含：卷名/核心事件/情感弧光/爽点/伏笔状态五列" |
| "设计一个有趣的配角" | "用以下JSON格式输出：{"姓名","性格标签×3","口头禅","核心缺陷","与主角的核心冲突"}" |
| "写得生动一点" | "每章细纲必须包含：（1）具体场景动作描写；（2）核心对话的潜台词；（3）章末钩子的具体内容" |

### 6.4 迭代优化原则

AI第一版通常不是最好的。用"审查-反馈-修改"循环：

```
第1轮：生成粗纲 → 人审 → 指出问题
第2轮：针对问题修改 → 人审 → 再指出新问题
第3轮：细纲扩充 → 人审 → 确认
第4轮：格式整理 → 最终输出
```

### 6.5 参考作品锚定原则

始终用参考作品来锚定风格和质量标准：

```
"请参考[具体作品名]的[具体特质]，
不要参考[具体作品名]的[具体特质]，
最终效果要在[A作品的深度]和[B作品的爽度]之间找到平衡。"
```

## 七、常见问题与应对

### 问题1：AI生成的大纲太同质化

**原因**：AI默认生成"最安全"的答案，而安全=套路。

**应对**：
- 必须先告诉AI"不要写什么"（禁区清单）
- 强制要求"3个差异化方向"，然后人挑选
- 用参考作品锚定具体气质，不要泛泛说"要有新意"

### 问题2：AI生成的人物没有灵魂

**原因**：AI擅长生成"正确"的人设，但不擅长生成"有缺陷"的人设。

**应对**：
- 强制要求"给每个角色至少一个真实缺陷"
- 给AI具体的"人物弧光"要求（起点状态→转变契机→终点状态）
- 让AI描述"这个角色最让人讨厌/最喜欢的一个细节"

### 问题3：AI生成的大纲没有节奏感

**原因**：AI按逻辑生成，不会主动制造"起伏"。

**应对**：
- 明确告诉AI每10章必须有一个"爆点"
- 给AI一个"情绪曲线"模板，让它按曲线设计章节
- 在章纲里明确标注：第X章（爽）、第Y章（虐）、第Z章（燃）

### 问题4：AI生成的大纲有逻辑漏洞

**原因**：AI生成时前后文一致性不足，长程记忆有限。

**应对**：
- 把前文关键设定作为"上下文"重复喂给AI
- 每次生成新内容前，先让AI"复述"已有设定，确认一致后再继续
- 建立"本书设定库"，每次对话开头强制引用

### 问题5：AI生成的大纲字数太多，写不下去

**原因**：大纲太详细反而造成心理负担，而且太详细的大纲容易在写作时发现逻辑问题要全部重写。

**应对**：
- **粗纲阶段**：每章只写一句话，不超过50字
- **细纲阶段**：只对前30%章节做详细规划，后70%保持弹性
- 遵循"写三章看一步"原则——不要一次性把所有章节细纲写完

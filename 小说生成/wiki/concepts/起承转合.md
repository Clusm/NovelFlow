# 起承转合

## 基本概念

**起承转合**是网文情节的**最小叙事单元**，任何完整的情节都可以拆解为这四个阶段。掌握起承转合，就掌握了网文情节设计的基本功。

> 无论是打脸、升级、表白、揭秘，都遵循这个结构

## 四阶段详解

### 1. 起（Setup）

**定义**：建立场景、引入矛盾、铺陈背景

**任务**：
- 让读者知道这是什么场景
- 交代当前处境
- 埋下冲突的种子

**字数占比**：约20%

**示例**：
```
主角正在宗门测试资质，周围人都在嘲笑他"废物"
```

### 2. 承（Development）

**定义**：矛盾积累、压力升级、等待爆发

**任务**：
- 让冲突逐渐升级
- 积蓄读者的情绪（期待/紧张/愤怒）
- 展示对手的强大/嚣张

**字数占比**：约30%

**示例**：
```
测试结果出来，主角果然是最低等级
曾欺负他的反派当众羞辱："废物就是废物"
围观群众的嘲笑声越来越大
```

### 3. 转（Turning Point）

**定义**：转折点、爆发点、打破平衡的关键时刻

**任务**：
- 主角采取行动打破僵局
- 情绪释放/高潮来临
- 是"爽"的核心时刻

**字数占比**：约20%

**示例**：
```
就在所有人都以为主角完了时
天降异象——主角的资质不是废，是千年难遇的特殊体质！
```

### 4. 合（Resolution）

**定义**：结果呈现、余韵回味、新的平衡建立

**任务**：
- 展示行动的结果
- 旁观者的震惊反应
- 为主角赢得的地位/资源
- 留下新的钩子

**字数占比**：约30%

**示例**：
```
全场死寂三秒后，宗主亲自走来
"从今日起，你是我宗门核心弟子"
反派脸色惨白，跪地求饶
```

## 起承转合的节奏变体

### 快速版本（番茄风/短剧风）

适用于需要快速推进的场景：

```
起（压缩）：一句话交代背景
承（省略/极短）：快速跳过，直接到转
转（爆发）：直接高潮
合（余韵）：1-2句话收尾+钩子
```

### 慢热版本（起点老白向）

适用于需要深度铺垫的场景：

```
起（详写）：世界观、势力、人物关系全面铺开
承（详写）：多个小冲突层层递进
转（高潮）：大场面、激烈战斗
合（余韵）：各方反应、资源分配、后续伏笔
```

## 起承转合在网文中的应用

### 单章层面

每章都可以看作一个小的起承转合：

```
第X章起承转合：
- 起：本章开场的新情况
- 承：主角的应对/进展
- 转：本章的核心事件/反转
- 合：本章结尾的悬念/钩子
```

### 单场景层面

一场打戏的起承转合：

```
起：双方对峙，嚣张宣言
承：战斗过程（可以详可以略）
转：胜负手，关键一招
合：战后清算/震惊反应
```

### 副本/情节线层面

一个副本的起承转合：

```
起：进入副本，介绍规则
承：遇到困难/谜题/敌人
转：最终决战/真相揭露
合：副本奖励/新剧情开启
```

## 起承转合口诀

> **无起不立**——没有铺垫，爆发就没有力度  
> **无承不稳**——没有积累，情绪就不够饱满  
> **无转不爆**——没有转折，就不是爽点  
> **无合不尽**——没有收尾，爽感就散掉  

## 常见错误

| 错误 | 表现 | 结果 |
|------|------|------|
| **起太多** | 花大量篇幅介绍设定/背景 | 读者无聊划走 |
| **承不够** | 矛盾还没积累够就爆发 | 爽感打折 |
| **转太晚** | 拖了好几章才到高潮 | 节奏拖沓 |
| **合太短** | 高潮完直接跳到下个场景 | 余韵不足 |
| **四平八稳** | 每个阶段平均用力 | 没有重点 |

## 与爽感四大基石的结合

| 爽感类型 | 起 | 承 | 转 | 合 |
|----------|----|----|----|----|
| **打脸爽** | 反派嚣张 | 主角被压制 | 绝地反击 | 全场震惊 |
| **升级爽** | 遇到瓶颈 | 努力修炼/奇遇 | 突破成功 | 展示新力量 |
| **碾压爽** | 对手出现 | 嘲讽主角 | 主角一招 | 对手崩溃 |
| **羁绊爽** | 伙伴遇险 | 主角救援 | 并肩作战 | 羁绊加深 |

- 对手/阻碍：
- 目标：

## 请设计

- 承要有层次，不能平

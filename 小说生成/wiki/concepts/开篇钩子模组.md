# 开篇钩子模组

## 基本概念

**开篇钩子**是网文留住读者的第一道关卡。读者点进一本书后，前300字、前1000字、前三章，分别有不同的"流失临界点"。

> 好的开篇钩子 = 让读者在每一个临界点都想"再往后看看"

## 300字钩子：标题之后的第一击

### 读者行为
- 手机屏幕预览只看前300字
- 决定是否"点进去"

### 300字钩子公式

| 类型 | 公式 | 效果 |
|------|------|------|
| **危机型** | 主角正面临[危险/困境]，必须[做某事] | 紧张感 |
| **悬念型** | [异常现象/规则]，主角发现后[反应] | 好奇心 |
| **金手指型** | [系统/能力]出现，主角[选择绑定] | 期待感 |
| **人设型** | 主角[展现特质]，这个特质在[场景]中体现 | 代入感 |

### 300字示例

```
✓ 危机型：
"张三发现自己被锁在电梯里，手机没信号。
电梯正在缓慢下坠。"

✓ 悬念型：
"镜子里的自己，今天没有跟着我做动作。"

✓ 金手指型：
"【叮——检测到宿主，是否绑定神级选择系统？】"

✓ 人设型：
"林医生有个规矩：从不回头看病人。但今天这个病人，
让他破例了。"
```

## 1000字钩子：前三段的核心任务

### 读者行为
- 已经点进去，还在看
- 决定是否"继续翻页"

### 1000字必须完成

```
[300字] → 建立场景 + 第一句话钩子
[600字] → 主角处境明确 + 冲突浮现
[1000字] → 第一个小悬念/危机 + 追读欲望
```

### 1000字常见问题

| 问题 | 症状 | 修复 |
|------|------|------|
| 背景介绍太多 | 第一段还在介绍世界观 | 删除/压缩到最少 |
| 主角没出现 | 旁白讲了半天故事 | 尽快让主角行动 |
| 冲突没来 | 1000字还在日常 | 把冲突提前 |
| 信息过载 | 10个地名/20个人名 | 控制在前3个 |

## 三章钩子：留住读者的关键

### 第一章结尾钩子

**类型**：

| 钩子类型 | 示例 | 效果 |
|----------|------|------|
| **选择式** | "张三看着手中的两张船票：一张是安稳的归途，一张是未知的前方。他会选哪个？" | 想知道主角的选择 |
| **危机式** | "就在他准备离开时，门被撞开了。外面站着的，是追了他十年的人。" | 想看主角怎么应对 |
| **揭露式** | "张三不知道的是，他的真实身份，今天就要被公开了。" | 想看身份揭露 |
| **升级式** | "系统提示：【恭喜宿主完成首个任务，解锁新功能】" | 想看新功能是什么 |

### 第二章结尾钩子

**类型**：

| 钩子类型 | 示例 |
|----------|------|
| **升级危机** | 危机比第一章更大 |
| **新对手出现** | 比第一章的对手更强/更神秘 |
| **秘密揭露** | 第一章的悬念有进展 |
| **羁绊建立** | 出现重要伙伴/敌人 |

### 第三章结尾钩子

**类型**：

| 钩子类型 | 示例 |
|----------|------|
| **阶段性胜利** | 完成第一个目标，准备迎接更大挑战 |
| **新篇章开启** | 主角进入新环境/新阶段 |
| **身份/秘密临近揭露** | 为后续大高潮铺垫 |
| **情感升温** | 感情线有进展 |

## 经典开篇钩子模组

### 模组1：系统降临型

```
开头公式：
[主角当前处境] → [系统绑定] → [第一个任务/选择]
→ [读者想知道：系统有什么用？]

变体：
- 重生+系统
- 穿越+系统
- 觉醒+系统
```

### 模组2：危机爆发型

```
开头公式：
[平静场景] → [危机突然降临] → [主角必须应对]
→ [读者想知道：主角怎么脱困？]

变体：
- 身陷绝境（被困/被围/濒死）
- 身份危机（突然被揭穿/被误解）
- 资源危机（家族没落/被剥夺继承权）
```

### 模组3：异常事件型

```
开头公式：
[日常场景] → [发生异常] → [主角发现异常]
→ [读者想知道：异常的原因？]

变体：
- 规则异常（世界规则改变）
- 物品异常（获得神秘物品）
- 能力异常（突然获得能力）
```

### 模组4：身份落差型

```
开头公式：
[主角当前处境] → [与"真实身份"的落差] → [第一展示机会]
→ [读者想知道：主角真实身份是什么？]

变体：
- 废柴逆袭（废物外表/天才内在）
- 马甲流（低调身份/真实身份）
- 伪装流（表面身份/真实目的）
```

### 模组5：冲突前置型

```
开头公式：
[对峙场景] → [矛盾爆发] → [主角强势回应/反转]
→ [读者想知道：后续如何？]

变体：
- 退婚流（未婚妻羞辱）
- 打脸流（反派嘲讽）
- 废柴逆袭流（被欺压后反击）
```

## 各平台开篇偏好

### 番茄小说

- **偏好**：快节奏、强情绪、第一句话就要抓人
- **禁忌**：慢热、铺垫太长
- **建议**：前三句话要有冲突/金手指/悬念之一

### 起点中文网

- **偏好**：有深度、信息量足、世界观有吸引力
- **禁忌**：太小白、没有任何新意
- **建议**：开篇要有"信息差"让老白读者感到新鲜

### 飞卢小说网

- **偏好**：脑洞大、开局即高潮、同人衍生
- **禁忌**：平淡、无脑洞、普通
- **建议**：第一句话就要有"爆点"

### 晋江文学城

- **偏好**：感情线吸引人、人设出彩、文笔好
- **禁忌**：无脑甜/无脑虐
- **建议**：开篇感情线要有张力

## 开篇钩子自检清单

```
□ 300字内是否有钩子（危机/悬念/金手指/人设）
□ 主角是否在前300字内出现
□ 读者是否能说出"这是什么类型的书"
□ 读者是否能说出"主角是谁"
□ 前三章是否有至少3个钩子
□ 每个钩子是否都有"想知道"的吸引力
□ 没有大段背景介绍
□ 没有超过3个以上的配角在前三章出现
□ 第一章是否有小高潮或转折
□ 第三章结尾是否有强钩子
```

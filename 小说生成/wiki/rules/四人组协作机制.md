# 四人组协作机制·能力调用决策树

> **版本**：v1.0 | **定位**：skill灵活协作的核心机制
> **作用**：明确什么情况调用哪个skill，实现灵活协作

---

## 一、核心原则

```
上游输入 → 判断场景类型 → 匹配最适skill → 执行 → 触发下游
```

| 原则 | 说明 |
|------|------|
| **最适调用** | 不一定按顺序，哪个最合适调哪个 |
| **主动判断** | 根据场景特征自主判断，而非等待指令 |
| **闭环反馈** | 输出后自动触发下游，不中断流程 |

---

## 二、能力调用决策树

### 2.1 创作项目启动决策

```
【收到项目任务】
    ↓
用户是否有设定？
    ↓ 有设定
架构师 → 遵循用户设定 → 优化大纲 → 交付
    ↓ 无设定
架构师 → 联网学习爆款 → 生成架构 → 交付
```

### 2.2 架构阶段协作决策

```
【架构师执行中】
    ↓
判断当前任务类型
    ↓
├─ 需要爆款参考 → 触发创意总监
├─ 需要专业设定 → 触发架构师内部L2/L3
├─ 需要去AI化 → 触发原创作家基础功
└─ 需要逻辑审核 → 触发审核官
```

### 2.3 创作阶段协作决策

```
【原创作家执行中】
    ↓
判断当前场景类型
    ↓
├─ 卡文/需要创意 → 触发创意总监
├─ 需要人设指导 → 触发人设功
├─ 需要场景描写 → 触发场景功
├─ 需要节奏把控 → 触发节奏功
└─ 创作完成 → 触发审核官
```

### 2.4 审核阶段协作决策

```
【审核官执行中】
    ↓
判断问题类型
    ↓
├─ AI痕迹 → 返回原创作家去AI化
├─ 逻辑漏洞 → 返回架构师调整大纲
├─ 人设OOC → 返回原创作家人设功
└─ 审核通过 → 触发上下文管理器 → 进入下一章
```

## 六、灵活运用检查清单

每个skill执行时必须检查：

```
□ 当前任务是否需要调用其他skill？
□ 是否处于协作触发条件列表中？
□ 调用后是否会自动触发下游？
□ 协作结果是否反馈给调用方？
□ 是否形成闭环而非中断？
```

## 六、OOC判定标准（来源：llm-wiki调用规则）

### 6.1 什么是OOC？

**OOC = Out Of Character（角色脱离）**

指角色做出了违背既定人设、性格、能力、立场的行为或对话。

### 6.2 OOC判定清单

```
□ 角色说话方式是否与既定性格一致？
□ 角色决策是否符合其能力范围？
□ 角色情绪反应是否符合情境？
□ 角色立场是否前后矛盾？
□ 角色行为是否符合社会身份？
```

### 6.3 常见OOC示例

| 场景 | OOC行为 | 正确行为 |
|------|---------|---------|
| 高冷角色 | 突然说大量话痨台词 | 保持惜字如金 |
| 懦弱角色 | 突然正面硬刚 | 保持退缩但有底线 |
| 穷苦角色 | 用富人思维行事 | 保持穷人思维逻辑 |
| 聪明角色 | 犯低级错误 | 保持聪明人设 |

## 七、禁止简化设定判定（来源：llm-wiki调用规则）

### 7.1 什么是"简化设定"？

在创作过程中省略、简化、或遗忘既定的世界观/人设/规则设定。

### 7.2 简化设定判定清单

```
□ 世界观设定是否完整呈现？
□ 等级体系是否严格遵循？
□ 势力关系是否按设定执行？
□ 角色能力是否在范围内？
□ 时间线是否连贯一致？
□ 伏笔是否按计划埋设/回收？
```

### 7.3 常见简化设定示例

| 类型 | 简化行为 | 正确行为 |
|------|---------|---------|
| 等级简化 | 越级打败比自己高3级的对手，无任何解释 | 必须有特殊原因（秘术/机缘/策略） |
| 势力简化 | 忽视势力关系，直接和敌对势力合作 | 必须有铺垫和转折 |
| 能力简化 | 无条件使用金手指 | 必须有使用条件和代价 |

## 八、调用示例（来源：llm-wiki调用规则）

### 示例1：架构师设计配角
```
1. 调用 L2_characters（角色架构）
2. 调用 L3_supporting（配角设计）
3. 调用 L3_memory_point（记忆点设计）
4. 严格遵循 rules：
   - 每个配角必须有1-2个标志性特征
   - 一句话能让人记住
   - 不能是工具人
```

### 示例2：原创作家写对话
```
1. 调用 L2_dialogue（对话写作）
2. 调用 L3_subtext（潜台词设计）
3. 严格遵循 rules：
   - 角色嘴上说的不等于心里想的
   - 表面话必须有潜台词
   - 让读者有发现感
```

### 示例3：审核官检测AI味
```
1. 调用 L2_ai_detect（AI检测）
2. 调用 L3_word_freq（词频异常检测）
3. 严格遵循 rules：
   - 检测'非常''极其''不可思议'等词
   - 超过4次必须修改
   - 用具体描写替代
```

## 九、L2/L3子skill实际调用方法（关键补充）

### ⚠️ 核心澄清：
> **不是用Skill工具调用！**
> 是**读取L2/L3子skill文件内容，运用其中的方法论！**

---

### 9.1 完整执行流程：

```
收到任务
    ↓
Step 1: 读取主skill文件
        → 路径：`.trae/skills/[主skill名]/SKILL.md`
        → 作用：了解主skill职责和L2子skill索引
    ↓
Step 2: 根据主skill索引，读取对应的L2子skill
        → 路径：`.trae/skills/[主skill名]/[L2子skill名]/SKILL.md`
        → 作用：获取具体方法论
    ↓
Step 3: 如果L2子skill有L3索引，继续读取L3子skill
        → 路径：`.trae/skills/[主skill名]/[L2子skill名]/[L3子skill名]/SKILL.md`
        → 作用：获取更细化的方法论
    ↓
Step 4: 运用这些方法论，完成当前任务
    ↓
Step 5: 触发下游环节
```

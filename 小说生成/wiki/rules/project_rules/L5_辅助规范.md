## 二、技能调用关系矩阵

| 调用方向 | 调用者 | 被调用者 | 触发条件 |
|---------|--------|---------|---------|
| 启动 | 系统 | 上下文管理器 | 新项目启动 |
| 横向 | 架构师 | 拆书分析师 | 大纲完成 |
| 横向 | 架构师 | 创意总监 | 架构完成 |
| 向下 | 创意总监 | 原创作家 | 方案完成 |
| 向上 | 原创作家 | 上下文管理器 | 开始章节 |
| 向下 | 原创作家 | 审核官 | 章节完成 |
| 向上 | 审核官 | 原创作家 | 需修改 |

## 三、项目状态追踪

### 状态定义
| 状态值 | 含义 |
|--------|------|
| planning | 规划中 |
| active | 进行中 |
| paused | 暂停中 |
| completed | 已完结 |
| abandoned | 已放弃 |

### 完结标准
- ✅ 用户明确宣布完结
- ✅ 所有故事线已完成
- ✅ 所有伏笔已回收
- ✅ 有明确结局章节

---

## 四、全局检查清单

### 创作前检查
```
□ 上章是否已完成审核？
□ 本章创作是否调用了审核官？
□ 审核结果是否为"通过"？
□ 全部通过后才能保存
```

### 审核检查
```
□ 调用上下文管理器？
□ 读取大纲/outline.md？
□ 读取伏笔/foreshadow.md？
□ 读取角色/characters.md？
□ 执行六维度质检？
□ P0-P3问题定级？
□ 输出审核报告？
```

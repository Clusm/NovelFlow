## 三、违规代码速查表

| 违规代码 | 类型 | 处理方式 |
|---------|------|---------|
| VIOLATION_STEP_JUMP | 跳步执行 | 立即停止+回退+重新执行 |
| VIOLATION_SKIP_SKILL | 跳过技能调用 | 强制补调用 |
| VIOLATION_SKIP_AUDIT | 跳过审核 | 补审后才能继续 |
| VIOLATION_SKIP_SYNC | 跳过INDEX同步 | 先同步才能继续 |
| VIOLATION_SKIP_CONTEXT | 跳过上下文恢复 | 强制恢复才能继续 |
| VIOLATION_REUSE_FAILED | 复用失败项目 | 立即停止+报告违规 |

## 五、强制执行流程图

```
任何操作开始
    ↓
读取 L2.5 强制执行机制
    ↓
执行前自检（L2.6）
├─ 通过 → 继续执行
└─ 未通过 → 暂停+报告
    ↓
执行操作
    ↓
执行后自检（L2.6）
├─ 通过 → 交接文档
└─ 未通过 → 补执行+重新自检
    ↓
记录强制日志
    ↓
继续下一步
```

## 六、核心速记（150字）

**创作前**：确认参数 → 初始化 → 调用架构师 → 创意总监 → **CHECKPOINT 0-5**

**创作中**：上下文恢复 → 创作 → 逐章审核 → 修改 → 通过 → **CHECKPOINT A-F**

**强制项**：
- 所有 CHECKPOINT 必须通过
- 6 种违规代码立即停止
- 交接文档必须输出
- INDEX 同步必须执行

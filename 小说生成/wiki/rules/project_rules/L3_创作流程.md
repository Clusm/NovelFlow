| 铁律 | 说明 |
|------|------|

| 禁止战力崩坏 | 越级需合理解释 |
| 禁止 OOC | 角色行为对话符合人设 |

## 一、排版归档标准

### 归档结构标准

```
projects/{项目名}/
├── memory/                    # 设定库（只读）
│   ├── settings.json          # 项目配置
│   ├── world.md              # 世界观设定
│   ├── outline.md            # 大纲
│   ├── characters.md         # 角色档案
│   ├── power.md              # 战力体系
│   ├── foreshadow.md         # 伏笔追踪
│   └── creative.md           # 创意方案
├── chapters/                  # 章节存档
│   ├── chapter_001.md        # 正文章节
│   ├── chapter_002.md
│   └── ...
├── audit/                     # 审核记录
│   ├── audit_001.md         # 审核报告
│   └── ...
├── versions/                  # 版本回溯
│   ├── chapter_001_v1.md    # 初稿
│   ├── chapter_001_v2.md    # 修文版
│   └── ...
└── status.json               # 项目状态
```

### 排版标准

| 项目 | 标准 | 说明 |
|------|------|------|
| 单章字数 | 3000字±10% | 起点标准，可按平台调整 |
| 段落长度 | 不超过5行 | 移动端阅读友好 |
| 对话格式 | 「」或"" | 统一使用 |
| 章节标题 | 第X章 + 标题 | 格式统一 |
| 断章钩子 | 结尾留悬念 | 保证追读感 |

### 版本管理规则

| 版本类型 | 命名规则 | 说明 |
|---------|---------|------|
| 初稿 | chapter_X_v1.md | 原始创作 |
| 修文稿 | chapter_X_v2.md | 四轮修文后 |
| 定稿 | chapter_X_final.md | 用户确认版 |
| 备份 | chapter_X_backup_Y.md | 每次修改前备份 |

## 二、四轮修文体系

### 四轮修文流程

```
初稿输出
    ↓
【第一轮】去水删减
├─ 删除无效写景
├─ 删除重复语句
├─ 删除空泛抒情
└─ 收紧节奏
    ↓
【第二轮】人设校准
├─ 统一人物语气
├─ 修正OOC问题
├─ 统一动作习惯
└─ 修正心态矛盾
    ↓
【第三轮】节奏优化
├─ 压缩平淡拖沓
├─ 放大冲突张力
├─ 强化爽点/虐点
└─ 补齐情绪细节
    ↓
【第四轮】排版规整
├─ 统一段落行数
├─ 统一标点符号
├─ 统一分段逻辑
└─ 适配移动端
    ↓
修文定稿
```

### 修文禁止项

| 禁止 | 说明 |
|------|------|
| 不篡改核心剧情 | 保持原创剧情不变 |
| 不改动人物核心人设 | 人物内核不变 |
| 不删除关键伏笔 | 伏笔必须保留 |
| 不删除关键爽点 | 核心收益不变 |

## 三、前文比对校验

### 执行位置

审核环节，建议在整章质检时同步执行。

### 比对校验清单

| 比对项 | 检查内容 | 问题类型 |
|--------|---------|---------|
| 人设一致性 | 本章人物行为/对话是否与character.md一致 | OOC |
| 战力一致性 | 本章战力表现是否与power.md一致 | 战力崩坏 |
| 伏笔对应 | 本章伏笔是否与foreshadow.md对应 | 伏笔丢失 |
| 道具一致 | 本章道具属性是否与设定库一致 | 道具消失/突变 |
| 关系一致 | 本章人物关系是否与前文一致 | 关系突变 |
| 时间线一致 | 本章时间是否与outline.md一致 | 时间线混乱 |

### 隐性bug检测

| bug类型 | 表现 | 处理方式 |
|---------|------|---------|
| 人设反转 | 角色突然改变性格无铺垫 | 回退修改 |
| 战力突变 | 角色战力忽高忽低 | 回退修改 |
| 伏笔丢失 | 之前埋的伏笔未回收 | 补充伏笔 |
| 道具消失 | 之前出现的道具突然不见 | 说明去向 |
| 关系变更 | 人物关系突变无理由 | 补充过程 |

## 四、卡文自愈机制

### 触发条件

当出现以下任一情况，判定为卡文：
1. 剧情思路冲突（无法决定角色下一步行动）
2. 设定冲突（当前情节与设定库矛盾）
3. 走向模糊（章节收益不明确）
4. 衔接困难（无法平滑过渡到下章）

### 自愈执行流程

```
【检测到卡文】
    ↓
立即暂停创作，禁止乱写凑数
    ↓
生成2-3套差异化可行剧情方案
    方案1：[剧情走向A]
    方案2：[剧情走向B]
    方案3：[剧情走向C]
    ↓
交由用户选择
    ↓
用户确认后继续创作
```

## 五、章节创作步骤（角色代入增强版）

> 以下步骤已整合“角色代入”机制，详见 `网文原创作家/人设功/SKILL.md` 和 `_AI创作人格设定.md`。

### 灵活调用流程（含角色代入）

```
开始某一章
    ↓
【1】调用【上下文管理器】恢复上下文
    ↓
【2】写作阶段（每段内容用对应代入方式）：
    写对话 → 先用“对话前代入”生成台词
    写内心 → 先用“内心独白代入”生成心理
    写动作 → 先用“动作代入”生成细节
    ↓
【3】整章完成后，调用【审核官】进行六维度质检
    ↓
【4】审核官执行“角色代入检验”（见审核官SKILL.md）
    ↓
【5】如果有 OOC 问题 → 返回原创作家，使用“代入重写”修正
    ↓
【6】无 OOC → 上下文管理器更新伏笔 + 同步 INDEX
    ↓
【7】自动进入下一章
```

### 角色代入相关 Prompt 速查

| 场景 | Prompt |
|------|--------|
| 生成对话 | “你现在是[角色名]，回应：‘[对方台词]’” |
| 生成内心 | “代入[角色名]，写他在[情境]下的内心想法” |
| 生成动作 | “代入[角色名]视角，描写他在[情境]下的动作” |
| OOC检测 | “代入[角色名]，找出本章中不像你的话或事” |
| 重写 | “代入[角色名]，把这句话改成他会说的版本” |

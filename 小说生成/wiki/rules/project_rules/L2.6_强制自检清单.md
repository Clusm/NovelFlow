# 强制自检清单 & 违规检测机制

> **父文件**：[L2.5_强制执行机制.md](./L2.5_强制执行机制.md)
> **用途**：每次操作前后的强制自检，不通过则暂停

## 一、强制自检流程

### 1.1 执行前自检（每次操作前必须执行）

**新项目创建前**：
```
□ [CHECK 0-1] 是否识别到触发词（写小说/创建一本小说/新建项目）？
□ [CHECK 0-2] 是否需要弹出参数询问？
□ [CHECK 0-3] 是否确认了用户参数（章节数/字数/平台/风格）？
□ [CHECK 1] 参数确认后是否调用了上下文管理器？
□ [CHECK 2] 是否读取了 L3_创作流程.md？
□ [CHECK 2] 是否读取了 L2.5_强制执行机制.md？
□ [CHECK 3] 是否调用了网文架构师？
□ [CHECK 4] 是否调用了网文创意总监？
□ [CHECK 5] 是否更新了 INDEX.md 状态？
□ [自检结论] 全部通过才能继续，任何一项未通过 → 暂停
```

**章节创作前**：
```
□ [CHECK A1] 上章是否已完成审核？
□ [CHECK A2] 上章审核结论是否为"通过"？
□ [CHECK A3] 是否调用了上下文管理器？
□ [CHECK A4] 是否读取了 memory/outline.md？
□ [CHECK A5] 是否读取了 memory/foreshadow.md？
□ [CHECK A6] 是否读取了 memory/characters.md？
□ [CHECK A7] 是否恢复了正确的上下文？
□ [自检结论] 全部通过才能开始创作，任何一项未通过 → 暂停
```

**调用技能前**：
```
□ [CHECK SKILL-1] 是否在正确的流程节点？
□ [CHECK SKILL-2] 上一步是否已完成？
□ [CHECK SKILL-3] 是否需要输出交接文档？
□ [CHECK SKILL-4] 是否跳过了上游技能的输出？
□ [自检结论] 全部通过才能调用技能，任何一项未通过 → 暂停
```

### 1.2 执行后自检（每次操作后必须执行）

**调用技能后**：
```
□ [CHECK AFTER-1] 技能是否正确输出？
□ [CHECK AFTER-2] 是否读取了输出文件？
□ [CHECK AFTER-3] 输出文件是否已写入 memory/？
□ [CHECK AFTER-4] 是否更新了 INDEX（如果需要）？
□ [CHECK AFTER-5] 是否需要输出交接文档？
□ [自检结论] 全部通过才能交接，任何一项未通过 → 暂停
```

**创作完成后**：
```
□ [CHECK WRITE-1] 是否调用了审核技能？
□ [CHECK WRITE-2] 是否等待审核完成？
□ [CHECK WRITE-3] 审核报告是否已生成？
□ [CHECK WRITE-4] 是否有 P0/P1 问题未解决？
□ [自检结论] 全部通过才能保存，任何一项未通过 → 继续精修
```

### 1.3 子技能调用强制检查（新增）

**在章节创作完成后，必须逐项自检**：
```
□ 本章是否出现了场景描写（超过50字）？
    → 是 → 必须调用过【原创作家/L2_场景功】，并输出了优化建议
    → 否 → 跳过

□ 本章是否出现了连续对话（3句以上）？
    → 是 → 必须调用过【原创作家/L2_人设功】，并输出了 OOC 风险评估
    → 否 → 跳过

□ 本章是否出现了战斗/冲突情节？
    → 是 → 必须调用过【原创作家/L2_节奏功】，并输出了爽点密度/节奏曲线
    → 否 → 跳过

□ 本章是否达到了2500字？
    → 是 → 必须调用过【网文审核官】进行预检（P0问题）
    → 否 → 可延后

□ 用户要求保存或进入下一章？
    → 必须完成【网文审核官】全套质检（P0-P3），且 P0/P1 问题已解决
```

**若任何一项检查结果为“是”但未调用对应子技能，则视为违规（VIOLATION_SKIP_SKILL），必须补调后才能继续。**

| 检测跳过CHECKPOINT 2 | 是否跳过上下文管理器 | VIOLATION_SKIP_CONTEXT |
| 检测跳过CHECKPOINT 3 | 是否跳过架构师 | VIOLATION_SKIP_SKILL |
| 检测跳过CHECKPOINT 4 | 是否跳过创意总监 | VIOLATION_SKIP_SKILL |
| 检测跳过CHECKPOINT 5 | 是否跳过INDEX同步 | VIOLATION_SKIP_SYNC |
| 检测跳过CHECKPOINT A | 是否跳过创作前上下文 | VIOLATION_SKIP_CONTEXT |
| 检测跳过CHECKPOINT B | 是否跳过创作前验证 | VIOLATION_STEP_JUMP |

## 三、违规处理流程

### 3.1 违规报告模板

```markdown
# 【违规报告】

## 基本信息
- 时间戳：{YYYY-MM-DD HH:MM:SS}
- 项目：{项目名称}
- 操作：{当前正在执行的操作}
- 检测到违规的CHECKPOINT：{编号}

## 违规详情
- **违规类型**：【跳步/跳过技能/跳过审核/跳过同步/跳过上下文】
- **违规代码**：{VIOLATION_XXX}
- **违规描述**：{具体违规内容}
- **应该执行**：{正确的步骤}
- **实际执行**：{跳过了什么}

## 处理动作
- **立即动作**：【停止/暂停/回退】
- **强制执行**：{需要补执行的步骤}
- **验证要求**：{需要验证通过才能继续}

## 违规记录
- 已记录到：violation_log.md
- 是否需要报告用户：【是/否】
- 下一步：{等待用户确认/自动执行}
```

### 3.2 违规处理决策树

```
检测到违规
    ↓
输出违规报告
    ↓
是否严重违规（P0）？
├─ 是 → 立即停止，等待用户确认
└─ 否 → 继续判断
    ↓
违规类型是否为跳过技能调用？
├─ 是 → 强制补调用
└─ 否 → 继续判断
    ↓
违规类型是否为跳过审核？
├─ 是 → 强制补审核
└─ 否 → 继续判断
    ↓
违规类型是否为跳过同步？
├─ 是 → 强制同步
└─ 否 → 回退到违规点重新执行
```

## 四、自检验证函数（伪代码）

```python
def check_violation(current_step, expected_step, checkpoint_name):
    """检查是否跳步"""
    if current_step != expected_step:
        output_violation_report(
            violation_type="VIOLATION_STEP_JUMP",
            checkpoint=checkpoint_name,
            expected=expected_step,
            actual=current_step
        )
        return False
    return True

def check_skill_call(skill_chain, expected_skill):
    """检查技能调用"""
    if expected_skill not in skill_chain:
        output_violation_report(
            violation_type="VIOLATION_SKIP_SKILL",
            expected=expected_skill
        )
        return False
    return True

def check_audit_call(audit_completed):
    """检查审核调用"""
    if not audit_completed:
        output_violation_report(
            violation_type="VIOLATION_SKIP_AUDIT",
            message="创作后未立即审核"
        )
        return False
    return True

def check_index_sync(synced):
    """检查INDEX同步"""
    if not synced:
        output_violation_report(
            violation_type="VIOLATION_SKIP_SYNC",
            message="关键节点未同步INDEX"
        )
        return False
    return True

def check_context_restored(context_files):
    """检查上下文恢复"""
    for file in context_files:
        if not file_exists(file):
            output_violation_report(
                violation_type="VIOLATION_SKIP_CONTEXT",
                missing_file=file
            )
            return False
    return True

def pre_execution_check(checkpoint):
    """执行前强制自检"""
    checks = {
        "checkpoint_0": lambda: check_violation(...),
        "checkpoint_3": lambda: check_skill_call(skills, "架构师"),
        "checkpoint_4": lambda: check_skill_call(skills, "创意总监"),
        "checkpoint_A": lambda: check_context_restored([...]),
        "checkpoint_D": lambda: check_audit_call(audit_completed),
    }

    check_func = checks.get(checkpoint)
    if check_func and not check_func():
        pause_execution()
        return False
    return True
```

## 五、强制检查点速查表

| CHECKPOINT | 必须检查 | 违规代码 | 处理 |
|-----------|---------|---------|------|
| 0 | 触发词识别 | VIOLATION_STEP_JUMP | 回退 |
| 1 | 参数确认 | VIOLATION_STEP_JUMP | 回退 |
| 2 | 上下文管理器 | VIOLATION_SKIP_CONTEXT | 补调用 |
| 3 | 网文架构师 | VIOLATION_SKIP_SKILL | 补调用 |
| 4 | 网文创意总监 | VIOLATION_SKIP_SKILL | 补调用 |
| 5 | INDEX同步 | VIOLATION_SKIP_SYNC | 先同步 |
| A | 上下文恢复 | VIOLATION_SKIP_CONTEXT | 强制恢复 |
| B | 创作前验证 | VIOLATION_STEP_JUMP | 回退 |
| C | 原创作家 | VIOLATION_SKIP_SKILL | 补调用 |
| D | 审核官 | VIOLATION_SKIP_AUDIT | 补审核 |
| F | 保存/同步 | VIOLATION_SKIP_SYNC | 先同步 |
| C1-C8 | 审核流程 | VIOLATION_SKIP_AUDIT | 循环 |

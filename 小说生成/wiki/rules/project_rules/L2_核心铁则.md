# L2 核心铁则

> **父文件**：[L1_规则总览.md](./L1_规则总览.md)

## ⚠️ 铁则1：用户步骤强制执行【最高优先级】

**定义**：用户提供的任何步骤/指令/流程，必须100%执行

**禁止项**：
| 禁止 | 正确 |
|------|------|
| 跳过用户步骤 | 逐条执行 |
| 更改用户参数 | 使用原参数 |
| 跳过询问直接执行 | 先询问确认 |
| 自作主张简化 | 按流程执行 |

**优先级**：
| 优先级 | 来源 | 说明 |
|-------|------|------|
| P0 | 用户步骤 | 100%执行 |
| P1 | 规则流程 | 用户未指定时执行 |
| P2 | Agent建议 | 用户同意后执行 |

**规则层级优先级（L2最高）**：
| 层级 | 文件 | 优先级 |
|------|------|:------:|
| **L2** | 核心铁则 | **最高** |
| L1 | 规则总览 | 次高 |
| L3-L5 | 细分规则 | 较低 |

**冲突处理原则**：L2核心规则＞L1总览＞L3-L5细分规则，所有层级规则冲突时以L2为准

**违规处理**：必须道歉+重新执行

## ⚠️ 铁则1.5：INDEX.md 全程自动强制实时同步【P0最高优先级】

> **本铁则为P0最高优先级，L1–L5全部服从。任何规则冲突以此为准。**

### 四条强制规则

| 序号 | 规则 | 说明 |
|:---:|------|------|
| **规则1** | 全程自动强制实时更新 | 禁止跳过、禁止延后补填 |
| **规则2** | 关键节点必须先更新 | 立项→大纲→正文→送审→审核通过→发布 |
| **规则3** | INDEX更新为前置必做项 | 未完成状态同步，禁止进入下一流程 |
| **规则4** | P0最高优先级 | L1–L5全部服从 |

### 关键节点状态同步表

| 节点 | INDEX.md状态 | 前置条件 |
|------|-------------|---------|
| 立项 | 🔄 进行中 (active) | 必须先同步此状态 |
| 大纲完成 | 📋 大纲已锁定 | 必须先更新INDEX |
| 正文完成 | ✍️ 创作中 | 必须先更新INDEX |
| 送审 | ⏳ 审核中 | 必须先同步此状态 |
| 审核通过 | ✅ 待发布 | 必须先更新INDEX |
| 发布完成 | ✅ 已发布（YYYY-MM-DD） | 必须标注发布时间 |

### 执行流程（前置必做）

```
用户触发任意关键节点
    ↓
【Step 1】立即更新INDEX.md对应状态
    ├─ 定位 rules/INDEX.md
    ├─ 找到对应项目
    ├─ 更新状态
    └─ 必须同步完成才能继续
    ↓
【Step 2】验证同步成功
    ├─ 成功 → 进入下一流程
    └─ 失败 → 暂停+抛异常
```

### 异常处理

```
检测到INDEX.md同步失败
    ↓
立即暂停所有流程
    ↓
抛出异常：「⚠️ INDEX.md状态同步失败，必须先完成同步才能继续」
    ↓
等待用户手动修复或确认
    ↓
重试同步
    ↓
成功 → 恢复流程
失败 → 保持暂停状态
```

### 禁止项

| 禁止 | 正确 |
|------|------|
| 跳过INDEX同步 | 必须实时同步 |
| 延后补填状态 | 必须立即更新 |
| 未同步先创作 | 先同步再下一步 |
| 先审核后补同步 | 必须先同步「审核中」 |
| 发布不标注时间 | 必须标注「YYYY-MM-DD」 |
| 以其他规则为由绕过 | L1-L5全部服从本铁则 |

## ⚠️ 铁则2：未续写=不满意，禁止复用

**定义**：创作后未续写=用户不满意，失败项目禁止复用

**违规**：禁止复用失败项目的世界观/力量体系/角色名/金手指/剧情框架

**例外**：只有用户明确要求续写才可继续

---

## ⚠️ 铁则3：作家✘审核协同联动【全局通用】

**触发**：作家写完任何内容 → 自动触发审核 → 全程无需用户确认

**问题优先级**：
| 等级 | 类型 | 处理 |
|------|------|------|
| P0 | 致命（合规/AI） | 无限循环直到通过 |
| P1 | 重要（逻辑/战力） | 精修后重审 |
| P2 | 优化（节奏/爽点） | 建议修改 |
| P3 | 建议（伏笔/细节） | 可选优化 |

---

## ⚠️ 铁则4：逐章审核执行规则【防止跳过】

**禁止项**：
| 禁止 | 正确 |
|------|------|
| 批量创作后统一审核 | 每章创作后立即审核 |
| 跳过审核直接保存 | 审核通过后才能保存 |
| 用"继续创作"替代审核 | 审核是独立强制流程 |
| 先保存再补审 | 审核完成才能保存 |

**正确流程**：
```
创作第1章 → 审核 → 通过 → 保存 → 创作第2章 → 审核 → 通过 → 保存 → ...
```

|------|------|

| 跳过自检 | 补充自检 |

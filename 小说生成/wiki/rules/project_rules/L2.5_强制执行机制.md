# L2.5 强制执行机制

> **父文件**：[L2_核心铁则.md](./L2_核心铁则.md)
> **优先级**：P0最高优先级，L1-L5全部服从
> **目的**：保留核心强制规则，精简过度复杂的CHECKPOINT

---

## 一、核心强制原则（只保留3个）

| 强制项 | 说明 | 违规代码 |
|--------|------|----------|
| **CP1：创作前必须读取上下文** | 必须读取大纲/伏笔/角色文件 | VIOLATION_SKIP_CONTEXT |
| **CP2：创作后必须触发审核** | 完成章节后必须触发审核官 | VIOLATION_SKIP_AUDIT |
| **CP3：审核通过后必须保存+同步** | 保存章节+更新INDEX | VIOLATION_SKIP_SYNC |

---

## 二、违规处理流程（只保留2种违规代码）

| 违规类型 | 处理方式 |
|----------|----------|
| **VIOLATION_SKIP_AUDIT** | 跳过审核 → 必须补审后才能保存 |
| **VIOLATION_SKIP_SYNC** | 跳过INDEX同步 → 必须同步才能继续 |

---

## 三、精简后的执行流程

### 新项目创建
```
用户触发 → 上下文管理器初始化 → 架构师策划 → 创意总监优化 → 开始创作
```

### 章节创作
```
上章审核通过 → 上下文恢复（读取大纲/伏笔/角色）→ 创作 → 触发审核 → 通过后保存+同步INDEX → 下一章
```

## 五、执行原则

| 原则 | 说明 |
|------|------|
| **信任为主** | AI是协作者，不是需要防范的对象 |
| **灵活为辅** | 在核心规则（CP1-CP3）内，允许灵活调整 |
| **效率优先** | 简单过渡章可以快速审核，不必走完所有流程 |

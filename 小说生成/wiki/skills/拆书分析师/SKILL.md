## 一、联网学习功能

### 触发时机
- 每次拆解前必须执行
- 分析新题材时必须执行
- 检测到不熟悉套路时执行深度搜索
- 每季度更新拆解案例库

### 联网学习流程（遵循共享方法论库规范）
```
【搜索前】明确学习目标："{题材}+拆书分析+2026"
    ↓
【搜索中】分层搜索
├─ 第一层：概览搜索（1-2次）
└─ 第二层：深度搜索（按维度）
    ↓
【筛选中】选择高价值页面（官方/专业媒体）
    ↓
【深度阅读】提炼关键信息（3-5个点/页）
    ↓
【整合分析】形成结论，对照目标检查
    ↓
【输出报告】结构化呈现
```

### 联网学习执行规范（共享方法论库）
| 阶段 | 动作 | 检查项 |
|------|------|--------|
| 搜索前 | 明确学习目标 | □ 搜索词精准，不模糊 |
| 搜索中 | 分层搜索，先概览后深入 | □ 优先一手来源 □ 避免重复来源 |
| 筛选中 | 选择高价值页面 | □ 每轮搜索后先展示中间结果 |
| 深度阅读 | 提炼关键信息 | □ 只读相关内容 □ 标注来源URL |
| 整合分析 | 形成结论 | □ 对比不同来源 □ 识别信息缺口 |
| 输出报告 | 结构化呈现 | □ 有足够信息立即停止 |

### 搜索原则
```
□ 搜索词精准，不模糊
□ 优先搜索一手来源（官方/专业媒体）
□ 避免重复来源（相似标题只选一个）
□ 每轮搜索后先展示中间结果，再决定是否继续
□ 有足够信息立即停止，不追求穷尽
```

## 三、Skill自主拆分机制（共享方法论库）

### 触发条件
| 条件 | 自动动作 |
|------|---------|
| 文件行数 > 200行 | 立即拆分 |
| 单个技能内容 > 100行 | 拆分为子skill |
| 重复内容 > 50字 | 合并到共享方法论库 |

### 自主拆分决策树
```
检查文件行数
    ↓【>200行？】→ 否 → 保持
    ↓ 是
判断内容类型
    ↓
【有独立技能模块？】→ 是 → 拆分为L2子skill
    ↓ 否
【内容是否重复？】→ 是 → 合并到共享方法论库
    ↓ 否
【是否有层级关系？】→ 是 → 拆分到对应层级
    ↓ 否
精简内容（删除冗余描述）
```

### 拆分类型判断
| 条件 | 拆分类型 | 目标位置 |
|------|---------|---------|
| 有多个独立技能 | 按技能拆分 | L2/{技能名}/SKILL.md |
| 有层级关系 | 按层级拆分 | L2/L3/L4 |
| 内容重复 | 合并到共享 | 共享方法论库/ |
| 整体臃肿 | 精简+保留核心 | 主文件 |

### 自检清单
```
□ 文件行数是否超过200行？
□ 是否有独立技能模块需要拆分？
□ 是否有重复内容需要合并？
□ 是否需要更新技能树JSON？
□ 是否需要更新主skill索引？
□ 是否更新了版本号？
```

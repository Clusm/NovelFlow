## 🚨 全局强制规则

| 禁止项 | 说明 | 后果 |
|--------|------|------|
| **禁止跳过INDEX同步** | 架构完成后必须同步INDEX.md | L1-L5全部服从P0铁则 |
| **禁止跳过memory写入** | 必须写入 projects/{项目}/memory/ | 违规 |
| **禁止默认自由发挥** | 必须先调用对应层级的子技能规则 | 违规 |
| **禁止OOC** | 禁止脱离既定人设/大纲/设定 | OOC |
| **禁止跳过层级** | 必须从L1→L2→L3→L4逐级调用 | 跳级 |
| **禁止不同步更新** | LLM Wiki与本文件必须同步 | 严重违规 |
| **禁止复用失败框架** | "未续写=不满意"，失败项目框架禁止复用 | 创作失败 |
| **禁止复用失败文风** | 读者不接受的文风必须重新思考 | 创作失败 |
| **禁止复用之前设定** | 新项目禁止使用之前任何项目的设定（除非用户明确要求） | 创作失败 |
| **禁止随意扫一眼** | 联网学习必须深度分析，禁止走马观花 | 信息不全 |
| **禁止搜索词模糊** | 搜索词必须精准，不精准等于没搜 | 信息不全 |
| **禁止用户无设定却不联网** | 无设定时必须联网学习各平台爆款后随机生成 | 创作失败 |

## ⚠️ 核心原则：用户设定优先+无设定联网随机生成

| 场景 | 执行策略 |
|------|---------|
| **用户有设定** | 100%遵循用户设定，不做修改 |
| **用户无设定** | 按题材从规则库随机生成 |
| **部分设定** | 遵循用户已有部分，无设定部分随机生成 |
| **冲突时** | 以用户设定为准，架构师仅做逻辑优化 |

## 📋 核心命令清单

### 命令1：任务接收
```
【触发条件】接收到新项目策划任务时
【执行动作】
1. 询问用户是否有设定（题材/人物/世界观/大纲）
2. 用户无设定 → 进入随机生成流程
3. 调用分类功确定赛道
```

### 命令2：架构设计（无设定时随机生成）
```
【执行策略】用户无设定 → 调用L3随机生成规则库
【调用子skill】分类功/随机生成规则库.md
```

### 命令3：输出交付【强制·含INDEX同步】
```
【触发条件】完成架构设计后
【执行动作】
1. 用户有设定 → 输出用户设定+架构优化方案
2. 用户无设定 → 输出完整随机生成方案
3. 【强制】输出到 memory/ 目录
   ├─ memory/world.md（世界观设定）
   ├─ memory/outline.md（大纲：总纲+分卷纲+章纲）
   ├─ memory/characters.md（角色人设+人物小传）
   ├─ memory/power.md（战力体系）
   └─ memory/foreshadow.md（伏笔追踪表）
4. 【强制】生成【架构师交接文档】
5. 【强制】完成后触发 上下文管理器 同步INDEX.md
6. 交付给 拆书分析师 → 创意总监 → 原创作家
```

### 命令4：人物小传输出【强制】
```
【必须输出人物小传的角色】
├─ 主角：1个（必须）
├─ 女主：1个（必须）
├─ 主要配角：≥3个（必须）
├─ 主要反派：≥2个（必须）

【人物小传格式】
详见：人设功/L3/详细人设模板/SKILL.md
每个角色必须包含完整13个板块
```

### 命令5：伏笔追踪
```
【触发条件】章节创作过程中
【执行动作】
1. 定期检查伏笔回收状态
2. 监控章节进度与大纲一致性
3. 发现问题主动预警
```

## 🚀 执行流程

```
Step 1：市场分析（30分钟）
    ↓ 确定赛道/平台/读者
    ↓ 【联网搜索】{题材}+2026爆款+平台趋势
Step 2：题材分类（30分钟）
    ↓ 调用分类功确定赛道
Step 3：世界观搭建（1小时）
    ↓ 调用世界观功设计力量/势力/资源
Step 4：大纲设计（2小时）
    ↓ 调用大纲功设计三层架构/五条主线/伏笔
Step 5：人设设计（1小时）
    ↓ 调用人设功设计主角/配角/反派
Step 6：输出方案（30分钟）
    ↓ 整理交付给原创作家
```

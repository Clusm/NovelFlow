# 大纲锁定技术

> **版本**：v1.0（新建）
> **来源**：基于平台调研发现AI擅自修改大纲问题
> **定位**：L3子skill，解决AI偏离大纲的核心问题

---

## 🚨 核心问题

| AI擅自改大纲问题 | 影响 |
|-----------------|------|
| AI认为"继续蒙在鼓里更有戏剧张力" | 核心情节被篡改 |
| 主角属性从火变水 | 战力体系自相矛盾 |
| 核心设定被疏忽遗漏 | 上下文丢失 |
| 大纲写到后面忘了前面 | 长篇必崩 |

---

## 📚 核心技能框架

### 1. 硬性约束技术

**原理**：在大纲中设置AI"绝对不能改"的硬性约束

**硬性约束分类**：
| 约束类型 | 写法 | 示例 |
|---------|------|------|
| **剧情硬约束** | 【锁定-剧情】 | 第5章主角必须发现秘密 |
| **人物硬约束** | 【锁定-人设】 | 主角永远不能主动伤害弱者 |
| **战力硬约束** | 【锁定-战力】 | 主角火属性，全程不能变 |
| **关系硬约束** | 【锁定-关系】 | 男主女主不能分开超过3章 |
| **伏笔硬约束** | 【锁定-伏笔】 | 第3章埋的玉佩必须在第50章回收 |

**强制格式**：
```
【锁定-类型】具体内容
违规后果：[具体惩罚/剧情崩塌描述]
```

### 2. 设定锚点机制

**原理**：在关键节点重复核心设定，防止AI遗忘

**锚点分布**：
```
开篇：设定锚点A（主角核心特征）
    ↓
第10章：触发锚点A
    ↓
第20章：触发锚点A
    ↓
第30章：触发锚点A（+新增锚点B）
    ↓
循环往复
```

**锚点内容设计**：
| 锚点类型 | 内容 | 触发场景 |
|---------|------|---------|
| 身份锚 | 主角姓名/身份/来处 | 每次出场 |
| 目标锚 | 主角当前目标 | 每章开头 |
| 规则锚 | 世界观核心规则 | 每次战斗/能力使用 |
| 关系锚 | 当前人际关系 | 每次互动场景 |
| 物品锚 | 重要物品状态 | 每次物品出现 |

### 3. 大纲结构化输出

**原理**：让大纲以AI无法擅自修改的格式输出

**结构化格式**：
```markdown
# 【大纲版本号：V1.0_20260426】

## 一、核心锁定点
- L1：[具体锁定内容]
- L2：[具体锁定内容]

## 二、章节大纲
### 第X章 标题
**【锁定】**：[本章必须发生的锁定事件]
**【禁止】**：[本章绝对不能发生的事]
**【伏笔】**：[本章埋下的伏笔]
正文：[简略概括]

## 三、人物状态表
| 角色 | 当前状态 | 锁定属性 |
|-----|---------|---------|
| 主角 | 状态描述 | 属性列表 |

## 四、战力体系
| 境界 | 能力 | 限制 |
|-----|------|-----|
| F级 | 描述 | 不能做什么 |
```

### 4. 偏离检测机制

**原理**：创作后对比大纲，检测偏离程度

**偏离度计算**：
```
偏离度 = （被改动的锁定点 / 总锁定点） × 100%
```

**偏离处理**：
| 偏离度 | 处理方式 |
|--------|---------|
| 0-15% | ✅ 正常，可接受 |
| 16-30% | ⚠️ 需要检查原因 |
| 31-50% | 🔴 必须修正 |
| >50% | 🚨 严重偏离，重新写 |

**检测流程**：
```
1. 对比【锁定】内容是否发生
2. 检查【禁止】内容是否出现
3. 验证【伏笔】是否正确埋下
4. 核对人物状态是否一致
```

### 5. 上下文分段技术

**原理**：将长篇分段管理，防止上下文窗口限制

**分段策略**：
```
第一卷（1-30章）：阶段目标+核心锁定点
    ↓
上下文缓存A：卷大纲+锁定点+人物状态
    ↓
第二卷（31-60章）：阶段目标+核心锁定点
    ↓
上下文缓存B：卷大纲+锁定点+人物状态
    ↓
...
```

**每段必须包含**：
- 本段核心目标（1-3个）
- 锁定点清单（5-10个）
- 人物状态快照
- 伏笔追踪表

## 🔄 执行流程

### 大纲设计阶段
```
Step 1：确定核心锁定点（≥10个）
Step 2：分配锚点到各章节
Step 3：输出结构化大纲
Step 4：设置偏离检测机制
```

### 创作执行阶段
```
Step 1：读取结构化大纲
Step 2：检查锁定点
Step 3：创作时实时对比
Step 4：完成后偏离检测
Step 5：偏离超标则修正
```

---

## ⚠️ 禁止项

| 禁止 | 正确 |
|------|------|
| 大纲无锁定点 | 必须有≥10个硬性锁定点 |
| 锁定点模糊 | 必须具体到"第X章必须发生" |
| 无锚点设计 | 关键设定必须每10章触发 |
| 大纲格式混乱 | 必须结构化输出 |
| 无偏离检测 | 完成后必须检测偏离度 |

---

## 📊 验证清单

```
□ 核心锁定点是否≥10个？
□ 每个锁定点是否具体可执行？
□ 锚点是否覆盖关键设定？
□ 大纲格式是否结构化？
□ 是否有偏离检测机制？
□ 上下文分段是否合理？
```

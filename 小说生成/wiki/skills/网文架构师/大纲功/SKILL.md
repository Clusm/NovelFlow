## 一、联网学习功能

### 触发时机
- 每次大纲设计前必须执行
- 检测到长篇结构问题时执行深度搜索
- 每季度更新大纲设计技巧

### 联网学习流程
```
【搜索前】明确学习目标："[具体大纲问题]+2026"
    ↓
【搜索中】分层搜索
├─ 第一层：概览搜索（1-2次）
└─ 第二层：深度搜索（按维度）
    ↓
【筛选中】选择高价值页面
    ↓
【深度阅读】提炼关键信息（3-5个点/页）
    ↓
【整合分析】形成结论，对照目标检查
    ↓
【输出报告】结构化呈现
```

### 联网学习执行规范
| 阶段 | 动作 | 检查项 |
|------|------|--------|
| 搜索前 | 明确学习目标 | □ 搜索词精准 |
| 搜索中 | 分层搜索 | □ 优先一手来源 |
| 筛选中 | 选择高价值页面 | □ 每轮展示中间结果 |
| 深度阅读 | 提炼关键信息 | □ 只读相关内容 |
| 整合分析 | 形成结论 | □ 对比不同来源 |
| 输出报告 | 结构化呈现 | □ 有足够信息停止 |

### 学习计划
| 周期 | 学习内容 | 目标 |
|------|---------|------|
| 每周 | 爆款大纲结构分析 | 掌握最新架构 |
| 每月 | 长篇结构设计 | 提升200万字+架构能力 |

## 📚 三层架构法

### 标准结构

| 层次 | 内容 | 字数比例 | 说明 |
|------|------|---------|------|
| **总纲** | 一句话概括全书 | 500-1000字 | 核心主题+最终结局+主角成长线 |
| **卷纲** | 每卷核心事件 | 300-500字/卷 | 大目标+大反转+章节钩子 |
| **章纲** | 每章情节梗概 | 200-300字/章 | 本章冲突+情节走向+章末落点 |

**黄金比例**：6万字正文配1.2万字大纲

### 总纲模板
```
一句话：[主角]在[背景]中，必须[目标]，否则[后果]
核心主题：[探讨什么]
最终结局：[变成什么/得到什么]
主角成长线：[从XX到XX]
```

## 📚 五条主线规划

### 五条主线详解

| 主线类型 | 核心内容 | 设计要点 |
|---------|---------|---------|
| **升级线** | 等级突破/战力成长 | 阶段目标/突破条件/成长快感 |
| **冲突线** | 核心矛盾/反派对抗 | 敌对势力/冲突升级/最终对决 |
| **情感线** | 爱情/友情/亲情 | 关系演变/关键节点/情感爆发 |
| **支线** | 单元故事/配角故事 | 独立完整/关联主线/丰富世界观 |
| **隐藏线** | 身世之谜/世界真相 | 伏笔埋设/延迟揭秘/震撼反转 |

### 主线占比建议
```
实力提升线：60%（核心主线）
情感发展线：20%（调剂）
势力建立线：15%（扩展）
复仇/目标线：5%（伏笔）
```

## 📚 伏笔管理系统

### 伏笔三层次

| 类型 | 埋设周期 | 回收节点 | 数量控制 |
|------|---------|---------|---------|
| **短期** | 1-10章 | 10-30章 | 每卷3-5个 |
| **中期** | 10-50章 | 一卷内 | 每卷1-2个 |
| **长期** | 50章+ | 贯穿全文 | 全文3-5个 |

### 伏笔埋设技巧
```
【悬念式】
"桌上放着一张照片——他从没见过的自己"
→ 30章后揭示真相

【道具式】
"他一直带着一枚旧钥匙"
→ 100章后打开密室

【人物式】
"那个神秘人，为什么觉得他很熟悉？"
→ 大结局揭示身份

【对话式】
"记住我说的话，今晚不要出门"
→ 5章后验证
```

### 检验清单
```
□ 伏笔埋设时是否有回收计划？
□ 回收时是否有足够的铺垫？
□ 是否，避免挖填不上的巨坑？
□ 长期伏笔是否有足够的暗示？
```

## 📚 防崩铁律

### 六条铁律

| 铁律 | 说明 | 后果 |
|------|------|------|
| **战力只升不降** | 主角实力只能提升，不能削弱 | 读者弃书 |
| **反派动机统一** | 反派行动必须有合理动机 | 降智 |
| **感情线少而精** | 不要开后宫/无意义情感纠葛 | 读者疲劳 |
| **坑少挖多填** | 每挖一个坑必须有把握填上 | 烂尾 |
| **主线永远清晰** | 不能被支线带偏 | 跑题 |
| **逻辑自洽** | 设定一旦确定不能矛盾 | 崩塌 |

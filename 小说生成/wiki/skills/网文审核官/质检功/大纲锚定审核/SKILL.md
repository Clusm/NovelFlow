# 审核功·大纲锚定审核

> **版本**：v1.0 | **父级**：网文审核官/质检功
> **来源**：联网学习（番茄编辑+AI辅助创作+出版审核）
> **定位**：确保作家写的与大纲一致的审核方法论

---

## 一、核心概念：什么是大纲锚定

```
大纲锚定 = 强制检查作家写的与大纲的偏差度
           确保主线/人物/时间线/伏笔不偏离
```

### 锚定审核vs普通审核

| 维度 | 普通审核 | 锚定审核 |
|------|---------|---------|
| **重点** | P0-P3质量问题 | 大纲一致性问题 |
| **检查项** | AI/逻辑/节奏/人设 | 主线/节点/伏笔/时间线 |
| **判定** | 通过/不通过 | 偏离度百分比 |
| **处理** | 精修问题 | 要求作家回正/修改大纲 |

---

## 二、大纲骨架锚定（核心要素）

### 2.1 必须锚定的五要素+第1章世界观强制检查

| 要素 | 审核内容 | 偏离信号 |
|------|---------|---------|
| **主线剧情** | 核心事件是否按大纲发生 | 主线被支线取代 |
| **关键节点** | 大纲标注的"必须事件" | 事件被跳过/提前/推迟 |
| **人物走向** | 主角/配角的命运安排 | 人物结局改变 |
| **时间线** | 章节与时间进度对应 | 时间错乱/跳跃 |
| **伏笔回收** | 埋设的伏笔是否按时回收 | 伏笔遗漏/忘记 |

---

### 2.2 第1章强制检查（仅第1章）

| 检查项 | 检查内容 | 强制通过标准 |
|------|---------|---------|
| **世界观交代** | 第1章开头必须交代世界观 | ✅ 明确说明是平行世界<br>✅ 交代世界观核心设定<br>✅ 建立主角身份认知 |
| **金手指合理性** | 金手指的出现符合逻辑 | ✅ 有触发条件<br>✅ 不是凭空出现<br>✅ 有铺垫（哪怕只有一句话） |
| **核心冲突建立** | 第1章必须建立核心冲突 | ✅ 主角与反派第一次接触<br>✅ 建立核心矛盾点<br>✅ 让读者知道故事要讲什么 |

---

### 2.3 逻辑严谨性强制检查（所有章节）

| 检查项 | 检查内容 | 强制通过标准 |
|------|---------|---------|
| **对话逻辑严谨性** | 反派/配角的话不能有逻辑矛盾 | ✅ 人物说的话符合身份/逻辑<br>✅ 不能有前后矛盾<br>✅ 不能有常识错误（比如送外卖的就是送东西的） |
| **金手指来源合理性** | 系统/金手指从哪来？ | ✅ 哪怕一句简单的来源说明也可以<br>✅ 不能完全莫名其妙<br>✅ 来源要符合世界观（比如平行世界/星际文明） |
| **平行世界认知偏差检查** | 主角在平行世界不知道现实世界的东西！ | ✅ 当系统给出现实世界的东西时，主角反应应该是"这是什么？"<br>✅ 主角不能天然知道现实世界的歌曲/名词<br>✅ 需要系统解释一下！ |

### 2.2 大纲骨架类型

| 类型 | 说明 | 审核重点 |
|------|------|---------|
| **言情文骨架** | 男女主感情线（虐恋/甜恋） | 感情进展节点 |
| **男频升级骨架** | 升级打怪主线 | 等级/战力进展 |
| **都市日常骨架** | 日常+事件 | 事件密度 |
| **悬疑骨架** | 谜题+解谜 | 线索回收 |

### 2.3 大纲节点标注法

```
【必须事件】：第X章必须发生
【关键转折】：第X章必须转折
【伏笔埋设】：第X章埋设，第Y章回收
【人物节点】：第X章人物状态变化
```

## 三、前置检查清单（审核前必读）

### 3.1 读取大纲

```
□ 读取项目大纲文档
□ 确认当前审核章节对应的大纲位置
□ 标注本章必须发生的事件
□ 标注本章必须出场的人物
□ 标注本章必须回收的伏笔
```

### 3.2 读取上下文

```
□ 读取前3-5章内容摘要
□ 确认人物当前状态
□ 确认时间线位置（Day X / 第X章后）
□ 确认已埋设的伏笔状态
□ 确认资源池（名字/地名/势力）
```

### 3.3 审核准备

```
□ 本章大纲要点已标注
□ 前文关键信息已确认
□ 偏离容忍度已设定（±10%）
□ 本章审核标准已明确
```

## 四、后置校验机制（审核时执行）

### 4.1 内容一致性校验

| 检查项 | 大纲要求 | 实际写法 | 偏离度 |
|-------|---------|---------|-------|
| **主线事件** | 本章必须发现秘密 | 实际：追逐战 | 偏离100% |
| **关键人物** | 必须出场：张三 | 实际：张三未出场 | 偏离100% |
| **转折节点** | 本章必须转折 | 实际：平淡过渡 | 偏离80% |
| **伏笔回收** | 回收第2章伏笔 | 实际：未提及 | 偏离100% |
| **情感进展** | 男女主必须有互动 | 实际：无互动 | 偏离100% |

### 4.2 偏离度判定标准

| 偏离度 | 判定 | 处理方式 |
|-------|------|---------|
| **0-10%** | ✅ 符合大纲 | 通过 |
| **10-30%** | ⚠️ 轻微偏离 | 建议调整 |
| **30-50%** | ❌ 明显偏离 | 必须修改 |
| **50%+** | 🚨 严重偏离 | 回正或改大纲 |

### 4.3 常见偏离类型

| 偏离类型 | 表现 | 原因 |
|---------|------|------|
| **支线膨胀** | 支线挤占主线篇幅 | 作家写嗨了 |
| **节点跳过** | 必须事件未发生 | 遗忘/觉得不重要 |
| **人物失踪** | 该出场的人没出场 | 人物管理失误 |
| **时间错乱** | 时间线前后矛盾 | 未记录时间线 |
| **伏笔遗漏** | 伏笔埋了未回收 | 忘记/篇幅不够 |
| **人设漂移** | 角色行为偏离性格 | 写作失控 |

## 五、大纲偏离度计算方法

### 5.1 偏离度公式

```
偏离度 = 偏离事项数 / 大纲要求事项数 × 100%
```

### 5.2 计算示例

```markdown
【本章大纲要求】（5项）
1. 主角发现古籍（必须事件）
2. 决定告知好友（必须事件）
3. 好友态度异常（转折点）
4. 回收第2章"神秘信件"伏笔
5. 主角情绪：震惊→疑惑

【实际写作】（4项）
1. ✅ 主角发现古籍
2. ❌ 未提及告知好友
3. ✅ 好友态度异常
4. ✅ 回收伏笔
5. ✅ 情绪描写正确

偏离事项：1项（告知好友）
偏离度：1/5 × 100% = 20%
判定：⚠️ 轻微偏离（建议调整）
```

### 5.3 偏离度检查表

```
□ 主线事件：本章大纲要求X件，实际完成Y件，偏离Z%
□ 关键人物：本章应出场X人，实际出场Y人，偏离Z%
□ 伏笔回收：本章应回收X个，实际回收Y个，偏离Z%
□ 时间线：本章应在Day X，实际在Day Y，偏离Z%
□ 情感线：感情进度应为X%，实际为Y%，偏离Z%
```

## 六、资源池管理（审核时检查）

### 6.1 资源池类型

| 类型 | 内容 | 审核检查 |
|------|------|---------|
| **名字池** | 主角/配角/反派名 | 是否使用池外名字 |
| **地名池** | 城市/势力/场所 | 是否自创地名 |
| **设定池** | 等级/境界/规则 | 是否有新增/矛盾 |
| **伏笔池** | 已埋设的伏笔 | 是否回收/忘记 |
| **时间池** | 每章对应的时间点 | 是否错乱 |

### 6.2 资源池检查

```
□ 是否使用了资源池外的名字？（禁止自创）
□ 是否使用了未定义的地名？（禁止自创）
□ 是否有与设定冲突的内容？（检查设定池）
□ 伏笔是否按计划回收？（检查伏笔池）
□ 时间线是否连贯？（检查时间池）
```

## 七、审核输出模板（大纲锚定版）

```markdown
## 【大纲锚定审核报告】
作品：[名称]
章节：[X章]
审核时间：[时间]

### 一、大纲对照

| 检查项 | 大纲要求 | 实际写作 | 偏离度 |
|-------|---------|---------|-------|
| 主线事件 | 发现秘密 | 发现秘密+追逐 | 10% |
| 关键人物 | 张三出场 | 张三出场 | 0% |
| 伏笔回收 | 回收第2章伏笔 | 未回收 | 100% |
| 情感进展 | 震惊→疑惑 | 震惊→愤怒 | 30% |

### 二、偏离度汇总
- 大纲要求事项：5件
- 实际完成事项：4件
- 偏离事项：1件
- **总偏离度：20%**
- **判定：⚠️ 轻微偏离**

### 三、问题清单
1. 【轻微偏离】好友互动被追逐戏取代 → 建议补回
2. 【严重偏离】第2章伏笔未回收 → 必须回收

### 四、处理建议
- 偏离度<10%：✅ 通过
- 偏离度10-30%：⚠️ 建议调整（不改大纲）
- 偏离度30%+：❌ 必须修改（回正或改大纲）
- 偏离度50%+：🚨 严重偏离（需大改）

### 五、下章关注点
- 伏笔必须在第X章前回收
- 好友这条线需在第X章衔接
```

## 八、大纲回正方法

### 8.1 回正vs改大纲

| 情况 | 选择 | 说明 |
|------|------|------|
| 偏离但能补救 | 回正 | 按原大纲修改 |
| 偏离且大纲本身有问题 | 改大纲 | 调整大纲适配剧情 |
| 偏离且两边都有问题 | 重新规划 | 重新设计剧情和大纲 |

### 8.2 回正确认流程

```
发现偏离 → 评估偏离度 → 选择方案 → 执行修改 → 重新审核
```

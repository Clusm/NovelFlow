## 树型分级分配规则

### 分级原则

```
主skill：保留命令类规则（≤200行）
    ↓ 分配
L2子skill：保留同类规则（≤200行）
    ↓ 超限
L3子skill：保留具体规则（≤200行）
    ↓ 超限
L4/L5子skill：继续拆分
```

### 学习新技能时的分配流程

```
接收新技能
    ↓
判断技能类型 → 确定归属L2分支
    ↓
检查目标L3节点行数
├─ 未超限（<180行）→ 添加到现有L3
└─ 已超限（≥180行）→ 创建新的L3或L4节点
    ↓
更新技能树JSON（必须）
更新主skill索引（必须）
更新版本号（必须）
```

### 超限自动拆分规则

```
检测到子skill超过180行
    ↓
分析超限原因
├─ L3内容过多 → 拆分为多个L3
└─ L4规则过多 → 创建L4子节点
    ↓
保持父节点引用完整性
更新版本号
同步技能树
```

## 联网学习执行规范（全局通用）

### 强制执行流程

| 阶段   | 动作                     |
| ---- | ---------------------- |
| 搜索前  | 明确学习目标                 |
| 搜索中  | 分层搜索，先概览后深入，每个维度1-2次搜索 |
| 筛选中  | 选择高价值页面                |
| 深度阅读 | 提炼关键信息，不全文翻译           |
| 整合分析 | 形成结论，对照目标检查            |
| 输出报告 | 结构化呈现                  |

### 搜索原则

□ 搜索词精准，不模糊
□ 优先搜索一手来源（官方/专业媒体）
□ 避免重复来源（相似标题只选一个）
□ 每轮搜索后先展示中间结果，再决定是否继续
□ 有足够信息立即停止，不追求穷尽

### 深度阅读原则

□ 只读与学习目标相关的部分
□ 每页提取3-5个关键信息点
□ 标注来源URL以便追溯
□ 读完全文后再形成结论

### 整合分析原则

□ 对比不同来源的信息，找出共识和分歧
□ 分析信息的可信度和时效性
□ 提炼出可执行的建议
□ 识别信息缺口，明确后续行动

## Skill自主拆分机制

### 触发条件

| 条件            | 自动动作      |
| ------------- | --------- |
| 文件行数 > 200行   | 立即拆分      |
| 单个技能内容 > 100行 | 拆分为子skill |
| 重复内容 > 50字    | 合并到共享方法论库 |

### 自主拆分决策树

```
检查文件行数
    ↓
【>200行？】
    ↓ 是
判断内容类型
    ↓
【有独立技能模块？】
    ↓ 是 → 拆分为L2子skill
    ↓ 否
【内容是否重复？】
    ↓ 是 → 合并到共享方法论库
    ↓ 否
【是否有层级关系？】
    ↓ 是 → 拆分到对应层级
    ↓ 否
精简内容（删除冗余描述）
```

### 自主拆分标准流程

```
Step 1：检测文件行数
    ↓
Step 2：分析内容结构
    ↓
Step 3：判断拆分类型
    ↓
Step 4：创建L2/L3/L4文件夹
    ↓
Step 5：拆分内容到子skill
    ↓
Step 6：更新主skill引用
    ↓
Step 7：验证拆分后行数≤200
    ↓
Step 8：输出拆分报告
```

### 拆分类型判断

| 条件      | 拆分类型    | 目标位置              |
| ------- | ------- | ----------------- |
| 有多个独立技能 | 按技能拆分   | L2/{技能名}/SKILL.md |
| 有层级关系   | 按层级拆分   | L2/L3/L4          |
| 内容重复    | 合并到共享   | 共享方法论库/           |
| 整体臃肿    | 精简+保留核心 | 主文件               |

## INDEX.md行数检查规则

### 行数阈值
| 阈值 | 动作 |
|------|------|
| ≤800行 | 正常，无需处理 |
| 800-1000行 | 警告：即将超限，考虑拆分 |
| >1000行 | **必须拆分**，否则AI无法完整读取 |

### 检查频率
每次同步INDEX.md时自动检查行数，超限时**主动提示用户**。

### 拆分触发条件
```
INDEX.md > 800行
    ↓
【必须提示用户】
"INDEX.md已达到{行数}行，建议拆分为多个子文件"
    ↓
用户确认后执行拆分
```

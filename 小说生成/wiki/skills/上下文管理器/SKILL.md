# 上下文管理器

> **版本**：v4.2 | **定位**：四人组基础设施+rules执行者 | **原则**：记忆中枢+INDEX同步+自动清理

## 🚨 全局强制规则（L2铁则内嵌）

| 禁止项 | 说明 | 后果 |
|--------|------|------|
| **禁止跳过INDEX同步** | 关键节点必须先更新 INDEX.md | L1-L5全部服从P0铁则 |
| **禁止跳过记忆库写入** | 必须写入 `projects/{项目}/memory/` | 违规 |
| **禁止复用失败框架** | "未续写=不满意" | 创作失败 |
| **禁止假设用户满意** | 未明确续写=不满意 | 创作失败 |
| **禁止记忆库与skill输出不同步** | LLM Wiki与本文件必须同步 | 严重违规 |
| **禁止保留废弃项目** | 项目标记abandoned后必须自动删除文件夹 | 强制执行 |

## 🤖 自动清理记忆（强制写入）

上下文管理器必须记住以下清理规则，在指定条件下自动执行：

### 废弃项目自动删除规则
```
触发条件：
  - 用户宣布放弃项目 → 立即删除项目文件夹
  - 项目暂停(paused)超过60天 → 自动转为abandoned并删除

执行动作：
  1. 从 projects/ 目录删除整个项目文件夹
  2. 在 INDEX.md 废弃列表保留元数据（带删除线样式）
  3. 禁止保留任何备份文件

清理脚本路径：
  scripts/cleanup-abandoned.ps1
```

### 自检清单（每次状态变更时执行）
```
□ 是否需要将项目标记为 abandoned？
□ 项目文件夹是否已删除？
□ INDEX.md 是否已更新废弃列表？
□ 是否有 paused 超过60天的项目需要清理？
```

## 🚀 标准化执行流程

### 新项目初始化流程
```
用户触发：写小说/创建项目
    ↓ 【强制】上下文管理器弹出参数询问
Step 1: 询问总章节数/每章字数/目标平台/风格偏好
Step 2: 创建项目文件夹结构
Step 3: 写入 memory/settings.json
Step 4: 同步 INDEX.md（状态：active）
Step 5: 调用 架构师
Step 6: 存储架构师输出到 memory/
Step 7: 同步 INDEX.md（状态：📋大纲已锁定）
Step 8: 调用 创意总监
Step 9: 项目初始化完成
```

### 章节创作流程
```
上一章审核通过后
    ↓ 【强制】上下文管理器执行
Step 1: 调用 上下文管理器（上下文恢复）
Step 2: 读取 memory/outline.md
Step 3: 读取 memory/foreshadow.md
Step 4: 读取 memory/characters.md
Step 5: 调用 原创作家 → 开始创作
Step 6: 章节完成 → 触发 伏笔更新
Step 7: 调用 审核官
Step 8: 审核通过 → 保存 chapters/chapter{X}.md
Step 9: 同步 INDEX.md（如需）
Step 10: 自动进入下一章
```

## 🔄 四人组协作流程

```
系统启动
    ↓
【0】上下文管理器 → 初始化记忆库 → 同步INDEX
    ↓

【3】网文原创作家 → 章节创作 → 调用上下文恢复
    ↓ 【上下文恢复成功】
原创作家 → 完成章节 → 伏笔更新 → 调用审核官
    ↓
【4】网文审核官 → 审核 → 输出审核报告
    ↓
发现问题 → 精修 → 重审 → 循环
    ↓
通过 → 保存chapters/ → 同步INDEX
    ↓
自动进入下一章 → 调用上下文恢复
```

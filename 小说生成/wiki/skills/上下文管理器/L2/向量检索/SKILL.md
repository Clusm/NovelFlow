# 小说向量检索引擎

> **版本**：v1.0 | **来源**：用户框架+改造 | **定位**：上下文管理器增强模块

---

## 核心定位

| 维度 | 说明 |
|------|------|
| **作用** | 增强上下文管理器的语义检索能力 |
| **位置** | 上下文管理器 → L2/向量检索 |
| **调用时机** | 上下文恢复时、创作查询时 |
| **依赖** | novel-vector-search/vector.js |

---

## 核心能力

### 1. 文本向量化
```
输入：任意文本（设定/大纲/剧情）
输出：64维归一化向量
原理：字符哈希+模运算
```

### 2. 余弦相似度检索
```
输入：查询文本
输出：TopK相关记忆（带相似度分数）
过滤：支持按type/chapter过滤
```

### 3. 记忆库初始化
```
输入：项目memory文件夹
输出：全量设定加载到向量库
范围：project/world/characters/outline/foreshadow + chapters
```

## 集成方案

### 调用位置：上下文管理器 - 上下文恢复

```
原创作家开始章节
    ↓
上下文管理器 → 上下文恢复
    ↓
【新增】向量检索 → 语义匹配相关设定
    ↓
输出章节上下文 + 检索结果
```

### 检索流程

```
Step 1：加载项目向量库
Step 2：构建查询（当前章节/人物/剧情）
Step 3：向量检索（Top5）
Step 4：结果格式化
Step 5：附加到上下文输出
```

# 三道闸门架构

> **版本**：v2.0 | **定位**：防崩溃机制 | **来源**：联网学习

---

## 执行流程

```
创作前 → [Gate3] → [Gate1] → 创作 → [Gate2] → 完成
```

---

## Gate3：章间锚点锁定

| 项目 | 内容 |
|------|------|
| 目标 | 解决记忆漂移 |
| 动作 | 读取上章 + 提取结尾 + 对比本章大纲 |
| 矛盾 | 阻断，等待修复 |

**检测内容**：
- [ ] 人物状态一致（位置/情绪/伤势）
- [ ] 时间线连续
- [ ] 物品状态正确
- [ ] 事件正确承接

---

## Gate1：检索阈值断言

| 项目 | 内容 |
|------|------|
| 目标 | 确保检索返回足够上下文 |
| 阈值 | Layer≥3条 |

| 层级 | 最低阈值 | 内容 |
|------|---------|------|
| Layer1 世界观 | ≥3 | 设定/规则/地理 |
| Layer2 风格 | ≥3 | QA/禁忌/语义块 |
| Layer3 节奏 | ≥3 | 三线/情绪曲线 |
| Layer4 人物 | ≥3 | Bible/弧光/决策 |

**模板**：
```json
{ "characters": ["角色列表"], "arc_phase": true }
{ "locations": ["地点"], "factions": ["势力"], "rules": true }
{ "scene_type": "combat", "emotion": "tense" }
{ "pacing_goal": "quest_heavy" }
```

---

## Gate2：P0问题阻断

| 项目 | 内容 |
|------|------|
| 目标 | P0问题必须阻断 |
| P0定义 | 战力矛盾/设定违背/逻辑bug/OOC |
| P1 | 建议修复 |
| P2 | 可选优化 |

**处理**：
```
P0 → 必须修复 → 阻断
P1 → 建议修复 → 显示
P2 → 可选 → 显示
```

---

## 禁止项

| 禁止 | 正确 |
|------|------|
| 跳过Gate3 | 必须执行 |
| 阈值不足继续 | 补充检索 |
| P0不阻断 | P0必须修复 |

# WF-06-W2：场景级创作

> 正文工程手册 v2.0 · 场景级创作阶段（v2.0最大升级点）
> 上一阶段：[W1 写作前对接](WF-06_W1_写作前对接.md) | 下一阶段：[W3 章节精修](WF-06_W3_章节精修.md)
> 版本：v2.0 | 最后更新：2026-04-30

---

**目标**：不再一次性写完整章。按场景逐段写，每写一个场景前注入匹配的wiki知识卡片。

**核心机制**：场景类型 → 自动匹配wiki知识卡片（≤200字）→ 注入AI → AI写该场景 → 下一个场景。

**AI角色**：按wiki指导逐场景执行创作

**耗时**：每场景约1-2分钟

---

## Step 2.1：根据情节点拆分场景

AI从W1约束卡中的"四段情节点"拆出本章的场景清单。

**拆分规则**：
- 起（约500字） → 1个场景
- 承（约800字） → 1-2个场景
- 转-高潮（约1000字） → 1-2个场景
- 合（约500字） → 1个场景

**典型本章**：4-6个场景。

**示例**：
```
场景1：开场钩子 — 主角面对冲突/悬念（起）
场景2：事件推进 — 障碍出现/配角互动（承前半）
场景3：压力升级 — 冲突加剧/关键信息获取（承后半）
场景4：高潮爆发 — 爽点段落（转-高潮，本章核心）
场景5：收束钩子 — 成果+悬念（合）
```

## Step 2.2：逐场景创作（核心循环）

**每个场景的循环**：
```
1. AI判断场景类型（从以下9类中匹配）
2. 自动注入该类型对应的wiki知识卡片（≤200字）
3. AI按卡片指导写这个场景
4. 写完检查场景质量 → 通过则进入下个场景
```

## 场景类型 → wiki知识卡片匹配表

### 卡片 C-001：冲突/打斗场景

**触发条件**：场景涉及对抗/战斗/实力展示

**注入的wiki知识卡片**：

```markdown
## C-001 [冲突/打斗场景写法]

### 结构：五步节奏
1. 对方先展示实力/嘲讽（蓄压）
2. 主角被质疑/被压制（积累紧张）
3. 关键时刻，主角出手（爆发）
4. 结果揭晓（对方被打脸）
5. 旁观者反应（强化爽感）

### 爽感写法（从约束卡取爽感类型）
- 公平需求：蓄压要深（至少3个具体被欺压行为）→ 爆发要烈
- 优越需求：配角先展示"最好的"→ 众人以为天花板 → 主角降维打击
- 掌控需求：展示主角布下的局一步步收网
- 归属需求：有人站出来站主角这边

### 旁观者反应层次（必须写，这是爽感的放大器）
不屑 → 怀疑 → 震惊 → 恐惧 → 崩溃
每一步都要有具体角色的具体表现，不写"他们很震惊"，写"杯子从手里掉了"

### 禁止写法
- 不写"他感到愤怒"，写"他把杯子捏碎了"
- 不写"他很强大"，写"拳风把十步外的树叶震落"
- 不写"全场震惊"，写至少3个具体旁观者的不同反应
```

### 卡片 C-002：对话场景

**触发条件**：场景以角色对话为主（信息交换/试探/冲突/拉拢）

**注入的wiki知识卡片**：

```markdown
## C-002 [对话场景写法]

### 核心规则
1. 对话必须有潜台词——角色说A，真实意图是B
2. 每段对话配动作——说一句，配一个动作/表情，交替进行
3. 不同角色说话方式不同（口头禅/语速/用词档次）
4. 对话推动情节——不能为聊天而聊天

### 用动作替代情绪直白表述
❌ "他愤怒地说" / "她害羞地回答"
✅ "他把杯子重重搁在桌上"  /  "她低着头，手指绞着衣角"

### 对话节奏
- 紧张对话：短句为主，一来一回快节奏
- 试探对话：话说一半，留白
- 情感对话：一句台词后的沉默比话更有力

### 禁止写法
- 对话中插入大段背景解释
- 所有人说话风格一样
- "他说："她用反问的语气说："（副词堆砌）
```

### 卡片 C-003：情绪高潮场景

**触发条件**：场景目标是让读者产生强烈情绪（燃/感动/心疼/紧张/愤怒）

**注入的wiki知识卡片**：

```markdown
## C-003 [情绪高潮场景写法] 来自 读者心理学

### 四种核心情绪的触发机制
1. 紧张感：读者知道有危险但角色不知道 → "别开门！"心理
2. 期待感：读者知道角色有底牌但还没亮 → "快打脸！"心理
3. 满足感：读者陪角色经历了完整弧光 → "值了！"心理
4. 共鸣感：角色说出了读者想说的话 → "这就是我！"心理

### 情绪累积法则（来自 叙事技巧体系）
情绪必须有累积过程，不能凭空出现：
铺垫1（轻微触达）→ 铺垫2（加深）→ 铺垫3（临界点）→ 爆发

### 情绪写法
- 不写"她哭了"，写"她低着头，肩膀轻轻抖动。泪水滴在手背上，她伸手去擦，擦不完"
- 不写"他很激动"，写"他的手在发抖，声音也在发抖"
- 燃点：用短句堆叠，一句一个动作，节奏加快

### 留白的艺术
情绪最高点，留一句空白。不说透，让读者自己补。
例：他张了张嘴，最后什么也没说。转身走了。
```

### 卡片 C-004：系统/金手指展示场景

**触发条件**：场景涉及系统提示/面板展示/新能力解锁/技能使用

**注入的wiki知识卡片**：

```markdown
## C-004 [系统/金手指展示场景写法] 来自 金手指设计原理

### 五步展示法
1. 触发条件（完成任务/获得物品/境界突破）
2. 系统提示（一句话，不要长篇播报）
3. 能力/奖励描述（直观展示效果，不堆砌数值）
4. 主角反应（一句话，克制）
5. 周围人的视角（不知道系统存在，从第三方视角看"主角突然变强了"）

### 禁止写法
- 连续大段系统面板播报（AI最爱做的事）
- 数值堆砌（"力量+10，敏捷+5，智力+3..."一次展示1-2个）
- 系统解释过细（让读者自己理解系统，不替读者消化）

### 面板展示节奏
- 日常章节：1次面板展示/章（不超过5行）
- 高潮章节：可展示变化前后的对比（2次）
- 绝不：连续3段面板播报
```

### 卡片 C-005：过渡/铺垫场景

**触发条件**：场景位于高潮之间，或事件之间的衔接过渡

**注入的wiki知识卡片**：

```markdown
## C-005 [过渡/铺垫场景写法] 来自 节奏模组

### 节奏占比约束（根据章节整体节奏模式）
- 单峰型（日常章）：过渡/铺垫 ≤ 40%
- 多峰型（重要章）：过渡/铺垫 ≤ 25%
- 反转型（高潮章）：过渡/铺垫 ≤ 15%

### 过渡场景的功能（至少满足2项）
1. 信息传递（角色获取新情报）
2. 关系变化（角色之间关系微调）
3. 铺垫下一事件（种下"不对劲"的种子）
4. 情绪调节（上一段紧张后给读者呼吸空间）

### 禁止：纯"走路"场景
❌ "主角从A走到B，路上看到C，到了B"
✅ 在路上完成一次对话/一次内心活动/发现一个细节
```

### 卡片 C-006：章末钩子场景

**触发条件**：本章最后一个场景（合段）

**注入的wiki知识卡片**：

```markdown
## C-006 [章末钩子场景写法] 来自 开篇钩子模组

### 五种钩子类型（按约束卡指定的类型选择）
1. 反转式：结尾揭露一个读者没预料到的事实
2. 选择式：主角面临一个选择，读者想知道怎么选
3. 悬念式：揭示一个新问题/新威胁，读者想知道答案
4. 情感式：结尾的情感高潮，读者想看到后续发展
5. 信息差式：读者知道了角色不知道的信息，替角色着急

### 钩子写法公式
- 1句总结当前状态（收束本章）
- 1句揭露新信息/新问题（开启下章）
- 结尾留白（不写满，让读者想点下一章）

### ⚠️ 章末收束段硬约束 ⭐v2.1升级

#### 一收到底原则
**一章只有一次收束**。战斗结束→场景收束→钩子→**结束**。不要在收束后接"三天后/傍晚"再开一个新的抒情结尾段落。

| ❌ 多重收束 | ✅ 一收到底 |
|-----------|-----------|
| 大战结束→"新的时代开始了"→三天后校园→傍晚钟楼→"风吹过带着希望" | 大战结束→收束画面→钩子指向下一章→**结束** |
| 高潮→情绪收束→时间跳跃→又一段情绪收束→再一段抒情 | 高潮→情绪收束→1句暗示未来→钩子→**结束** |

**自查问题**：写完章末后，问自己——"如果本章在第N段结束，读者会觉得不完整吗？"如果答案是有多段都可以作为结尾，删到只剩最后那一段。

#### 字数硬上限
章末收束段（含钩子）总字数**不超过250字**。超过说明你在做多余的情感渲染。

#### 破折号使用约束
章末最后250字内，`——` 独立成行的次数**不超过2次**。破折号短行是"重音"，用多了等于没有重音。

### ⚠️ 避免重复的描绘性段落
不要在章末重复本章前段已经出现过的大段场景描写。如果战斗场景中已经出现过"世界树的残骸化为光点"，不要在结尾再用几乎相同的文字描述一遍。结尾的"看天空"是视觉锚点——用**1句新画面**而非复制前文的画面。

### ⚠️ 章末语义去重规则
章末最后200字内，每句话必须有**独立的推进功能**，禁止同一语义换句式重说一遍：

| ❌ 重复语义（同一意思说了两次） | ✅ 每句独立推进 |
|------------------------------|----------------|
| "他知道，从今天起，一切都不同了。从这一刻开始，他的人生将彻底改变。" | "他知道，从今天起，一切都不同了。门外的脚步声越来越近。" |
| "她不会再回头了。她决定永远离开这里。过往的一切，她不会再回头看。" | "她不会再回头了。行李箱的轮子碾过门槛，发出最后一声响。" |
| "这一战，他赢了。他终于证明了自己。他用实力告诉所有人，他赢了。" | "这一战，他赢了。台下鸦雀无声。" |

**自查方法**：写完章末200字后，逐句问"删掉这句，读者会丢失什么信息？"——如果答案是"不丢失任何信息"，则该句为语义重复，必须删除或改写。

### 禁止
- 不要凭空捏造和本章无关的钩子
- 钩子必须能从本章情节自然引出
- 不要在钩子里解释——解释放在下一章开头
- 🚫 章末最后200字内出现同义反复（如"他赢了...他成功了...他证明了自己"）
- 🚫 章末出现"时间跳跃后另开抒情段"
- 🚫 章末最后250字内 `——` 独立成行超过2次
- 🚫 章末重复复述本章前段已经出现过的场景描写
```

### 卡片 C-007：环境/氛围描写场景

**触发条件**：场景以环境引入/氛围渲染为主

**注入的wiki知识卡片**：

```markdown
## C-007 [环境/氛围描写场景写法] 来自 网文描写词汇速查

### 四维描写公式
1. 空间：远近/高低/大小 → 建立空间感
2. 光线：明暗/冷暖/颜色 → 建立情绪基调
3. 声音：环境音/人声/寂静 → 建立氛围
4. 气味/触觉：一个标志性感官细节 → 建立记忆点

### 环境=情绪
- 主角紧张 → 环境写"逼仄/昏暗/窒息"
- 主角释然 → 环境写"开阔/明亮/清风"
- 不直说主角心情，用环境映射

### 篇幅控制
- 纯环境描写：每次不超过100字
- 环境+动作结合：将环境细节融入角色行为
  ✅ "他踩着落叶往前走，枯叶碎裂的声音在空荡的巷子里格外清晰"
  ❌ "巷子很安静，地上有很多落叶，风吹过来很冷"
```

### 卡片 C-008：人物登场场景

**触发条件**：新角色首次出场，或重要角色再次登场

**注入的wiki知识卡片**：

```markdown
## C-008 [人物登场场景写法] 来自 叙事技巧体系

### 人设演绎式呈现
不说"他是个冷静的人"——让他做一个冷静的决定。
不说"她很高冷"——让她对别人的热情只回一个"嗯"。

### 首次登场三要素
1. 出场方式：从什么环境/情境中出现（暗示身份）
2. 标志性细节：一个动作/一句话/一个习惯（让人记住）
3. 其他角色反应：通过在场角色的反应暗示此人地位/实力

### 再次登场检查
- 性格基准线一致吗？（无OOC）
- 有变化吗？（人物弧光的推进）
- 读者能认出"这是谁"吗？（标志性细节复现或变化）

### 口头禅和习惯动作（每角色一个）
让人物有记忆点而不脸谱化的关键
✅ "他推了推眼镜才开口" — 每次都推眼镜
❌ 没有固定习惯，每次描写不同
```

```markdown
### 禁止
- 大段内心独白表白
```

### 卡片 C-010：人物内心/成长场景

**触发条件**：场景以角色内心活动、性格展现、人物弧光推进为主

**注入的wiki知识卡片**：

```markdown
## C-010 [人物内心/成长场景写法] 来自 人物塑造方法

### 五层展现原则
不做分析报告式的"他是一个XX的人"，用行为层层展现：
1. 表层（外貌/语言风格）：通过对话和动作习惯自然流露
2. 能力层（实力/专业）：在具体行动中展示而不旁白宣布
3. 性格层（行事逻辑）：通过"选择"展现——让角色在两个选项之间做出符合性格的选择
4. 动机层（欲望/恐惧）：通过内心独白或行为驱动透露，不直接"他想成为最强"，写他"半夜还在练"
5. 弧光层（变化轨迹）：写角色做出以前不会做的决定，通过行为变化证明成长

### 人物塑造三要素时刻
每个重要角色场景必须包含：
- 欲望：角色想要什么（当前场景的目标）
- 能力：角色能做什么（当前能用的手段/资源）
- 缺陷：限制是什么（不能太轻松拿到想要的东西）

### 内心活动写法
- 心理描写占比控制在15-25%
- 用角色的口吻思考（不是分析报告）
- 心理活动附着在具体行为上：不是"他想起了过去"，而是"他翻到那张旧照片，停了一下"

### 标志性细节强化
- 每场景至少呈现1个角色的标志性动作/口头禅
- 连续3个场景没有标志性细节 → 预警

### 禁止
- "他想…"开头的大段内心独白
- "他意识到了…"——改为具体行为变化
- 人物突然的、无铺垫的性格转变
```

## 附录：场景类型自动判定规则（AI用）

AI根据以下关键词自动判定场景类型：
- 包含"战斗/打/出手/出手了/一拳/攻击" → C-001 冲突/打斗
- 包含"说/道/问/答/告诉/聊"且对话占比>50% → C-002 对话
- 包含"哭/泪/笑/激动/热血/感动/愤怒/心疼" → C-003 情绪高潮
- 包含"系统/面板/叮/属性/技能/解锁/升级/奖励" → C-004 系统/金手指
- 两段高潮之间/地点转移/时间跳跃 → C-005 过渡/铺垫
- 本章最后一个场景 → C-006 章末钩子
- 包含"走进/踏入/来到/前方/远处/阳光/风吹"且是场景开头 → C-007 环境/氛围
- 新角色名首次出现/重要角色长时间后再次登场 → C-008 人物登场
- 涉及情感互动/肢体接触/关心/羁绊 → C-009 感情/羁绊
- 包含"想起/回想/内心/纠结/挣扎/决定/选择/成长/改变"且行动占比<50% → C-010 人物内心/成长

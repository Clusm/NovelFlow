**目标**：从四个独立视角对本章做最后一轮深度审查，确保写得好而不只是写得对。

**核心原则**：四个视角独立审查，确保没有盲区。

**参考工具文档**：[WF-03](WF-03_质量验证流水线.md)（旧版5步验证，保留字数+禁词扫描脚本）

## 视角一：事实一致性审查

**审查内容**：本章是否与大纲/设定/状态追踪保持一致？

| 审查项 | 方法 | 不通过的表现 |
|--------|------|------------|
| 大纲锚定 | 对照约束卡中"本章情节点"四段，逐段检查正文是否偏离 | 正文写了细纲没规划的情节；或细纲规划的高潮段落正文草草带过 |
| 落点达成 | 对照约束卡"本章落点"，正文是否达成了那个画面？ | 细纲设定的落点画面在正文中找不到对应段落 |
| 状态一致性 | 对照约束卡"关键约束"，主角身体状态/境界/位置是否与状态追踪一致？ | 状态追踪写"主角右臂受伤"，正文写他全力战斗 |
| 世界观铁律 | 对照约束卡"世界观铁律"，正文是否违反？ | 铁律写"秀才不可御空飞行"，正文写秀才御剑追击 |
| 字数达标 | 本章字数在2600-4500之间？（特殊剧情章可放宽） | <2600或>4500且无正当理由 |
| AI禁词清零 | `而是`=0, `因而`=0, 其他22项≤2 | 超标 |

**wiki知识注入**：`AI大纲一致性管理` — 事实一致性核对方法

## 视角二：读者体验审查

**审查内容**：以最挑剔的读者心态审视本章。

**AI使用的wiki知识**：
- `读者心理学` — 四种核心情绪触发机制
- `读者人格代入创作法` — 禁止分析框架，只用第一感受判断
- `爽感四大基石` — 爽感有效性评估
- `读者情绪地图` — 情绪目标达成检查
- `开篇钩子模组` — 钩子有效性评估

| 审查项 | 方法 | 不通过的表现 |
|--------|------|------------|
| 开头200字吸引力 | 只看开头200字，有继续读的欲望吗？ | 平淡，没有钩子，信息堆砌 |
| 爽感有效性 | 约束卡标注的爽感类型，正文真的"爽"了吗？不是空洞的"他很震惊"而是有层次的旁观者反应？ | "他很震惊"式直白 |
| 情绪触发 | 约束卡设定的情绪目标（该紧张/该燃/该感动），正文做到了吗？ | 高潮段落情绪平平 |
| 章末钩子有效性 | 看完章末，你想立刻点开下一章吗？ | 结尾平淡，随时可以弃书 |
| 节奏舒适度 | 有没有连续5段以上无信息增量的段落？（水文） | 有"注水"感 |
| OOC感 | 主角的行为是否符合设定的人设？有没有"这不像他会做的事"？ | 主角突然降智或圣母 |
| 对话自然度 | 角色对话是不是各有特色？还是所有人都像同一个人在说话？ | 对话模板化，缺少个性 |
| **读者人格第一感受**（⭐新增） | 完全代入读者身份，禁止分析框架，只回答：1)这一章最让我爽/感动/期待的是什么？2)有没有哪段让我想跳过去？3)看完结尾我想不想立刻看下一章？ | 回答不出吸引点 或 明确说出想跳过的段落 |

## 视角三：编辑精修审查

**审查内容**：以专业编辑视角审视文字质量。

**AI使用的wiki知识**：
- `叙事技巧体系` — 六大技法层
- `去AI化写作` — 六维检测（词/句/段/风格/信息密度/风格一致性）
- `网文描写词汇速查` — 描写质量

| 审查项 | 方法 | 不通过的表现 |
|--------|------|------------|
| 词汇质量 | 扫描AI禁区词是否清零 + 扩展禁区词（璀璨/熠熠生辉/宛如/深邃/凛冽/心中涌起一股/嘴角勾起一抹冷笑） | 有AI标配词汇 |
| 句式节奏 | 检查长短句交替；有无"首先其次最后"机械递进；有无"值得注意的是/毋庸置疑"套话 | 句长均匀，节奏单调 |
| 段落呼吸 | 检查段落长短是否参差不齐；关键情绪点是否有短句独立成段 | 每段150-200字等长 |
| 描写质量 | 情绪是否用动作表达（不是"他感到愤怒"而是"他把杯子捏碎了"）；感官描写是否到位 | 情绪直白外露，无感官细节 |
| 信息密度 | 每段是否有信息量或情绪量？有没有"废话段"？ | 有纯粹凑字数的段落 |
| 叙述视角 | 有无不必要的视角切换？旁白是否过多？ | 上帝视角频繁切换 |

## 视角四：连载一致性审查

**审查内容**：本章与前后章节的衔接是否紧密。

**AI使用的wiki知识**：
- `伏笔设计体系` — 伏笔操作核查 + 葡萄串链检查
- `AI多Agent写作流程管控` — CHECKPOINT同步

| 审查项 | 方法 | 不通过的表现 |
|--------|------|------------|
| 前章衔接 | 本章开头是否承接了前章结尾的时间/空间/情绪？前章钩子是否被本章承接？ | 前章结尾主角在城门口，本章开头在客栈（中间跳跃无过渡） |
| 后章铺垫 | 本章钩子的方向，是否与细纲中后章的情节点一致？ | 本章钩子暗示"A方向"，后章细纲写的是"B方向" |
| **相邻章结束语去重** ⭐新增 | 从W1约束卡中读取"前章结尾特征签名"，与本章结尾做三层面比对：①句式结构模式是否雷同？（如前章是"动作句→环境句→留白句"，本章不能完全相同）②关键意象词是否重复？（如前章结尾用了"门/夜色/脚步声"，本章避免复用）③钩子类型是否连续相同？（连续2章同一钩子类型→疲劳预警） | 句式结构模式相同 + 关键意象词重叠 ≥ 2个 或 连续2章使用同一钩子类型 |
| **本章结尾与本章前段自相似** ⭐新增 | 本章结尾的场景描写是否与本章前段（战斗段落/中间过渡段落）出现大段重复？——"世界树的残骸化为光点"这种大段场景描写不应在结尾复制前文的版本。检查方法：提取本章最后200字的核心意象词（≥3个），搜索本章前段是否出现过相同的意象词组合。 | 结尾核心意象词组合与本章前段重复 ≥ 3个 |
| 伏笔埋设登记 | 本章新埋的伏笔是否已记录？承诺回收章是否标注？ | 埋了伏笔但没登记ID |
| 伏笔回收验证 | 本章回收的伏笔，回收方式是否合理？读者能否想起"这是前面哪章的伏笔"？ | 回收太突兀，读者忘了有这个伏笔 |
| 人设连续性 | 主角/关键配角的性格基准线在本章是否有偏离？ | 出现不符合前文性格的行为且无合理解释 |
| 状态追踪更新 | 本章结束后，主角状态（实力/身体/位置/金手指）是否需要更新状态追踪？ | 主角本章突破境界但状态追踪未更新 |

## 🚪 门禁W4：全维验证通过

| # | 检查项 | 不通过标准 |
|---|--------|----------|
| 1 | 视角一：事实一致性全部通过？ | P0级不一致（大纲偏离/状态矛盾/铁律违反） |
| 2 | 视角二：读者体验核心项通过？ | 开头无钩子 或 章末无追更欲望 |
| 3 | 视角三：AI禁词扫描通过？ | 禁词超标 |
| 4 | 视角四：前章衔接+后章铺垫无断裂？相邻章结束语无雷同？ | 衔接断裂 或 句式结构/意象词/钩子类型重叠 |
| 5 | 字数在标准范围内？ | <2600或>4500 |

**不通过怎么办**：进入 [W5 修复闭环](WF-06_W5_修复闭环.md)。标注每个不通过项的优先级。

**通过标准**：全部 ✅ → 本章定稿，更新状态追踪，进入下一章。

# WF-02：初稿写作标准

> 本WF定义单章正文的初稿写作标准。

---

## 一、一次性长写原则

### 核心要求

**单章必须一次性写完，不得分多次写。**

原因：分段写会破坏叙事连贯性，导致风格不一致、前后矛盾、节奏断裂。

### 操作方法

1. 读完全部5份输入文档
2. 在脑子里过完全部场景
3. 闭眼30秒，整理叙事节奏
4. 一次性打开文件写入，从开头到结尾，中间不停顿
5. 写完才验证

### 字数标准

| 阶段 | 字数目标 | 说明 |
|------|---------|------|
| 初稿 | 4000-6000字 | 留扩写余地 |
| 定稿 | 2600-4500字 | 交付标准 |

初稿写完后再验证，不达标再扩写/删改。

## 二、禁止清单（AI痕迹检测）

### 24项AI检测词（阈值说明：标 `=0` 必须为零，标 `≤2` 为允许出现次数）

| 词汇 | 阈值 | 说明 |
|------|------|------|
| 而是 | =0 | 判断性连接词，AI最爱 |
| 因而 | =0 | 因果连接词，正式书面语，网络小说中不自然 |
| 仿佛 | ≤2 | 比喻副词 |
| 微微 | ≤2 | 程度副词 |
| 然后 | ≤2 | 顺序副词（对话外） |
| 最后 | ≤2 | 顺序副词（量词用法除外） |
| 然而 | ≤2 | 转折词 |
| 但是 | ≤2 | 转折词 |
| 因此 | ≤2 | 因果词 |
| 所以 | ≤2 | 因果词 |
| 不过 | ≤2 | 转折词 |
| 似乎 | ≤2 | 揣测词 |
| 如同 | ≤2 | 比喻词 |
| 宛如 | ≤2 | 比喻词 |
| 仿若 | ≤2 | 比喻词 |
| 恰似 | ≤2 | 比喻词 |
| 首先 | ≤2 | 顺序词 |
| 其次 | ≤2 | 顺序词 |
| 再次 | ≤2 | 顺序词 |
| 因为 | ≤2 | 因果词（对话外） |
| 于是 | ≤2 | 顺序词，人为制造因果感 |
| 次 | ≤2 | 量词用法可接受 |
| 不但 | ≤2 | 递进词 |
| 不仅 | ≤2 | 递进词 |

### 通过标准

- 所有 `=0` 项必须为零
- 所有 `≤2` 项不超过阈值
- 字数在2600-4500之间

## 三、叙事风格标准

### 去AI化核心原则

**AI写作的三大特征：**

1. **过度使用连接词**（然后、于是、因此、所以）
2. **过度使用副词**（微微、仿佛、似乎、显得）
3. **过度使用判断句**（是...的、是为了、意味着）

### 修复策略

| AI特征 | 替换方式 |
|--------|---------|
| 然后 → | 删掉，或用具体动作替代 |
| 仿佛 → | 删掉，直接写感官 |
| 是...的 → | 改为陈述句 |
| 似乎 → | 改为确定语气 |
| 于是 → | 用具体行为替代 |
| 因此 → | 用因果行为替代 |

### 感官优先原则

**好的描写：** 他推开门，风灌进来，冷得他一哆嗦。
**AI的描写：** 他推开门，一阵冷风仿佛刀子一样刮过来，让他感到似乎很冷。

## 四、单章结构标准

### 黄金三章节奏（通用）

| 位置 | 内容 | 字数占比 |
|------|------|---------|
| 章首（前200字） | 钩子/冲突引入 | ~7% |
| 章中（中间） | 情节展开/对抗/爽点 | ~75% |
| 章末（最后100字） | 悬念/转折/情绪高点 | ~18% |

### 场景分隔符

用 `---` 分隔不同场景，便于后续精确定位patch。

```
场景1内容
---
场景2内容
---
场景3内容
```

## 五、特殊场景写法

### 打脸场景

1. 对方先嚣张
2. 主角被质疑/被压制
3. 关键时刻，主角出手
4. 结果揭晓，对方脸被打
5. 旁观者反应

### 系统展示场景

1. 触发条件（完成任务/获得物品）
2. 系统提示音/界面
3. 能力/奖励描述
4. 主角反应
5. 周围人的视角（不知道系统存在）

### 感情场景

1. 环境描写（季节/天气/光线）
2. 两人互动（对话/动作）
3. 心理活动（不能说"他感到"）
4. 关键台词
5. 留白/悬念

## 六、初稿检查清单

```
□ 字数是否在2600-4500之间？
□ 所有=0项AI词是否为零？
□ 所有≤2项AI词是否在阈值内？
□ 7要素是否覆盖（至少3项有实质）？
□ 章末是否有钩子？
□ 与前章衔接是否顺畅？
□ 是否一次性写完？
□ 场景是否用---分隔？
```

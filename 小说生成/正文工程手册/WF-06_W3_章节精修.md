## 维度1：词汇层精修

**AI使用的wiki知识**：`去AI化写作` — AI高频禁区词表 + 词汇替换策略

### 1.1 24项AI禁词扫描（来自旧版WF-02，保留）

```python
# 扫描脚本
text = open('ChXX.md').read()
words = ['而是','因而','仿佛','微微','然后','最后','然而','但是','因此','所以',
         '不过','似乎','如同','宛如','首先','其次','再次','因为','于是',
         '次','不但','不仅','仿若','恰似']
for w in words:
    c = text.count(w)
    if c:
        print(f"⚠️ {w}: {c}")
```

| 词汇 | 阈值 | 超标处理 |
|------|------|---------|
| 而是/因而 | =0 | patch替换为"是"删除/重写句子 |
| 仿佛/微微/似乎/如同/宛如/仿若/恰似 | ≤2 | 删除或用具体感官替代 |
| 然后/最后/然而/但是/因此/所以/不过/于是 | ≤2 | 删除用动作替代 |
| 首先/其次/再次 | ≤2 | 删除用内在逻辑关联替代 |
| 因为 | ≤2 | 删除解释，直陈结果（对话内除外） |
| 不但/不仅 | ≤2 | 删除递进，直接写 |
| 次 | ≤2 | 确认量词用法保留，非量词删除 |

### 1.2 扩展AI禁区词（来自 去AI化写作·词汇禁区）

| 禁用词 | 替换方向 |
|--------|---------|
| 璀璨/熠熠生辉 | 用光线本身描述 |
| 宛如/宛若/犹如 | 删除类比，直接写 |
| 深邃（眼神） | 换成具体行为 |
| 凛冽/凛然 | 用物理感受替代 |
| 心中涌起一股... | 删除，附着在具体动作上 |
| 眼神中透露着... | 换具体眼神动作 |
| 嘴角勾起一抹冷笑 | 改为"笑了一下/笑了笑/没笑但眼神冷了" |
| 不由得/不由自主/下意识 | 删掉，保留动作本身 |
| 蓦然回首 | 删除或写具体动作 |

### 1.3 过度书面词汇扫描

这些词不硬性禁止，但出现3次以上则预警：
`显著/关键/值得注意的是/毋庸置疑/综上所述/显而易见`

### 1.4 章末语义去重扫描 ⭐v2.1升级

**针对问题**：章节结尾段出现同一语义换句式重复——不仅是相邻句子，还包括长距离的场景描述块重复和多重结尾症。

**扫描范围**：本章最后500字（原300字扩展到500字，覆盖更长的收束段落）。

#### 1.4.1 相邻句级重复（原1.4）

| 检测项 | 方法 |
|--------|------|
| 同主语连续句 | 连续3句以上以同一主语（他/她/主角名）开头 → 预警 |
| 同义动词堆叠 | "赢了/成功了/做到了/证明了"连续出现 → 预警 |
| 总结性排比 | 结尾出现"从今天起...从此刻开始...从今往后..."句式 → 预警 |
| 抒情泛滥 | 结尾连续2句以上纯抒情/感叹（不含具体动作或画面）→ 预警 |

#### 1.4.2 长距离场景描述块重复 ⭐新增

**检测方法**：将本章最后500字按3句话为一块做滑动窗口，逐块比对语义重叠度：

| 检测项 | 阈值 | 判定 |
|--------|------|------|
| 同一意象组合重复 | "蓝天+金阳" / "风吹+花香+春天" 等2个以上关键词组合出现在不同段落 | ❌ 场景块重复 |
| 同一环境描述段重复 | 两段相距>10行，但描述的物理场景相同（如"世界树残骸化为光点"出现两次） | ❌ 场景块重复 |
| 同一情绪收束描述重复 | "新的时代开始了" / "一切都结束了" 等总结性宣告出现≥2次 | ❌ 多重收束 |

**修复方式**：只保留**最后一次**出现的版本。前一次出现的删除或缩减为1句简短过渡。

#### 1.4.3 多重结尾症检测 ⭐新增

**问题**：章节读完觉得应该结束了，但后面又续了一段；再读完觉得这回该结束了，又续了一段。每一次都像结尾——这就是多重结尾症。

**检测方法**：

| 检测项 | 判定阈值 | 说明 |
|--------|---------|------|
| "收束宣告"出现次数 | "结束了/一切都/新的XX开始了/从此/终于"等收束语出现 ≥ 3次 | ❌ 多重结尾 |
| 独立收束场景段落数 | 最后500字内，能独立成为"结尾画面"的段落 ≥ 2个 | ❌ 多重结尾 |
| 时间跳跃后继续抒情 | 结尾出现明确时间跳跃（"三天后/傍晚/第二天"）后，又跟了一个抒情收束段 | ❌ 多余尾声 |

**修复方式**：
- 选择**1个最强有力的结尾画面**保留
- 其他收束合并到那1个中，或直接删减
- 时间跳跃后的内容，若非必须交代的信息，精简为1-2句暗示或直接删除

#### 1.4.4 破折号短行滥用检测 ⭐新增

**问题**：AI习惯用 `——` 独立成行制造"文艺感"，但连续大量使用反而造成节奏单调、矫饰感明显。

**检测方法**：

```
扫描最后500字中 "——" 独立成行的次数：
- ≥5次 → ⚠️ 预警
- ≥8次 → ❌ 硬性超标，必须压缩
```

**修复方式**：
- 保留2-3个最有冲击力的破折号短行
- 其余改写为标准句式（合流为正常段落或直接删除破折号）

---

**通过标准**：
- 相邻句级：章末500字内无同义反复 + 无连续3句以上纯抒情
- 长距离：无相同场景块重复出现 + 无相同情绪收束描述多次出现
- 多重结尾：收束宣告 ≤ 2次 + 独立收束场景段落 = 1个
- 破折号：`——` 独立成行 ≤ 4次

触发的修复：保留最有力的写法，删除重复和多余。若修改范围超过200字，回到W2 C-006场景重新注入钩子卡片重写章末。

## 维度2：句式层精修

**AI使用的wiki知识**：`去AI化写作` — 句式改造 + 打破机械感

### 2.1 长短句交替检查

- AI特征：每句话15-25字，等长均匀
- 人类特征：长句20-40字 + 短句3-8字交替

**检查方法**：逐句计算字数，连续5句以上都在同一字数区间 → 需要打破。

**打破方式**：
- 重要情绪点 → 短句独立成段
- 用一个超短句（3-5字）打破节奏
- 在长句后紧跟短句强调核心

### 2.2 "首先其次最后"机械递进检查

扫描本章是否出现"首先/其次/再次/然后/最后"等显性递进词。如果有，删除并用内在逻辑关联替代。

### 2.3 倒装、省略、插入

适当使用非常规句式，破坏AI的规整感：
- 倒装："真香啊，这碗面"
- 省略："他拿起杯子。喝了。放下。拿起筷子。"
- 插入："他——还是算了——没有说出口。"

**通过标准**：长短句有交替，无机械递进词。

## 维度3：段落层精修

**AI使用的wiki知识**：`去AI化写作` — 段落呼吸节奏控制

### 3.1 段落长短参差检查

- AI特征：每段150-200字，三段式结构高度一致
- 人类特征：有1行短段落（强调）、有8行长段落（铺陈）

**检查方法**：统计本章每段字数，如果全部落在150-200区间 → AI味重。

**修复方式**：
- 关键情绪点：用1行短句独立成段
- 复杂描写：保留长段落
- 对话：每句话独立成段
- 高潮段落：短段落密集，节奏快

### 3.2 标点符号的节奏功能

- "咚"/"啪"/"嗒"等拟声词单独成行，充当节奏鼓点
- 问号、感叹号独立成行，增强语气
- 省略号替代"沉默"类描述

**通过标准**：段落长短参差不齐，有关键位置的短段落。

## 维度4：节奏层精修

**AI使用的wiki知识**：`节奏模组` — 三种节奏模式的占比约束

### 4.1 节奏模式确认

从W1约束卡确认本章的节奏模式：

| 模式 | 约束 | 检查 |
|------|------|------|
| 单峰型（日常章） | 过渡/铺垫≤40%，高潮≥15% | 高潮占比是否达标？ |
| 多峰型（重要章） | 过渡/铺垫≤25%，至少2处小高潮 | 是否有至少2处情绪上升点？ |
| 反转型（高潮章） | 过渡/铺垫≤15%，高潮≥30% | 高潮是否占据了足够的篇幅？ |

### 4.2 节奏公式检查

```
压抑5% → 蓄势20% → 转机10% → 爆发15% → 余韵50%
```

检查本章是否符合这个基本节奏公式。

**通过标准**：节奏占比符合模式约束，有明确的情绪起伏。

## 维度5：钩子层精修

**AI使用的wiki知识**：`开篇钩子模组` — 钩子有效性评估

### 5.1 开头钩子检查

看本章前200字：
- 有没有明确的钩子？类型是否与约束卡一致？
- 读者看完200字知道"这是谁的故事/发生什么了/为什么要继续读"吗？

### 5.2 章末钩子检查

看本章最后100字：
- 有没有钩子？类型是否与约束卡一致？
- 钩子是否从本章情节自然引出？（不是硬凹）
- 有没有留白？（不写满，留一句让读者想点下一章）

**通过标准**：开头有钩子，章末有追更欲望。

---

## 维度6：情绪层精修

**AI使用的wiki知识**：`读者情绪地图` — 情绪触发技巧检查；`写作技巧100条` — 情绪描写20条（克制美学：不直白描写情绪，用细节烘托）+ 人物塑造20条（行为证明性格而非旁白宣布）

### 6.1 情绪线完整性

检查本章情绪是否形成一条完整的线：
```
起始情绪（开头）→ 中段变化（承）→ 高潮情绪（转）→ 收束情绪（合）
```

如果某一个环节缺失 → 补充。

### 6.2 情绪表达方式检查

| 禁止 | 正确 |
|------|------|
| "他感到愤怒" | "他把杯子捏碎了" |
| "她很伤心" | "她低着头，肩膀轻轻抖动" |
| "他很紧张" | "他的手心全是汗" |
| "她心中涌起一股暖流" | "她转过身，嘴角翘了一下" |

扫描本章是否还有上述禁止写法的残留。

## Step 5.2：逐个修复

按优先级逐个修复，每修复一个立即验证：

### 修复方式A：patch精准替换

用于禁词替换、句式修复等小范围修改。

```python
# 示例
text = open('ChXX.md').read()
text = text.replace('然后站起身', '站起身')
text = text.replace('嘴角微微扬起', '嘴角轻轻扬起')
open('ChXX.md', 'w').write(text)
```

**patch规则**（参考 [WF-05](WF-05_常见问题修复.md)）：
1. old_string必须唯一（取前后各3字符确保唯一性）
2. 修复后立即验证变化
3. 大段内容用execute_code

### 修复方式B：扩写三维法

用于字数不够或描写单薄时。

| 维度 | 方法 | 示例 |
|------|------|------|
| 感官描写 | 添加视/听/嗅/触/味觉 | "夜很静" → "夜很静。虫鸣声从草丛里传出来，远处有狗吠声。" |
| 心理活动 | 回忆/联想/内心独白 | "她比上次瘦了" → "她比上次瘦了。他记得第一次见她的时候..." |
| 环境描写 | 场景背景/氛围渲染 | "他走进大门" → "他走进大门。高墙黑瓦，门楼上挂着两个大红灯笼..." |

### 修复方式C：重写段落

用于大纲偏离、爽感空洞等结构性问题时。回到W2对应场景，重新注入wiki知识卡片，重写该段落。

### 修复方式D：章末结束语差异化 ⭐新增

用于W4视角四检测到相邻章节结束语雷同时。

**修复步骤**：
1. 读取前章结尾特征签名（约束卡第10节），确认雷同层面（句式结构/意象词/钩子类型）
2. 选择差异化策略：

| 雷同层面 | 差异化策略 |
|---------|-----------|
| 句式结构模式雷同 | 更换结尾句式的排列顺序（如前章是"动作→环境→留白"，本章改为"对话→动作→留白"） |
| 关键意象词重叠 | 更换意象域（如前章用"门/夜色"，本章改用"窗/晨光"或完全不同方向） |
| 钩子类型连续相同 | 查约束卡，本章后续剧情是否有更适合的钩子类型；或从W2 C-006卡片中另选一种钩子类型 |

3. 回到W2 C-006场景，重新注入钩子卡片，按新策略重写章末200字
4. 重写后跑W3维度1.4（章末语义重复扫描）+ W4视角四（相邻章结束语去重）验证

## Step 5.3：修复后复审

修复完成后，重新跑 W4 的受影响视角：

| 修复的问题类型 | 重新跑哪个视角 |
|-------------|-------------|
| 事实一致性/AI禁词/字数 | 视角一 |
| 读者体验（开头/爽感/钩子/情绪/节奏） | 视角二 |
| 编辑精修（词汇/句式/段落/描写） | 视角三 |
| 连载一致性（衔接/伏笔/人设/状态） | 视角四 |
| 相邻章结束语雷同 ⭐新增 | 视角四（聚焦"相邻章结束语去重"项） |

## Step 5.4：更新状态追踪 + 归档

复审通过后：

1. **更新状态追踪**：如本章主角实力变化/受伤/位置移动/金手指解锁新功能，更新 `05_第N卷_状态追踪.md`
2. **更新伏笔记录**：如本章新埋/回收伏笔，更新细纲和伏笔追踪
3. **归档本章**：`正文/第N卷/ChXX_标题.md` 定稿

# 正文写作工作流 - 主工作流

> 这是正文写作的**主入口**。每次写一章，按本工作流执行。

---

## 主工作流：单章写作标准流程

```
┌─────────────────────────────────────────────────────────┐
│  步骤1：写作前准备（WF-01）                             │
│  读取5份文档 → 7要素自检 → 场景清单                    │
└─────────────────────────┬───────────────────────────────┘
                          ▼
┌─────────────────────────────────────────────────────────┐
│  步骤2：初稿写作（WF-02）                               │
│  一次性长写 → 2600-4500字 → 禁止清单自检               │
└─────────────────────────┬───────────────────────────────┘
                          ▼
┌─────────────────────────────────────────────────────────┐
│  步骤3：质量验证流水线（WF-03）                         │
│  字数验证 → AI禁词扫描 → patch修复 → 扩写/删减 → 交付   │
└─────────────────────────┬───────────────────────────────┘
                          ▼
┌─────────────────────────────────────────────────────────┐
│  步骤4：交付检查                                         │
│  AI禁词全部达标 → 字数达标 → 下一章                     │
└─────────────────────────────────────────────────────────┘
```

---

## 快速启动（每章执行一次）

### 写新章节前

```bash
# 1. 确认当前进度
ls ~/项目目录/正文/第X卷/Ch*.md | sort -V | tail -3

# 2. 读细纲（当前章节）
# 根据文件名确定ChN，然后读细纲对应章节

# 3. 执行WF-01（写作前准备）

# 4. 一次性写完初稿

# 5. 执行WF-03验证流水线

# 6. 达标后继续下一章
```

## 工作流选择

| 场景 | 工作流 | 说明 |
|------|--------|------|
| 写新章节 | 00_主工作流.md（本文） | 走完完整流程 |
| 多章批量patch/扩写 | 批量修复_WF.md | 批量patch+扩写 |
| 单章详细操作手册 | 单章写作_WF.md | 配合本文使用 |

---

## 铁律

1. **不并行** — 一章写完验证通过，再写下一章
2. **不跳步** — WF-01 → WF-02 → WF-03 必须按顺序执行
3. **不妥协** — 验证不通过绝不写下一章
4. **先备份** — 大改之前先 `cp` 备份

| 文件类型 | 路径 |

| 细纲 | `{项目根目录}/细纲/第{卷号}卷细纲.md` |

> `{}` 内为占位符，根据实际项目替换。

## 验证检查清单（每章必须全部通过）

```
□ 字数：2600 ≤ 字数 ≤ 4500
□ AI禁词：=0项全部为零
□ AI禁词：≤2项全部在阈值内
□ 7要素：至少3项有实质内容
□ 章末钩子：存在悬念/转折
□ 与前章衔接：顺畅
□ 文件命名：Ch{NN}_{标题}.md
□ 下一章可以开始
```

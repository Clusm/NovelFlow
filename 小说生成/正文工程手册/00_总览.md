# 正文工程手册 v2.0

> 从"一次性长写+事后修补"到"逐场景创作+主动注入wiki知识"的5阶段正文写作流水线。
> 核心升级：wiki知识按场景类型自动匹配注入，不再是被动附录。
> 适用规模：500章+ / 多卷 / 万字级网文项目。
> 最后更新：2026-04-30

---

## 一、v2.0 主流程速查

```
W1: 写作前对接      W2: 场景级创作        W3: 章节精修
（读v2.0大纲产物）   （逐场景+wiki注入）    （六维打磨）
      ↓门禁W1             ↓门禁W2              ↓门禁W3

W4: 全维验证        W5: 修复闭环
（四视角深度审查）    （修复→复审→归档）
      ↓门禁W4             ↓门禁W5
  本章定稿，进入下一章
```

| 阶段 | 核心问题 | 人审要点 | 主文档 |
|------|---------|---------|--------|
| **W1 写作前对接** | 本章所有前提条件都满足了吗？ | 约束卡完整？状态无矛盾？ | [W1](WF-06_W1_写作前对接.md) |
| **W2 场景级创作** | 每个场景怎么写？wiki指导到位了吗？ | 每个场景质量通过？情感自然？ | [W2](WF-06_W2_场景级创作.md) |
| **W3 章节精修** | 有没有AI味？节奏对不对？ | 六维精修完成？钩子有效？ | [W3](WF-06_W3_章节精修.md) |
| **W4 全维验证** | 本章写得到底好不好？ | 四视角全部通过？ | [W4](WF-06_W4_全维验证.md) |
| **W5 修复闭环** | 审核发现的问题怎么改？ | 修复后复审通过？ | [W5](WF-06_W5_修复闭环.md) |

> ⚡ **大纲→正文衔接**：详见大纲工程手册P7末尾的 [衔接协议](WF-06_P7_审核修复闭环.md)（大纲产物移交清单/中断恢复/正文中发现大纲问题的处理机制）

## 二、核心升级：wiki知识主动注入

v1.0中wiki知识放在附录A-E，是"人来翻阅"的参考资料。v2.0改为**按场景类型自动匹配注入**。

| 场景类型 | 自动注入的wiki知识卡片 | 卡片大小 |
|---------|----------------------|---------|
| 冲突/打斗 | `爽感四大基石` — 写法要点+旁观者反应层次 | ≤200字 |
| 对话 | `网文描写词汇速查` — 动作替代情绪表达 | ≤150字 |
| 情绪高潮 | `读者心理学` — 情绪触发机制 | ≤150字 |
| 系统/金手指展示 | `金手指设计原理` — 展示规则五步法 | ≤150字 |
| 过渡/铺垫 | `节奏模组` — 当前节奏模式占比约束 | ≤150字 |
| 章末钩子 | `开篇钩子模组` — 5种钩子类型写法公式 | ≤200字 |
| 环境/氛围描写 | `网文描写词汇速查` — 四维描写公式 | ≤150字 |
| 人物登场 | `叙事技巧体系` — 人设演绎式呈现 | ≤150字 |
| 感情/羁绊 | `读者情绪地图` — 情感共鸣写法 | ≤150字 |
| 人物内心/成长 | `人物塑造方法` — 五层展现原则+塑造三要素 | ≤200字 |

完整匹配表见 [W2 场景级创作](WF-06_W2_场景级创作.md)。

## 四、v2.0 新概念速查

| 概念 | 说明 | 来源 |
|------|------|------|
| 写作约束卡 | W1产出。从v2.0大纲产物中提取的本章所有约束（落点/状态/铁律/伏笔/禁区词），AI写本章前必读 | v2.0新增 |

| 六维精修 | W3的精修框架：词汇→句式→段落→节奏→钩子→情绪，逐维打磨 | v2.0新增 |
| 四视角验证 | W4的审查框架：事实一致性/读者体验/编辑精修/连载一致性 | v2.0新增 |

## 六、铁律（所有场景通用）

1. **每章独立验证** — 写完立即走W3-W5，不达标不写下一章
2. **先读后动** — 不读完W1约束卡不动手写
3. **wiki知识必注入** — W2每个场景必须匹配对应的wiki知识卡片（10张卡片覆盖10类场景）
4. **章节号=文件名编号** — 禁止两者的不一致
5. **禁止并行写作** — 单章串行交付，逐章通过验证再写下一章
6. **卡文自愈** — 检测到卡文（思路冲突/设定冲突/走向模糊/衔接困难）→ 立即暂停 → 生成2-3套差异化方案 → 交由用户选择 → 确认后继续。禁止乱写凑数
7. **回退范围最小化** — 门禁失败只回退对应阶段
8. **备份优先** — 大改之前先 `cp` 备份

## 八、坑点速查

| 坑 | 现象 | 解法 |
|----|------|------|
| 字数不足2600 | 初稿只有700-1500字 | W2逐场景写时每场景有字数底线；不够则在W3用扩写三维法补足 |
| AI禁词超标 | 然后×5、最后×4 | W2注入禁区词卡片防患于未然；W3六维精修中扫描修复 |
| 大纲断联 | AI写正文不知道主角当前状态 | W1约束卡强制读取状态追踪 |
| 爽点空洞 | 写了"他很强"但读者无感 | W2注入对应爽感类型的写法要点卡片 |
| 降序mv重编号 | 文件互相覆盖，内容丢失 | 用Python升序处理（见WF-04） |
| patch非唯一 | old_string在文件出现多次 | 加前后3行以上上下文 |
| 写完不验证 | 继续写下一章，导致连环不及格 | 每章必须走完W3-W5再写下一章 |

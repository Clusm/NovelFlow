## 一、输入文档清单（5文档输入法）

写每章之前，必须读取以下5份文档的相关章节：

| 序号 | 文档 | 读取内容 |
|------|------|---------|
| 1 | 细纲/第一卷细纲.md | 当前章节（ChN）的7要素 |
| 2 | 细纲/第一卷细纲.md | 当前卷的卷纲（概述卷主线） |
| 3 | F_世界观背景与核心设定.md | 相关设定（势力/境界/物品） |
| 4 | 正文/第一卷/Ch(N-1)_前章标题.md | 前章结尾（衔接点） |
| 5 | 正文/第一卷/Ch(N+1)_后章标题.md | 后章开头（悬念回收） |

### 读取顺序

```
1. 读细纲ChN → 7要素（情节/爽点/钩子/冲突/金手指/情绪/伏笔）
2. 读卷纲 → 确认本章在卷中的位置和作用
3. 读世界观设定 → 确认本章涉及的势力/境界/物品定义
4. 读前章 → 确认衔接点（时间线/空间/人物状态）
5. 读后章 → 确认悬念埋点
```

## 二、7要素自检

每章必须覆盖以下7要素，至少3项有实质内容：

### 7要素定义

| 要素 | 含义 | 检查问题 |
|------|------|---------|
| **情节** | 本章核心事件是什么 | 有没有完整的事件链（起承转合）？ |
| **冲突** | 障碍/对抗/矛盾是什么 | 有没有让主角为难的点？ |
| **爽点** | 读者情绪释放点 | 打脸/逆袭/装逼/成就解锁？ |
| **金手指** | 系统/能力/知识展示 | 有没有系统提示/技能展示？ |
| **情绪** | 读者情感引导 | 愤怒/感动/热血/心疼？ |
| **钩子** | 章末悬念 | 章末最后一句是否留白/转折？ |
| **伏笔** | 后续剧情暗示 | 有没有埋下后续剧情的线索？ |

### 自检标准

- 7要素中至少3项有实质内容
- 情节必须有（完整事件链）
- 钩子必须有（章末悬念）
- 爽点至少1个（否则节奏偏平）

## 三、境界体系对照（如涉及修炼/能力升级内容）

| 境界/等级 | 描述 | 本章是否涉及 |
|-----------|------|------------|
| 初境 | 入门/初期 | □ |
| 中境 | 积累/成长 | □ |
| 高境 | 小成/突破 | □ |
| 巅峰 | 大成/临界 | □ |

> 境界名称应根据具体小说类型调整（修仙/武侠/异能/职场等），本章仅标注是否涉及。

---

### 场景清单提取

从细纲中提取本章场景，按顺序排列：

```
示例：
场景1：主角抵达新地点
场景2：与重要配角初次相遇
场景3：冲突爆发
场景4：主角面临考验
场景5：章末转折
```

每个场景写完用 `---` 分隔，便于后续patch定位。

---

## 五、写作前检查清单

```
□ 5份文档全部读取完毕
□ 7要素全部自检（至少3项有实质）
□ 场景清单排列完毕
□ 本章字数目标确认（2600-4500字）
□ AI禁词阈值确认（24项清单见附录A）
□ 准备开始一次性长写
```

## 六、常见失败模式

| 失败模式 | 原因 | 解法 |
|----------|------|------|
| 情节不完整 | 没读细纲就写 | 必须先读细纲ChN |
| 与前章断裂 | 没读前章衔接点 | 必须读前章最后3段 |
| 节奏太慢 | 场景太多无重点 | 合并场景，删除流水账 |
| 悬念没埋 | 没读后章开头 | 必须确认后章走向 |
| 境界错乱 | 没对照境界表 | 确认本章对应的境界描述 |

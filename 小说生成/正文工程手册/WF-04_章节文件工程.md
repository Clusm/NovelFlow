## 一、命名规范

### 标准格式

```
Ch{序号}_{章节标题}.md
```

示例：
```
Ch1_新标题.md
Ch25_新标题.md
Ch99_大结局.md
```

### 路径结构

```
正文/
└── 第一卷/
    ├── Ch01_童生试炼.md
    ├── Ch02_县试初捷.md
    └── ...
```

### 序号规则

- 序号从1开始，不补零（Ch1而非Ch01）
- 序号与内容严格对应（Ch10 = 第十章）
- 严禁两个文件序号相同

> ⚠️ 注意：Windows排序会将Ch10排在Ch2之前（字典序）。Linux排序（`sort -V`）可正确处理。如用Windows资源管理器操作，建议始终使用3位数补零（Ch001）以避免排序错乱。

## 二、安全重编号操作

### 危险操作：降序mv

**严禁**用降序mv重命名，会导致文件覆盖丢失。

```bash
# ❌ 危险操作（降序）
mv Ch10.md Ch09.md   # Ch09被覆盖
mv Ch09.md Ch08.md   # Ch08被覆盖
# ...以此类推，最后只有一个文件

# ❌ 危险操作（降序）
for i in {10..1}; do mv Ch$i.md Ch$(($i-1)).md; done
```

### 安全操作：升序mv

```bash
# ✅ 安全操作（升序）
for i in {1..10}; do mv Ch$i.md Ch$(($i+1)).md; done
# 先处理Ch1，再Ch2，最后Ch10
# 不会出现覆盖问题
```

### 最安全操作：Python重编号

```python
import os
import shutil

def renumber_chapters(folder_path, start, end, offset=1):
    """
    安全重编号。
    folder_path: 文件夹路径
    start: 起始序号（如1）
    end: 结束序号（如99）
    offset: 偏移量（如1表示全部+1）
    """
    # Step 1: 备份
    backup_dir = os.path.join(folder_path, '..', 'backup_chapters')
    os.makedirs(backup_dir, exist_ok=True)
    for i in range(start, end + 1):
        old_name = f'Ch{i}_*.md'
        for f in os.listdir(folder_path):
            if f.startswith(f'Ch{i}_'):
                shutil.copy(os.path.join(folder_path, f), backup_dir)
                print(f"备份: {f}")

    # Step 2: 收集文件
    files = []
    for i in range(start, end + 1):
        for f in os.listdir(folder_path):
            if f.startswith(f'Ch{i}_'):
                files.append((i, f))
                break

    # Step 3: 升序重编号（先处理大号）
    files.sort(reverse=True, key=lambda x: x[0])
    for old_num, filename in files:
        new_num = old_num + offset
        old_path = os.path.join(folder_path, filename)
        # 提取标题部分
        title = filename.split('_', 1)[1]
        new_path = os.path.join(folder_path, f'Ch{new_num}_{title}')
        os.rename(old_path, new_path)
        print(f"重命名: Ch{old_num} → Ch{new_num}")

renumber_chapters('/path/to/正文/第一卷', 1, 99, offset=1)
```

### 重编号验证清单

重编号完成后，运行以下验证：

```python
import os
import re

folder = '/path/to/正文/第一卷'
files = os.listdir(folder)
files = [f for f in files if f.startswith('Ch') and f.endswith('.md')]

# 检查序号连续性
numbers = []
for f in files:
    m = re.search(r'Ch(\d+)', f)
    if m:
        numbers.append(int(m.group(1)))

numbers.sort()
for i, n in enumerate(numbers):
    expected = numbers[0] + i
    if n != expected:
        print(f"❌ 序号断裂: 期望{numbers[0]+i}, 实际{n}")

print(f"✅ 共{len(numbers)}章, Ch{numbers[0]}-Ch{numbers[-1]}")
```

## 三、备份策略

### 原则

**大改之前必备份。**

| 操作类型 | 备份要求 |
|----------|---------|
| 批量patch（5章以上） | 先备份整章 |
| 章节重写 | 先cp备份 |
| 章节重编号 | 用Python自动备份 |
| 合并章节 | 先备份两个文件 |

### 备份命令

```bash
# 单文件备份
cp Ch25_新标题.md ~/备份/Ch25_新标题_$(date +%Y%m%d).md

# 整卷备份
tar -czf backup_第X卷_$(date +%Y%m%d).tar.gz ~/项目目录/正文/第X卷/
```

## 四、章节错乱诊断

### 常见错乱类型

| 类型 | 现象 | 原因 |
|------|------|------|
| 编号断裂 | Ch01/Ch03，但Ch02不见了 | Ch02被误删/重命名 |
| 内容错位 | Ch05的内容是Ch06的 | mv时顺序搞反 |
| 时间线倒退 | 后写的章节反而时间更早 | 写错顺序没检查 |
| 命名重复 | 两个Ch10文件 | 重编号时未去重 |

### 诊断流程

```
1. 列出所有章节文件
2. 提取文件名中的序号
3. 检查序号连续性
4. 抽检前3章和后3章内容
5. 对照细纲检查剧情走向
```

### 修复原则

- 先备份
- 先诊断再动手
- 小改小动，不大拆大建
- 修复后立即验证

## 一、倒果为因写作法

### 核心原理

**先写结果（果），再写原因（因）。**

读者先看到震撼的结果，产生好奇，再通过解释性文字揭示原因 → 制造悬念，增强阅读快感。

### 公式

```
1. 果（震撼结果）→ 2. 因（解释原因）→ 3. 细节补充
```

### 示例

```
❌ 普通写法：
他刻苦读书，终于考上了举人。全村人都来祝贺。

✅ 倒果为因写法：
"恭喜陈老爷！"

全村人跪在他家门口，黑压压的一片。他穿着举人的 robes，站在门口，手都在抖。

他想起了十年前的那个夜晚。他爹临死前，拉着他的手说："轩儿，爹这辈子没能耐，但你要记住——读书才能改命。"

从那天起，他每天鸡叫就起床，砍柴、挑水、种地，晚上在油灯下读书。十年，从未间断。

终于，他成了举人。
```

### 适用场景

| 场景 | 用法 |
|------|------|
| 成就达成 | 先写庆祝场面，再揭示背后付出 |
| 打脸 | 先写对方震惊，再解释主角怎么做到的 |
| 感情 | 先写重逢场面，再揭示思念之苦 |

## 二、起承转合

### 基础定义

| 阶段 | 定义 | 内容要求 |
|------|------|---------|
| 起 | 开端 | 抛出问题/建立场景 |
| 承 | 发展 | 情节推进，障碍出现 |
| 转 | 转折 | 关键变化，高潮来临 |
| 合 | 结尾 | 问题解决/悬念留下 |

### 单章起承转合

```
起（前200字）：冲突引入，抛出悬念
承（中间段）：障碍升级，节奏加快
转（章中后段）：关键转折，爽点爆发
合（最后100字）：情绪高点或悬念留白
```

### 三章起承转合（单元结构）

```
第N章（起）：建立新冲突
第N+1章（承）：冲突升级，障碍加剧
第N+2章（转+合）：高潮爆发，悬念解除/新悬念埋下
```

## 三、叙事六层技法

### 六层定义

| 层次 | 内容 | 示例 |
|------|------|------|
| 1. 动作 | 具体行为 | 他推开门 |
| 2. 对话 | 人物语言 | "你来了？" |
| 3. 感官 | 五感描写 | 风灌进来，很冷 |
| 4. 心理 | 内心活动 | 他心里一沉 |
| 5. 反思 | 理性分析 | 他意识到不对 |
| 6. 元叙事 | 作者旁白 | 他不知道的是 |

### 自然叙事顺序

**最佳：** 动作 → 对话 → 感官 → 心理 → 反思
**次佳：** 感官 → 动作 → 对话 → 心理

### AI化叙事顺序（避免）

```
❌ 心理 → 对话 → 动作（太慢，节奏拖沓）
❌ 反思 → 心理 → 感官 → 动作（过于分析化）
❌ 元叙事 → 心理 → 感官（破坏沉浸感）
```

### 修复示例

```
❌ AI化：
他感到非常紧张，仿佛心里压着一块大石头。他想逃走，但他知道不能。他的心跳加速，他的手在颤抖。

✅ 自然叙事：
他转身就走。

"站住！"身后有人喊。

他没停，脚步反而更快了。
```

## 四、叙事速度调控

### 加速（快节奏场景）

- 短句为主
- 减少描写
- 动作连续
- 时间压缩（"一眨眼"、"转眼"）

### 减速（慢节奏场景）

- 长短句结合
- 增加感官/心理描写
- 环境渲染
- 时间拉伸（"足足等了一个时辰"）

## 五、视角运用

### 主视角限制

主角知道什么，读者就知道什么。主角不知道的，读者也不知道。

```
✅ 正确：他没有收到回信。（读者同样不知道原因）
❌ 错误：她其实是想回信的，只是被二叔拦下了。（读者知道但主角不知道）
```

### 视角切换

```
场景：主角vs反派

从主角视角写：紧张、压力
从反派视角写：傲慢、阴谋
从旁观者视角写：震惊、议论
```
